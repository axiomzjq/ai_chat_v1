1. 总库说明

本总库基于用户上传的 6 份个人IP与文案技巧资料整理合并而成。

本总库的整理目标不是精简总结，而是为后续智能体调用建立一个结构清晰、可检索、可扩展、且完整保留原始信息的主文档。

本总库由两部分组成：第一部分为调用导航与整理归类，帮助快速检索；第二部分为来源文档完整整合正文，确保原始信息无删减保留。

一、文件定位与适用边界

本文件是“抖音个人 IP 向 ToC 文案创作技巧库”，用于第四阶段文案生成智能体在已锁定 ToC 主路线后调用。

本文件不是通用文案百科，也不是平台运营课。它的作用是告诉智能体：在抖音信息流竞争下，个人 IP 的 ToC 文案该优先抓什么、怎么写、哪些不能写、信息不足时如何降级。

调用原则：先服从用户本人、前序定位与真实资料，再调用本技巧库；不得让技巧压过人、压过事实、压过路线。

二、抖音个人 IP ToC 的核心传播逻辑

核心逻辑

智能体调用说明

首屏抓取

先留住人，再谈说服。前 3 秒要给冲突、结果、反差或强代入，不能慢热。

个人感优先

用户看的是“这个人”，不是标准答案。文案必须有经历感、判断感、口语感。

情绪与代入

ToC 更重情绪共鸣、场景代入、身份认同与变化感。

变化结果

要让用户看到“前后差异”“卡点与转折”，而不只是抽象道理。

节奏推进

每 8–12 秒至少要出现新的信息点、情绪点或转折点，避免长时间解释。

轻转化

抖音个人 IP 前期更适合先建立熟悉感与信任，再做轻行动引导，避免过早强卖。



三、个人 IP 核心表达逻辑

优先级一：真实经历。优先调用用户真实经历、真实卡点、真实变化，而不是凭技巧虚构故事。

优先级二：真实判断。个人 IP 需要“我为什么这样想”的判断感，而不只是把平台流行句式拼起来。

优先级三：真实口吻。尽量保留用户原有表达、句式、语气、停顿感和价值观，不要统一写成“网感套话”。

优先级四：真实边界。共鸣不是卖惨，情绪不是表演，转折不是强行逆袭，故事不是鸡汤模板。

四、抖音首屏抓取规则

前 3 秒任务

给冲突、给反差、给结果、给强代入，至少完成其中一项。不得用两句以上空铺垫。

前 8 秒任务

让用户明确“这条跟我有什么关系”，建立继续看的理由。

前 15 秒任务

给出推进信息：故事背景、关键卡点、反常识判断、结果落差或新问题。

六、中段留人规则

每 8–12 秒至少要有一个新信息点、情绪点、反差点或转折点，不能连续两段都在抽象解释。

优先使用“场景—情绪—卡点—判断—变化”的推进顺序，而不是长时间铺背景。

如果正文进入纯方法讲解，必须穿插具体经历、例子、对比或一句能被记住的判断。

中段任务不是“解释清楚”，而是“让人继续愿意看”。

类型

适用场景

共鸣收尾

人设建立、情绪沉淀

让用户觉得“这话像真的”，不急着转化。

观点型、经验型内容

用一句可截图判断收住，提升记忆点。

八、抖音平台适配规则

标题 / 封面 / 开头三件套要协同：标题负责点击预期，开头负责兑现预期，正文负责留人。

文案开头应尽量短句、口语、直给，不宜使用长书面句。

适当点出用户会搜的人群、场景、问题与结果，但不要堆关键词影响自然表达。

拍摄形式要与文案节奏匹配：怼脸口播适合判断型与实话型，边展示边讲适合变化型与过程型。

九、风险边界与降级逻辑

硬边界：不得夸张承诺；不得文不符题；不得为了共鸣虚构故事；不得为了结果伪造案例；不得在信息不足时硬写“逆袭式结果稿”。



缺痛点：降级为轻共鸣稿，不写强转化。

缺真实经历：不写完整反转故事，可写判断型或状态型文案。

缺结果证明：不写绝对化变化感，改写成“我目前更认可/更有感的是……”。

缺表达风格：优先使用平衡型口语，不强行模仿用户。

十、反模板化黑名单

不要把所有开头都写成“你是不是也……”

不要所有稿子都套“痛点—解决方案—关注我”三段式。

不要所有故事都写成“以前很惨—后来逆袭”。

不要连续三句都像网文或鸡汤，不像真人说话。

不要为了流量过度情绪化，导致与用户本人不符。

十二、供智能体调用的规则摘要

调用问题

优先判断

这条是不是像用户本人？

先看经历、判断、口吻，再看技巧。

这条适合抖音吗？

先看前 3 秒能不能抓住人，再看中段能不能留人。

信息不够怎么办？

优先降级，不要硬写强结果。

什么时候用强钩子？

有真实场景、冲突、结果支撑时。

什么时候不能强转化？

冷启动、弱信任、缺证据时。

【归纳结论】这套资料最明显的倾向，不是“纯流量型情绪爆点”，而是“以信任型内容为主、流量型钩子为辅、转化型机制承接为尾”的组合打法。四张表里“信任型”占比都明显最高，说明这批样本更适合沉淀为个人IP长期创作技巧，而不是一次性硬广话术。

四份资料组合后，覆盖了 IP 文案创作的哪些环节

【资料直接体现】开头环节：结果承诺、反常识、数据冲击、身份代入、情绪悬念、权威背书、挑战式提问等。

【资料直接体现】中段推进：案例展开、对比论证、类比解释、路线图拆解、榜单式分点、从经历到方法论的升维。

【资料直接体现】结尾环节：使命收口、口头禅收口、互动口子、评论区承接、关注/私信转化。

【归纳结论】人设层面：这批资料高度偏好“有反差、吃过苦、见过事、能把经验讲成方法”的IP表达。

【归纳结论】商业层面：ToB/老板IP更依赖责任、机制、案例、客户托付；ToC/个人IP更依赖成长、情绪、身份代入、可直接获得感。

2. 四份资料逐份拆解

2.1 爆款金句表拆解

【资料直接体现】金句表共有 28 条，名称覆盖产品方法论、AI工具、老板IP、人设成长、职业故事等方向，其中“信任型”金句占绝对多数。说明这批金句的核心作用，不是单纯制造热闹，而是做观点压缩、价值收束、身份表达与信任增强。

【资料直接体现】高频写法一：把复杂判断压缩成一句绝对表达。例：“AI使用真正的门槛从来不是提示词，而是你有没有自己的素材判断和表达系统。”

【资料直接体现】高频写法二：用对立句式制造记忆点。例：“人生不是轨道，是旷野”“温度没有技巧，而是把真实做到极致”。

【资料直接体现】高频写法三：把身份与命运感写进去。例：“我生来是要看世界的”“追光的人”。

【资料直接体现】高频写法四：把商业逻辑说成人话。例：“把人情味变成顾客愿意托付的理由”。

【归纳结论】金句表最有价值的地方，是它帮你解决“观点如何被记住”而不是“观点如何被完整解释”。

2.2 爆款开头表拆解

【资料直接体现】开头表共有 37 条，覆盖“口播·流量 / 信任 / 转化”“访谈·信任”“人设开头”等多种场景。仅从标题命名就能看到，开头打法高度集中在结果承诺、反差、反常识、数据冲击、情绪悬念、权威背书、提问挑战、口碑背书几个方向。

【资料直接体现】结果承诺型：如“看完这个视频，你将变成一个有三年工作经验的应届生”“1分钟之后，你会有无限创作爆款的能力”。

【资料直接体现】反常识纠偏型：如“用AI写出顶级文案的关键根本不是提示词”“大部分人用不好AI的核心原因，就是把AI变成了另一个维度的自己”。

【资料直接体现】对照冲击型：如“50年前我在猪圈长大，50年后我一年产值突破两个亿”“从月薪1500买不起社保到高级合伙人”。

【资料直接体现】反差显真型：如“别人以为我豪车豪宅…但真实的我凌晨4点失眠”“这是我拍了一年的老板，他在镜头前还是没有学会表演”。

【归纳结论】这批开头真正厉害的点，不在“新奇词汇”，而在“第一句先让用户完成心理站队”：要么觉得自己会得到结果，要么觉得自己一直搞错了，要么被人物命运/反差卷进去。

2.3 爆款模板表拆解

【资料直接体现】模板表共有 29 条，其中既有人设故事模板，也有老板IP模板、AI方法论模板、转化型本地服务模板。模板表最有价值的地方，是把“情绪、经历、案例、方法、收口”串成可以复用的骨架。

【资料直接体现】人设故事模板常见骨架：低起点/自卑/否定 → 关键转折 → 小胜利 → 大坠落 → 自救/翻盘 → 口头禅或使命收口。

【资料直接体现】老板IP模板常见骨架：低调反差 → 吃苦出身 → 创业踩坑 → 结果证据 → 顾客为什么信你 → 方法论升华。

【资料直接体现】方法论模板常见骨架：先掀翻旧认知 → 再举常见错误 → 再给类比/案例 → 再给步骤系统 → 最后升维到更底层能力。

【资料直接体现】转化模板常见骨架：口碑/背书 → 反套路 → 机制透明 → 行动号召。

【归纳结论】模板表不是在教“怎么写一段好句子”，而是在教“怎么把一句话的吸引力，顺延成整篇内容的完成度”。

2.4 爆款文案表拆解

【资料直接体现】文案表共有 37 条，但最能支撑细拆的完整原文，集中在 4 条：应届生经验包装、AI文案高手、豆包一分钟原创文案、去提示词化内容系统。它们恰好能把“结果承诺型”“反常识型”“步骤教程型”“系统方法论型”四条主线跑清楚。

【资料直接体现】“如何成为有3年工作经验的应届生”通过“身份升级 + 真实经历背书 + 模块化经验投喂”建立强获得感。

【资料直接体现】“如何用好AI 当个文案高手”用“反常识判断 + 错误示范 + 做菜类比 + 样本投喂”把抽象方法讲透。

【资料直接体现】“用豆包一分钟写出短视频爆款原创文案”用“极短时间承诺 + 三步演示”降低门槛，适合工具/教程类内容。

【资料直接体现】“用AI写出顶级文案的关键根本不是提示词”把“素材、提示词、审稿”三者关系重新定义，并给出四步系统。

【归纳结论】文案表对技巧库最大的价值，是它证明了：真正有效的口播，不是只靠一句狠话，而是靠“第一句钩住—中段让人信—最后给出口和动作”的完整闭环。

3. 跨资料共性规律总结

【归纳结论】爆款文案最常见的底层逻辑：先用结果/冲突/反差抓停留，再用经历/案例/类比做解释，最后用一句能记住的话完成收口。

【归纳结论】爆款开头最常见的抓人方式：结果承诺、认知打脸、时间/数据对照、身份跃迁、情绪悬念、挑战式提问。

【归纳结论】爆款金句最常见的表达特点：短、绝对、对立、可复述，且往往不是“描述事实”，而是“替用户下判断”。

【归纳结论】爆款模板最常见的结构骨架：经历线 + 冲突线 + 证据线 + 方法线 + 收口线同时存在，而不是平铺讲述。

【归纳结论】爆款文案最常见的情绪推进路径：被钩住 → 被验证 → 被说服 → 被鼓动行动。

【归纳结论】爆款内容最常见的人设塑造方法：不是直接喊“我很厉害”，而是让用户看见“我吃过什么苦、见过什么事、为什么有资格说”。

【归纳结论】爆款内容最常见的信任建立方法：量化结果、具体流程、行业黑话、真实代价、反套路机制、顾客为什么买单。

4. IP文案创作技巧总库

下面的技巧总库，尽量按照你要求的统一结构展开。每条技巧都优先给出来自四份资料的直接依据，再补归纳层解释。

A1. 结果承诺 + 身份升级开头

技巧定义：先不讲过程，先告诉用户“看完后你会变成谁 / 拿到什么结果”。

核心作用：强停留，尤其适合教程、经验投喂、能力包装类内容。

适用场景：求职经验、工具教学、方法论、课程导流。

更适合：ToC 为主，ToB 也可在“老板/团队会拿到什么结果”时使用。

资料依据：【资料直接体现】开头表《看完这个视频，你将变成一个有三年工作经验的应届生》；模板表《强结果承诺+身份升级型人群开头》；文案表对应完整原文。

背后原理：用户不是先关心你会讲什么，而是先关心“我能不能直接拿走”。身份升级比知识承诺更强，因为它触发的是自我投射。

常见句式：看完这条，你会___；这条视频会让你从___变成___；1分钟之后，你就能___。

使用步骤：先给结果 → 再补身份升级 → 再用个人背书稳住可信度 → 再展开模块化内容。

注意事项：结果承诺要和正文交付强匹配，否则会像标题党。

容易踩的坑：只喊结果、不补个人凭证；承诺太空泛；结果与适用人群不匹配。

可直接调用的方法：把“学会某个方法”改写成“看完你会变成一个更有___的人”。

A2. 反常识纠偏开头

技巧定义：先否掉用户习以为常的认知，再抛出更底层的新判断。

核心作用：制造“我可能一直都搞错了”的停留感。

适用场景：方法论、行业判断、AI内容、认知升级类IP。

更适合：通用，尤其适合 ToB 创始人 / 方法型 IP。

资料依据：【资料直接体现】开头表《用AI写出顶级文案的关键根本不是提示词》《大部分人用不好AI的核心原因，就是把AI变成了另一个维度的自己》；文案表对应两条完整原文。

背后原理：认知被挑战时，用户会自动往下看“那正确答案到底是什么”。

常见句式：真正关键根本不是___；大部分人做不好___的核心原因是___；你以为问题在___，其实问题在___。

使用步骤：先打掉旧认知 → 立刻举错误示例 → 指出失败根因 → 给新路径。

注意事项：反常识不是为了抬杠，而是为了把用户带到更底层的解释。

容易踩的坑：只有观点，没有后续证明；语气太像情绪输出，缺乏论证。

可直接调用的方法：把你想讲的方法，先改写成“问题不在A，问题在B”的形式。

A3. 时间 / 数据对照开头

技巧定义：用过去与现在、低点与高点、单点数字与巨大结果形成强冲击。

核心作用：让人物故事或结果型内容快速立起来。

适用场景：老板故事、逆袭案例、职业成长、项目结果。

更适合：ToB 老板IP、老板故事、人设故事。

资料依据：【资料直接体现】开头表《50年前我在猪圈长大，50年后我一年产值突破两个亿》《从月薪1500买不起社保到高级合伙人》；模板表对应老板人生模板、律师人设模板。

背后原理：数字和时间对照会给用户一种“跨度感”，跨度越强，故事越像真的。

常见句式：X年前我还___，现在我已经___；从___到___；别人看到的是___，真正发生的是___。

使用步骤：先甩跨度 → 再给人物名片 → 再补关键坎和关键证据。

注意事项：数字必须可被正文接住，不能只是一句夸张标题。

容易踩的坑：只有结果没有过程；只有结果没有代价，故事会飘。

可直接调用的方法：先把你/案例的“最差状态”和“当前结果”写成一组对照句。

A4. 反差显真开头

技巧定义：先抛一个外界印象，再用真实处境反打，拉出人物层次。

核心作用：去油腻、去表演感，让用户更快相信“这是活人”。

适用场景：老板IP、人设故事、二代接班、幕后视角。

更适合：ToB 老板IP与有真实经历的个人IP。

资料依据：【资料直接体现】开头表《别人以为我豪车豪宅…但真实的我凌晨4点失眠》《这是我拍了一年的老板，他在镜头前还是没有学会表演》；模板表《二代接班老板模板》《老板IP故事模板》。

背后原理：用户天然戒备“过度体面的人设”。反差一出来，真诚感会比漂亮标签更强。

常见句式：别人以为___，但真实的我___；看起来___，做起来___；外面看到的是___，背后其实是___。

使用步骤：先写外界误判 → 再给真实状态 → 再放入人物责任/压力/温度。

注意事项：反差要服务于真实性，而不是为了卖惨。

容易踩的坑：一味苦情；反差过大但没有证据。

可直接调用的方法：先列出外界对你的三个误解，再选一个最强的做开头。

A5. 挑战式提问开头

技巧定义：用“你敢不敢 / 你会不会 / 你认命吗”把用户直接拉进选择题。

核心作用：快速建立代入感，尤其适合路线图和成长型内容。

适用场景：成长计划、人生系统、逆袭叙事、使命型视频。

更适合：ToC 成长型 IP，也适合操盘手、人设故事。

资料依据：【资料直接体现】开头表《如果此生注定平凡，你会认命吗？》《5个月敢不敢重塑人生系统》；模板表《IP操盘手故事模板》《今年8月拥有全新人生系统》。

背后原理：提问不是为了问答案，而是为了让用户在心里先站队。

常见句式：如果___，你会___吗？你敢不敢用___个月把自己变成___？

使用步骤：先提出命题 → 再告诉他“这条内容为什么与你有关” → 再给路径。

注意事项：问题要具体，不要做空泛鸡汤提问。

容易踩的坑：提问完立刻跑题；问题太大，回答无法落地。

可直接调用的方法：把你的内容主题先改写成一个带代价感的提问。

A6. 情绪共鸣 + 悬念开头

技巧定义：先点中一类普遍焦虑或遗憾，再抛问题和转机。

核心作用：让用户觉得“你知道我在担心什么”。

适用场景：时代机会、AI焦虑、人生阶段、行业变化。

更适合：通用。

资料依据：【资料直接体现】开头表《黄金时代机遇再现》《AI发展下普通人不止失业》；模板表对应焦虑共鸣、危机警示类口播模板。

背后原理：先共情，再给方案，用户更容易接受你后面的观点。

常见句式：很多人正在担心___；你有没有觉得___；这可能是你最后一次___。

使用步骤：先共鸣焦虑 → 再抛悬念/问题 → 再给三步方案或新认知。

注意事项：共鸣之后必须给方向，不然会停在焦虑消费。

容易踩的坑：全篇制造危机，不提供出口。

可直接调用的方法：先写一句用户最近最真实的担心，再补一句“但机会还在___”。

B1. 观点压缩金句

技巧定义：把一段复杂论证压缩成一句可复述判断。

核心作用：提升记忆度、传播度、截图率。

适用场景：结尾收束、中段升维、封面文案、标题提炼。

更适合：通用。

资料依据：【资料直接体现】金句表《AI使用真正的门槛从来不是提示词…》《温度没有技巧，而是把真实做到极致》。

背后原理：用户不会完整记住你讲过什么，但会记住你替他做出的那个判断。

常见句式：真正决定___的，不是___，而是___；没有___，只有___；你缺的从来不是___。

使用步骤：先把长论证写出来 → 再抽出其中最底层的一组关系 → 压成一句话。

注意事项：压缩后的句子要保留因果关系，不要只剩情绪。

容易踩的坑：为了短而短，导致空洞。

可直接调用的方法：用“不是A，而是B”重写你的段落结论。

B2. 对立句式金句

技巧定义：用两端对立制造节奏和记忆点。

核心作用：更容易被评论区复述、做标题、做封面。

适用场景：价值观表达、方法纠偏、人物精神。

更适合：通用。

资料依据：【资料直接体现】金句表《人生不是轨道，是旷野》《普通人想成功：要么不要脸要么遇到贵人》。

背后原理：对立结构天然有节奏感，且帮助用户更快理解你站在哪一边。

常见句式：不是___，是___；要么___，要么___；看起来___，其实___。

使用步骤：先找对立关系 → 再选择更口语、更易记的一端表达。

注意事项：对立不要过于极端到失真。

容易踩的坑：只有姿态，没有解释，像喊口号。

可直接调用的方法：把你最核心的观点改成一组二选一。

B3. 使命 / 身份宣言金句

技巧定义：把个人IP的内在驱动力写成一句能长期挂在嘴边的话。

核心作用：做人设锚点，让账号有长期辨识度。

适用场景：人设故事、个人介绍、阶段总结、结尾收口。

更适合：ToC 与个人IP；ToB 也可转成品牌精神。

资料依据：【资料直接体现】金句表《我生来是要看世界的》《追光的人》《快乐的人设盲盒》。

背后原理：IP不是靠信息量记住，而是靠一两句能概括你的人生主线的话记住。

常见句式：我生来是要___的；与其___，不如___；我想成为一个___的人。

使用步骤：从经历里找反复出现的执念 → 压成一句带方向感的话。

注意事项：必须从真实经历里长出来，不能凭空装腔。

容易踩的坑：太大太满，失去个人纹理。

可直接调用的方法：把“我为什么一直做这件事”写成一句口头禅。

B4. 信任型商业金句

技巧定义：把客户为什么愿意买单，写成一句有人情味又有商业逻辑的话。

核心作用：增强信任和转化，而不显得在卖。

适用场景：老板IP、本地服务、ToB 服务、案例复盘。

更适合：ToB 与转化内容。

资料依据：【资料直接体现】金句表《把人情味变成顾客愿意托付的理由》《装的不只是空调，装的是农村娃儿进城生活，争一口气》。

背后原理：信任不是空喊“专业”，而是让用户知道你为什么值得托付。

常见句式：客户买的不是___，买的是___；做的不是___，做的是___。

使用步骤：先写产品功能 → 再往上提炼客户真正买单的原因。

注意事项：不能完全脱离业务本身，否则像煽情。

容易踩的坑：只会讲情怀，机制和专业能力缺位。

可直接调用的方法：把“我卖什么”改成“客户真正托付的是什么”。

B5. 结论先行 + 可复述金句

技巧定义：先给结论，再在正文慢慢补证据。

核心作用：增强口播节奏，让视频更像“有人在说话”，而不是“有人在写文章”。

适用场景：口播、短视频、观点类内容。

更适合：通用。

资料依据：【资料直接体现】四条完整文案都先给结论，如“1分钟之后，你会有无限创作爆款的能力”“提示词它其实解决不了你文案平庸的问题”。

背后原理：短视频先吃结论，后吃证明。

常见句式：我先告诉你答案___；我越来越明确一个判断___；这件事真正关键的只有一个___。

使用步骤：先把结论说出来 → 再解释为什么 → 再举例。

注意事项：结论不能太抽象，要能立刻接住后文。

容易踩的坑：结论像套话；后文没有完成证明。

可直接调用的方法：每段开头先写“这段我要下的判断是什么”。

C1. 低起点 → 受挫 → 小胜利 → 坠落 → 翻盘结构

技巧定义：这是人物型 IP 最常见也最稳定的情绪结构。

核心作用：让人设有张力，不平、不悬浮。

适用场景：个人成长故事、老板创业故事、操盘手故事。

更适合：ToB/ToC 都适合，尤其是人设故事。

资料依据：【资料直接体现】模板表多条人设故事模板都采用这一骨架，如《照片引子→…→翻盘》《小镇自卑→…→自救口头禅》《老板IP故事模板》。

背后原理：用户会对“顺利的人”无感，但会对“跌过、扛过、又站起来的人”产生信任。

常见句式或模块：出身定位 → 否定/羞辱 → 第一次小胜利 → 大坠落 → 自救动作 → 新结果。

使用步骤：先列出人生节点 → 再找两次转折和一次大低谷 → 最后找一句收口口头禅。

注意事项：故事里必须有具体代价，不然只是流水账。

容易踩的坑：一直讲苦，缺少结果证据；只有经历，没有认知沉淀。

可直接调用的方法：先画时间线，再标记“最低点 / 转折点 / 现在的结果”。

C2. 观点 → 错误示范 → 类比解释 → 正确路径结构

技巧定义：把抽象方法讲透的高可复制结构。

核心作用：适合方法论类IP，既能打脸旧认知，又能教会新做法。

适用场景：AI、内容方法、商业认知、行业纠偏。

更适合：ToB、方法型个人IP。

资料依据：【资料直接体现】文案表《如何用好AI 当个文案高手》《用AI写出顶级文案的关键根本不是提示词》都走这条结构；模板表也有对应骨架。

背后原理：用户并不害怕新方法，害怕的是“不知道为什么旧方法错”。

常见句式：很多人会___，但这一步错在___；打个比方，这就像___；正确做法是___。

使用步骤：先下判断 → 再举大众常见误区 → 再用类比解释差别 → 最后给方法。

注意事项：类比必须贴近用户能理解的事物。

容易踩的坑：只有判断没有错误示范；只有类比没有真正步骤。

可直接调用的方法：讲任何方法时，先补一个“大家一般怎么做，为什么错”。

C3. 三步 / 路线图模板

技巧定义：用有限步骤把复杂事情变成“我今天就能做”的事。

核心作用：降低门槛，适合转收藏和转化。

适用场景：工具教学、项目拆解、执行型教程。

更适合：通用，偏 ToC 和工具转化。

资料依据：【资料直接体现】模板表《三步借样本+AI改写型原创文案模板》《黄金时代机遇再现｜三步方案》；文案表《用豆包一分钟写出短视频爆款原创文案》。

背后原理：用户不是不想学，是怕太复杂。步骤一旦少于 3-4 步，行动成本会明显下降。

常见句式：第一步___，第二步___，第三步___；你只要做三件事。

使用步骤：把完整流程压成 3 个最关键动作 → 每一步说人话 → 最后给结果感。

注意事项：步骤少，不代表每一步含糊；每步都要可执行。

容易踩的坑：步骤只是目录，不足以操作；步骤之间断裂。

可直接调用的方法：先写出完整流程，再删到只剩必须动作。

C4. 榜单 / 分点 + 升维收口结构

技巧定义：中段用分点满足信息获取，尾段再升到更大命题。

核心作用：让内容既有干货密度，也有格局感。

适用场景：财富观、行业判断、趋势内容、资源推荐。

更适合：ToB、认知型、信任型内容。

资料依据：【资料直接体现】模板表《99%押错财富方向｜TOP5榜单+文明底座升维》；多条口播模板都有“先分点后升维”的收口方式。

背后原理：分点让人觉得值，升维让人觉得你不只是搬运知识。

常见句式：我先说前 3 个___；说到底，真正关键的是___。

使用步骤：先分点 → 每点给一句解释 → 最后把分点拉回统一底层逻辑。

注意事项：分点不是堆砌，最后一定要回到总判断。

容易踩的坑：只有列表，没有总论；像读提纲。

可直接调用的方法：写完分点后，补一句“这几件事背后其实都指向___”。

D1. 模块化推进，让正文不平

技巧定义：把一篇口播拆成几个功能位，而不是一口气讲完。

核心作用：减少平铺直叙，让观众有持续的“下一段会讲什么”的期待。

适用场景：所有口播与长一点的视频文案。

更适合：通用。

资料依据：【资料直接体现】模板表大量使用“开场钩子 → 人物名片/问题定义 → 证据/案例 → 方法/方案 → 收口互动”的功能位思维。

背后原理：视频不是作文，观众需要节拍点。

常见做法：每 2-4 句切一个功能位；每段只承担一个任务。

使用步骤：先定功能位，再往里填句子。

注意事项：段落之间要有递进，不要每段都在重复一个意思。

容易踩的坑：段落有了，但每段信息重复。

可直接调用的方法：写稿前先列 5 个功能位标签。

D2. 用类比把抽象事讲懂

技巧定义：把复杂的业务、方法、机制转成日常经验的类比。

核心作用：快速降低理解门槛，同时增强“讲明白了”的感觉。

适用场景：AI、商业方法、行业逻辑、流程机制。

更适合：通用。

资料依据：【资料直接体现】文案表中把 AI 写文案比喻为做菜，并区分食材、调味、试菜。

背后原理：类比能把陌生问题嫁接到熟悉经验上。

常见句式：这件事就像___；你可以把___理解成___。

使用步骤：找一个用户人人懂的场景 → 对应映射核心关系 → 再回到原问题。

注意事项：类比要解释关系，不是只做装饰。

容易踩的坑：类比过长，喧宾夺主。

可直接调用的方法：写方法论时，先问自己“这像生活里的什么”。

D3. 案例 / 细节 / 行业黑话增强真实感

技巧定义：在正文里插入具体流程、术语、场景细节，让内容像干过的人说的。

核心作用：增强专业感和可信度。

适用场景：求职经验、行业科普、老板IP、方法论。

更适合：ToB、专业型、经验型 IP。

资料依据：【资料直接体现】应届生经验文案里出现 PR、brief、蒲公英、星图、置换、返点、KPI 等大量行业细节。

背后原理：抽象观点不容易让人信，细节会让人信你真的做过。

常见句式：我们一般把___简称为___；具体流程是___；这里有一个行业黑话叫___。

使用步骤：每讲一个观点，就补一个具体场景/术语/动作。

注意事项：术语出现后最好顺手解释一下。

容易踩的坑：全篇只有概念，没有细节；或者术语过量导致难懂。

可直接调用的方法：把你做过的流程拆成 3 个真实动作写进去。

D4. 先给获得感，再给方法

技巧定义：先让用户觉得“这条值”，再进入步骤和解释。

核心作用：提高停留与完播。

适用场景：教程、资料型内容、经验型视频。

更适合：通用，尤其适合 ToC。

资料依据：【资料直接体现】应届生文案先给“三年经验”，豆包文案先给“一分钟拥有无限创作能力”，再讲步骤。

背后原理：用户先要确认投入观看是划算的，才愿意听方法。

常见句式：我会把___都交给你；这条会直接帮你解决___。

使用步骤：开头给收益 → 中段给方法 → 结尾再提醒可带走什么。

注意事项：收益要具体到“你能拿走什么”。

容易踩的坑：先讲原理讲太久，错过最佳停留窗口。

可直接调用的方法：每条视频先写一句“用户今天看完能直接拿走什么”。

D5. 从方法走到世界观 / 价值观

技巧定义：不要停在步骤，把方法背后的更大判断说出来。

核心作用：让账号从“会教”升级成“有立场”。

适用场景：信任型口播、个人IP、老板IP、长期内容。

更适合：通用。

资料依据：【资料直接体现】AI 两条完整文案最后都升到“真正的竞争力是样本/内容资产系统”；老板模板也会升到“温度=真实做到极致”。

背后原理：方法教会用户一件事，价值观决定用户会不会继续跟你。

常见句式：说到底，这件事拼的不是___，而是___；所以真正重要的是___。

使用步骤：讲完方法后，补一句底层逻辑，再补一句长期判断。

注意事项：升维必须建立在前文之上。

容易踩的坑：突然拔高，像强行上价值。

可直接调用的方法：写完教程后，补一句“这件事长期看真正拼什么”。

E1. 不完美人设比完美人设更容易信

技巧定义：承认自己的笨拙、局限、出身、情绪，但同时给出成长与结果。

核心作用：去表演感，增强亲近度。

适用场景：自我介绍、人设故事、老板IP、幕后型IP。

更适合：通用。

资料依据：【资料直接体现】模板表《人设自我介绍模板》里有“不完美自嘲”；《反表演老板开头》直接强调老板不会表演。

背后原理：用户对“毫无裂缝的完美人设”天然戒备。

常见句式：我不是一个___的人；我也踩过坑；到现在我___还不擅长。

使用步骤：先承认真实短板 → 再给真实能力和结果。

注意事项：短板是为了显真，不是为了装弱。

容易踩的坑：只自嘲不立能力。

可直接调用的方法：写一个“我没那么完美，但我确实做成过___”句型。

E2. 稀有标签 + 技能树快速立人设

技巧定义：不是一句“我是做新媒体的”，而是用稀有标签、反差标签、技能组合让人记住你。

核心作用：快速建立识别度。

适用场景：个人IP自我介绍、账号置顶、第一条视频。

更适合：ToC 个人IP、幕后型/操盘手型IP。

资料依据：【资料直接体现】模板表《连夜盘点资产→稀有标签+技能树→快乐盲盒收口》。

背后原理：人设识别度来自组合，而不是单标签。

常见句式：我是一个___但又___的人；我能做___、___、___。

使用步骤：先写一个主标签 → 再补两个反差标签 → 再铺技能树。

注意事项：标签不要太多，3-5 个最够用。

容易踩的坑：堆满形容词，没有可验证技能。

可直接调用的方法：先写“我是谁”，再写“我还能做什么”。

E3. 让“懂行业、懂用户、懂表达”同时出现

技巧定义：专业不是靠一句“我很懂”，而是行业细节、用户视角、表达能力同时露出来。

核心作用：建立高质量信任。

适用场景：老板IP、顾问IP、操盘手IP、ToB 内容。

更适合：ToB。

资料依据：【资料直接体现】应届生经验文案能讲流程与黑话；AI文案文案能讲错误示范与样本拆解；老板模板会讲客户为什么托付。

背后原理：真正有说服力的人，既能讲内部逻辑，也知道外部人怎么理解。

常见句式：站在客户视角，他真正担心的是___；行业里一般会这样做___。

使用步骤：每条内容至少同时放入一个行业细节、一个用户心理、一个你自己的判断。

注意事项：不要只讲业务，不讲人。

容易踩的坑：专业性很强，但听起来离用户很远。

可直接调用的方法：每段都自查：行业、用户、表达三个维度有没有同时出现。

F1. 用机制证明而不是用口号证明

技巧定义：别只说“我们靠谱”，而要说为什么靠谱、机制如何避免踩坑。

核心作用：提升转化率，尤其适合本地服务/ToB服务。

适用场景：装修、服务型公司、顾问服务、交付说明。

更适合：ToB / 转化型内容。

资料依据：【资料直接体现】模板表《杭州装修老闻｜口碑背书+反套路对比+合同透明》明确写了“合同透明/不增项”。

背后原理：用户愿意买，不是因为你说自己好，而是因为你降低了他的风险。

常见句式：我们不玩___；合同里会写清___；这件事我们怎么避免___。

使用步骤：先说口碑/背书 → 再说反套路 → 再说机制。

注意事项：机制必须真实可执行。

容易踩的坑：还是停留在“我们很真诚”。

可直接调用的方法：把“靠谱”换成一个具体机制。

F2. 用顾客为什么愿意托付来做信任闭环

技巧定义：转化不是最后喊一句“来找我”，而是让用户明白别人为什么已经在找你。

核心作用：软转化，不生硬。

适用场景：老板IP、服务型视频、案例型内容。

更适合：ToB。

资料依据：【资料直接体现】老板IP模板里有“顾客选择理由（转化逻辑）”；金句表有“把人情味变成顾客愿意托付的理由”。

背后原理：用户更相信“他为什么被选择”，而不是“他希望你选择他”。

常见句式：客户最后选我们的原因，不是___，是___。

使用步骤：先展示经历/结果 → 再点出托付理由 → 最后给轻行动。

注意事项：理由要来自真实交付，不要虚构“客户都说”。

容易踩的坑：托付理由太抽象。

可直接调用的方法：把成交原因写成一句“客户为什么放心”。

F3. 评论区 / 私信承接型 CTA

技巧定义：不强硬成交，而是把评论区、私信、收藏当成下一步动作。

核心作用：更符合个人IP内容语境。

适用场景：教程、方法论、资料型、经验型内容。

更适合：通用。

资料依据：【资料直接体现】豆包文案以“把你的问题放在评论区”收尾；多条模板提到“评论互动口子”“评论私信承接”。

背后原理：用户在短视频里更愿意先做轻动作，再逐步信任你。

常见句式：如果你还不知道怎么做，把问题放在评论区；需要我把模板发你，评论区留___。

使用步骤：正文交付 → 轻度提问/索取动作 → 评论区二次承接。

注意事项：CTA 要和正文强关联。

容易踩的坑：结尾突然硬卖；CTA 与正文脱节。

可直接调用的方法：把 CTA 改成“我要不要把下一步也拆给你”。

G1. 口语化表达优先于书面化表达

技巧定义：短视频文案更像人在说话，而不是在写论文。

核心作用：去 AI 味、提高听感。

适用场景：所有口播。

更适合：通用。

资料依据：【资料直接体现】完整文案大量使用“那今天就用一条视频讲清楚”“打个比方”“大家不用担心”等口语衔接。

背后原理：真人感来自语气词、转折、补充、现场感，而不来自辞藻。

常见句式：我先跟你讲清楚一件事；打个比方；你可以理解成；说白了；说句实在的。

使用步骤：写完书面稿后，再口语化重写一遍。

注意事项：口语化不等于废话化。

容易踩的坑：一股书面报告味；大量并列名词。

可直接调用的方法：把每段都读一遍，改成嘴巴顺着能说出来的样子。

G2. 少做身份设定，多喂样本和素材

技巧定义：尤其在 AI 辅助创作里，少写“你是顶级专家”，多提供真实样本、素材、口头禅、行业信息。

核心作用：让内容更像你，而不是像一个标准答案。

适用场景：AI 写稿、爆款改写、个人IP训练。

更适合：通用。

资料依据：【资料直接体现】两条 AI 完整文案都在反复否定“过度写提示词”，强调“样本投喂”“素材系统”。

背后原理：AI 最会做的是模式识别，不是凭空理解你的业务和表达纹理。

常见句式：我给你几条样本，你先拆它们的共性；把我的行业、习惯、口头禅也一起吃进去。

使用步骤：给样本 → 给行业信息 → 给个人表达习惯 → 再让 AI 改写。

注意事项：样本质量比提示词花哨程度更重要。

容易踩的坑：只有身份设定，没有素材投喂。

可直接调用的方法：做一个“样本包 + 素材包 + 个人表达包”。

G3. 避免假、大、空、模板腔

技巧定义：文案里最容易显 AI 味的，不是用了 AI，而是用了空泛话术。

核心作用：保护真实感和信任感。

适用场景：所有内容。

更适合：通用。

资料依据：【归纳结论】四份资料里真正跑得通的内容，几乎都有具体代价、场景、机制、细节；而两条 AI 完整文案也明确批评“写出来很假、完全没有那个味儿”。

背后原理：空话没法被验证，用户听到会自动划走。

常见危险表达：赋能、深度思考、全方位升级、打造闭环、全面提升……如果没有具体对象和动作，就容易空。

使用步骤：每出现一个大词，就逼自己补一个具体动作或具体场景。

注意事项：不要为了显专业而堆概念。

容易踩的坑：整篇都像 PPT。

可直接调用的方法：给每一个抽象词都找一个真实例子。

5. ToB / ToC / 通用 三类适配总结

5.1 更适合 ToB 创始人 / 老板IP 的技巧

【归纳结论】更适合 ToB 的技巧，核心都围绕“责任、机制、案例、托付、行业判断”。例如反差显真开头、时间/数据对照、信任型商业金句、机制证明、顾客为什么托付。

为什么适合：ToB 决策更慢，用户更需要判断你是否真懂业务、是否能承担风险、是否值得长期合作。

适合内容：老板故事、公司案例、服务说明、行业判断、交付方法。

使用注意：不要全篇只有情绪，要把流程、机制、客户视角、行业细节说出来。

5.2 更适合 ToC 个人IP 的技巧

【归纳结论】更适合 ToC 的技巧，核心是“获得感、身份代入、成长叙事、情绪共鸣、可立即使用”。例如结果承诺+身份升级、挑战式提问、三步模板、使命宣言、优点清单与人设盲盒。

为什么适合：ToC 用户更容易被“我也能成为___”“我现在就能拿走___”触发。

适合内容：第一条视频、自我介绍、成长方法、教程、工具使用、经验包装。

使用注意：不要只讲情绪，要给可以立刻执行的动作。

5.3 两者都能用的通用技巧

【归纳结论】通用技巧包括反常识纠偏、观点压缩金句、模块化推进、类比解释、细节增强真实感、评论区承接型 CTA、口语化表达、样本投喂与素材系统。

为什么适合：这些技巧不依赖赛道，而依赖内容传播机制本身。

适合内容：所有短视频口播、图文拆解、个人IP方法论。

使用注意：同一个技巧在 ToB 里要更“重证据”，在 ToC 里要更“重代入”。

7. 反面清单：不建议使用的写法

【归纳结论】只有大词、没有场景：比如“深度赋能、系统升级、全面提升”，如果不接具体动作，容易空。

【归纳结论】只有人设标签、没有经历与证据：只说“我是专业的”“我是资深的”，用户不会信。

【归纳结论】只有反常识，没有后续证明：会像故意抬杠。

【归纳结论】只有情绪，没有出口：危机讲很多，方法讲很少，用户会疲劳。

【归纳结论】只有步骤，没有结果承诺：会让教程显得很普通。

【归纳结论】只有结果，没有过程代价：人物故事会像编的。

【归纳结论】一味追求漂亮句子，忽略口语节奏：更容易有 AI 味。

【归纳结论】套模板不套纹理：结构像了，但没有你的口头禅、行业细节、个人判断，最后会“形似神不似”。

8. 最终浓缩版总结

8.1 《IP文案创作核心原则清单》

先给钩子，再讲道理；先给获得感，再给方法。

金句不是修辞，而是替用户下判断。

真正的信任来自细节、机制、代价和结果，不来自自夸。

人物型内容一定要有转折、低谷、代价和翻盘。

方法型内容一定要有“错误示范—正确路径—可执行步骤”。

每条内容都要有一个能被记住的句子、一个能被相信的细节、一个能被带走的方法。

ToB 更重责任、机制、案例、托付；ToC 更重代入、成长、获得感、行动门槛低。

去 AI 味的关键，不是不用 AI，而是多用样本、素材、口语和真实经验。

模板是骨架，个人纹理才是灵魂。

收口不要硬卖，先做轻承接，再做深转化。

第二部分：个人IP打造技巧系统整理

1. 书中关于个人IP打造的核心观点

好内容首先要“方向正确”，即与IP定位一致，内容要聚焦核心用户痛点，不能跑偏、不能泛泛而谈。

好内容还要“表达真实”，即老板表达自然、内容可信、有人温度，拒绝演戏摆拍、装模作样。

转化型内容不能只讲观点，还要在评论引导、私信诱因、产品带入上形成行动提示。

内容审核时，是否贴合定位/IP方向、是否表现出老板真实气质、是否突出IP人设，都是重要判断项。

说明：以上四点中，前两点和第四点是原书直接写到的原则或评分项；“个人IP打造”的概念性提炼属于基于这些页面内容做的结构化归纳。

2. 书中提出的个人IP打造方法

用“内容产出原则”统一团队判断：先看方向、再看真实性、再看信息密度、最后看转化动作。

通过“搞访谈内容结构”组织嘉宾内容：开头用精彩片段和嘉宾/品牌成绩介绍吸引好奇，中间用成长故事与项目介绍建立人物与价值，结尾用金句总结升华情绪。

通过“用户证言内容结构”让真实用户从自身身份、使用场景、使用前问题、收获、信心变化、与他人不同点等角度发声，既展示价值，也帮助塑造IP。

通过“内容审核判断维度”持续检查内容是否围绕老板强项和方向展开，而不是做偏。

3. 书中涉及的人设 / 定位 / 差异化方法

定位：原书没有系统讲“定位方法论”，但明确要求内容必须与IP定位一致。

人设：原书没有系统讲“人设塑造步骤”，但在用户证言结构中明确提出“你感觉IP和其他人有什么不同”，其作用就是“突出IP人设”。

差异化：原书没有专章讲“差异化打造”，但在用户证言和访谈内容中，强调通过成长故事、项目经历、个人金句、与其他人的不同点来建立区别。

表达风格：原书明确要求“老板表达自然、内容可信、有人温度”，这实际上是在给IP表达方式设边界。

4. 书中涉及的流量、信任、转化相关打法

流量层：强调“5秒抓注意、15秒讲清一个重点”，以及开头要有一句话吸引或一句话说明观点。

信任层：强调老板表达自然可信、故事要基于真实经历、用户证言要讲清真实场景和真实收获。

转化层：强调评论引导、私信诱因、产品带入；视频结尾要形成“行动提示”。

审核层：如果内容没有引导转化/引发互动，评分会被直接拉低。

5. 书中案例里体现出的个人IP技巧

搞访谈内容结构中的案例表格，体现了通过“嘉宾事迹/品牌成绩介绍 + 成长故事 + 项目介绍 + 金句总结”来建立嘉宾可信度与身份感。

用户证言内容结构中，“你感觉IP和其他人有什么不同”这一项被明确对应为“突出IP人设”，说明真实用户对IP差异的感知可直接成为内容素材。

故事型内容设计中，第二种“人生建议 | 自己经历过才有什么感悟”，强调从亲身经历里提炼利他价值，也可视为IP建立信任的一种方式。

6. 可直接提炼成“规则”或“原则”的内容

内容必须与IP定位一致，不能跑偏。

内容必须聚焦核心用户痛点，不能泛泛而谈。

表达必须自然可信，避免摆拍、表演化表达。

信息传达必须高效：5秒抓注意，15秒讲清一个重点。

转化型内容必须有行动提示，不能只停留在表达层。

搞访谈和用户证言都要服务于“建立人物可信度 + 输出价值 + 突出差异化”。

第三部分：文案写作技巧系统整理

1. 书中关于文案写作的核心观点

文案不是随便说，而是要套进标准化内容结构框架。

不同内容类型对应不同结构：干货型、故事型、观点型，以及访谈、过程、Vlog、产品、新闻体、用户证言等。

“好内容”的判断不只看观点本身，还看抓人效率、信息密度、真实性和转化动作。

2. 书中提出的标题技巧

这本白皮书没有系统展开“标题写法大全”，但在干货型视频开头、故事型内容开头和新闻体内容中，能看出它偏好的标题/开头方向：

疑问式开头：例如“为什么你 XXX，还不 XXX”。

纠正式开头：例如“你是不是以为 XXX 就 XXX，我告诉你大错特错”。

新闻体标题：倾向直接概括核心事件，并能迅速说明变化结果。

需要说明的是：这些更多是“开头/题眼”方法，不是原书单独命名的标题学体系。

3. 书中提出的开头技巧

干货型：一句话讲感受或抛出信息差，再进入分析与方法。

解决问题型：一句话先说痛点，直接让目标用户产生代入。

故事型：开头极重要，要“一句话吸引人”，既可从“成事”切，也可从“面对冲突”切。

观点型：先用一句话说明观点，再展开支撑。

访谈型：用精彩片段剪辑和嘉宾介绍做前30秒开头，制造悬念和好奇。

4. 书中提出的正文结构 / 内容结构

4.1 干货型内容

信息差型结构：一句话吸引 + 分析 + 具体方法 + 总结/钩子。

解决问题型结构：一句话痛点 + 具体场景/方法 + 解决方案 + 钩子。

干货型视频开头设计：疑问型、纠正型。

干货型视频结尾：可引流、可给行动提示。

4.2 故事型内容

讲故事结构：背景 + 行动 + 取得 + 转机 + 胜利 + 总结感悟。

面对冲突型内容：也是开头很重要，且原书特别提醒“讲故事，情绪状态 > 内容”。

故事型内容设计四种：自我介绍、人生建议、人丑猎奇、逆袭/正能量。

4.3 观点型内容

讲观点结构：一句话说明观点 + 观点支撑点 + 金句总结。

观点型内容设计要求：有争议、与大众认知有反差。

4.4 其他类型内容

搞访谈内容结构：开头30秒 / 中间9—15分钟 / 结尾30秒—1分钟。

晒过程内容结构：告知视频主题 + 步骤1、2、3、4（可配合讲解） + 结束。

Vlog内容结构：主题 + 主线 + 画面素材。

产品展示内容结构：核心产品信息呈现 + 功能/优势细节说明 + 使用场景/价值强化。

新闻体内容结构：核心事件概括（5W1H） + 事件详情展开 + 背景/影响补充。

用户证言内容结构：用户身份及使用场景 + 具体使用体验描述 + 推荐态度表达。

5. 书中提出的爆点、冲突、情绪、观点、故事写法

爆点：干货型里体现为“我知道你不知道的细节”“跟目标用户痛点链接”的信息差或解决方案。

冲突：故事型中的“面对冲突”是明确结构类别，说明冲突本身可以成为内容入口。

情绪：原书提醒“讲故事，情绪状态 > 内容”，说明讲述状态会直接影响故事成立。

观点：观点型必须先明确立场，再给支撑点，最后金句总结。

故事：不仅有“成事型”，还有“自我介绍”“人生建议”“逆袭/正能量”等多种呈现方式。

6. 书中提出的收尾、转化、CTA技巧

干货型结尾可用“总结/钩子”，把下一步动作留给用户。

解决问题型结尾可做“引导链接/钩子”。

转化型标准中，原书明确要求有评论引导、私信诱因、产品带入。

视频结尾要形成“行动提示”，否则不算完整转化表达。

7. 书中出现的文案公式、模型、模板、框架

讲知识结构：一句话吸引 / 痛点 + 分析 + 具体方法 + 总结/钩子。

讲故事结构：背景 + 行动 + 取得 + 转机 + 胜利 + 总结感悟。

讲观点结构：一句话说明观点 + 观点支撑点 + 金句总结。

搞访谈内容结构：精彩片段 + 嘉宾介绍 + 成长故事/项目介绍 + 金句总结。

晒过程内容结构：主题 + 步骤 + 讲解 + 结束。

Vlog结构：主题 + 主线 + 画面素材。

产品展示结构：产品信息 + 功能/优势 + 场景/价值强化。

新闻体结构：5W1H + 详情展开 + 背景/影响补充。

用户证言结构：身份场景 + 体验描述 + 推荐表达。

8. 案例里体现出的文案技巧

干货型案例用服装店里的“工艺和质量只是客户在乎的一小部分，真正想要的是情绪价值”来做信息差开头，先抓认知反差，再进入分析。

解决问题型案例先说“很多专业做销售的人留不住客”，再用“风格诊断表”作为解决方案，体现“问题—场景—方案—引导”的链路。

故事型案例说明了故事需要背景、行动、取得、转机、胜利和感悟，而不只是流水账。

晒过程案例把茶叶制作过程拆成若干步骤，说明“过程内容”最重要的是主题清楚、步骤清楚、讲解清楚。

新闻体案例把“孩子从不想读书到想回学校”的变化与家长痛点、真实素材、真实配音、励志音乐配套，体现新闻体内容的组合方式。

9. 可直接提炼成“写作规则”的内容

先判断内容类型，再决定结构，不要所有内容都用一种写法。

开头必须抓人：干货要有信息差/痛点，故事要有一句话吸引，观点要先亮立场。

正文必须围绕一个核心重点推进，不能散。

故事类内容不能只讲事实，还要讲情绪状态和转折。

观点类内容不能只给结论，还要有支撑点。

转化型内容必须留出行动提示。

发布前要按定位、信息量、真实性、互动/转化、节奏五项维度做审核。

第二部分：个人IP打造技巧系统整理

1. 书中关于个人IP打造的核心观点

个人IP要先有内容分层：书中把IP内容分成破圈型、信任型、转化型三类，而不是把所有内容混成一类。

个人IP要围绕“人”来搭内容入口：作者强调“以心印心，聚焦IP价值”，并把打开IP心扉、追求真实表达放在重要位置。

表达力、场景和包装本身就是IP的一部分：这套体系把表现力、场景布置、拍摄角度、精剪包装和账号包装都纳入个人IP建设。

个人IP必须兼顾传播与转化：既要有破圈内容，也要有信任内容和转化内容。

2. 书中提出的个人IP打造方法

先做选题库，再做表达：先从市场火过的内容中建立爆款库，再做复制/改造，而不是完全从自我表达出发。

用“专家人设 + 社会议题”制造进入点：把与你自己有关、情绪共鸣、对立观点或高认知密度的题材加进来，让用户一眼觉得“这条和我有关”。

先把IP内容按破圈/信任/转化三层做清：破圈负责流量，信任负责相信你，转化负责成交或行动。

访谈和内容采集要从“用户关心”切入：满足好奇心、与我有关、借出圈话题强化人设。

主动设计互动与调整：根据IP状态和表达情况，用不同回复风格和补充策略去迭代内容。

前期要研究行业爆款并抽取规律：下载目标行业头部视频，分析核心话题、提问方式、受众反馈和可复用规律。

3. 人设 / 定位 / 差异化方法

差异化首先体现在“内容角色”上：不是泛泛地发内容，而是被识别成某一类人。

差异化体现在表达方式上：真诚、自然、多拍、对镜练习、假设对面真的有人、放大情绪。

差异化还体现在场景与画面信息点上：背景、颜色、文字、道具、前景、深度都能为IP服务。

差异化进一步落实到包装：知识干货类、观点评论类、故事叙事类、技能实操类IP，需要不同的BGM、字体和特效。

4. 流量、信任、转化相关打法

流量主要靠破圈型内容完成，关键词是“新、奇、爽”；信任主要靠故事和真实经历完成，重点是让用户觉得这个人真实、有得失、有反思；转化则依靠结构，而不是靠硬卖，书中给出了 AIDA、PAS、FAB 三种模型，并进一步压缩为“用户痛点 + 解决方案 + 钩子”。

5. 案例里体现出的个人IP技巧

教育类IP示例突出“结果导向 + 人设可信 + 标题明确”。

美容/行业类IP示例突出“行业身份 + 话题冲突 + 人群筛选”。

财经/观点类IP示例突出“见识型切口 + 强好奇标题”。

创始人/内容顾问类IP示例突出“专家型人设 + 知识型表达”。

6. 可直接提炼成规则或原则的内容

先建立爆款库，再做原创表达。

个人IP内容至少分破圈、信任、转化三类。

破圈内容的核心是“新、奇、爽”。

信任内容的核心是“讲好一个故事”。

表达力要追求真诚、自然，而不是假装专业。

信息点是场景的灵魂，背景也要为IP服务。

拍摄和精剪不是附属工序，而是IP呈现的一部分。

账号包装要从昵称、简介、头像到后台数据一起看。

本资料没有系统展开品牌心智理论、私域成交全模型、多平台矩阵增长打法大全，也没有系统讲“个人IP的哲学定义”。整体更偏短视频个人IP实操，而不是完整品牌理论书。

第三部分：文案写作技巧系统整理

1. 书中关于文案写作的核心观点

文案先决定这条内容属于哪一类：破圈型、信任型还是转化型。

文案要先解决“用户为什么会点开”：满足好奇心、与我有关、借出圈话题。

文案真正重要的是结构：作者更重视结构模型，而不是金句堆砌。

2. 书中提出的标题技巧

书中明确提出“标题越情绪化越容易火”。

从可见案例看，作者偏好的标题类型包括：结果型、对比型、冲突型、问题型、人群型。

标题不是单独存在的，它和封面、背书、字幕一起构成信息强化系统。

3. 书中提出的开头技巧

问题切入：如“为什么……”“是不是……”“该怎么……”。

满足好奇心与“与我有关”：让用户立刻判断这条内容和自己有关。

用出圈话题做入口：借热点、议题或争议话题切入。

4. 正文结构 / 内容结构

破圈型内容强调“新、奇、爽”；信任型内容强调讲好一个故事；转化型内容强调使用结构模型推进用户行动。

书中明确出现的结构模型• AIDA：注意 → 兴趣 → 欲望 → 行动• PAS：问题 → 激化 → 解决• FAB：特征 → 优势 → 利益• 简化结构：用户痛点 + 解决方案 + 钩子



书中还列出 5 种常用视频结构：1）观点 + 故事 + 总结；2）提问 + 分析论据 + 总结；3）奇遇 + 故事 + 反转（升华）；4）前面当钩子 + 后面引方法 + 引导互动；5）价值 + 如何获得 + 案例 + 钩子。

5. 爆点、冲突、情绪、观点、故事写法

爆点：新的观点、新的知识、爽感内容。

冲突：对比、反差、社会议题、争议性话题。

情绪：既可以体现在标题，也可以体现在故事推进上。

观点：书中常以“观点 + 故事 + 总结”的方式组织观点内容。

故事：主要服务信任建立，让用户相信这个人真实、有经历。

6. 收尾、转化、CTA技巧

书中没有单独成章写 CTA 话术大全，但从结构页可确认：收尾通常承担引方法、引互动、留钩子的作用；转化型内容则依靠 AIDA / PAS / FAB 之类的结构，把用户推到下一步行动。

第二部分：个人IP打造技巧系统整理

1. 书中关于个人IP打造的核心观点

• 账号想被记住，不能只有内容，还要有清晰定位与鲜明人设。书中明确指出：如果定位、选题都有了，却没有一个好人设，用户未必会和你形成情感联系。

• 个人IP要同时解决“被看见”和“被记住”两个问题。前者依靠内容与平台机制，后者依靠差异化、梗点、统一主页和持续重复。

• 个人IP不是随意记录生活。书中反复强调“专一”“垂直”“聚焦”，认为用户因为某类内容关注你，就希望你持续产出相似内容。

• 个人IP不能只自我表达，必须找到“自我表达与用户需求的交集”。书中把“过分自嗨”列为第一个大误区，核心就是内容不能只让创作者自己满意。

• 个人IP最终要形成可转化路径。书中并没有把账号只看成“涨粉工具”，而是直接把广告、电商、直播、私域、课程、咨询、线下引流、出版都纳入账号经营逻辑。

2. 书中提出的个人IP打造方法

（1）先避开四大误区

• 过分自嗨：不要只拍自己觉得高级或有趣的东西，而要拍用户能理解、能代入、愿意讨论和转发的东西。

• 不够专一：不要今天发旅游、明天发美食、后天又拍舞蹈模仿。书中认为账号只发一类内容非常重要，否则垂直度会下降。

• 没有特点：没有独特点就不会被关注。特点可以来自形象、动作、表达方式、出镜限制、内容切口等。

• 没有梗点：梗是记忆点。口头禅、笑声、固定动作、固定句式都可以成为让用户第一时间想到你的符号。

（2）用五种定位方法做出差异化

• 年龄反差法：让人物行为、内容或气质与年龄形成反差，制造新鲜感与稀缺感。

• 性别反差法：通过男扮女装、女扮男装，或声音反差、伪音反差等方式打破常规。

• 场景切换法：把大家熟悉的事情放到陌生场景中做，产生耳目一新的观看体验。

• 典故定位法：借用大众熟悉的神话、童话、典故人物来做人设定位。

• 影视定位法：把已经被市场验证过、用户已经喜欢的影视角色类型挪到抖音账号中。

（3）用人格化IP把定位变成“可追随的人”

• 主要人物：中心人物，是矛盾冲突主体，也是内容主线承载者。

• 次要人物：承担对比、陪衬、铺垫作用，帮助突出主角。

• 群像式人物：根据具体情节设定，虽然未必是固定主角，但能对氛围与叙事起到潜移默化的作用。

（4）用主页设置强化IP识别

• 昵称：要有辨识度、与定位相关、最好还能带一点趣味或标签。

• 头像：可以是真人照、形象照、主题海报或设计图，但要与昵称、简介形成合力。

• 简介：要突出你最强的特点，甚至可用适度自嘲把缺点转成标签。

• 视频封面：要统一边框、字体与风格，让用户一点进主页就感到内容调性稳定。

（5）先发散再聚焦

书中并不主张一开始就把所有内容方向锁死。它提出：前期发布的内容要先发散，科学试错；等看到哪些内容反馈更好，再聚焦。这个方法特别适合刚起号阶段。

3. 书中涉及的人设/定位/差异化方法

• 年龄反差的本质：用“认知不匹配”制造注意力。比如高龄奶奶做女团、奶奶穿旗袍环球走秀、老人打卡年轻人美食等，都会让用户先产生新鲜感，再进入内容。

• 性别反差的本质：用“与日常经验不一致”的形象和声音调动注意力。书中明确说，最有效的吸引注意方式之一，就是打破常规。

• 场景切换的本质：不是换背景这么简单，而是让用户看到熟悉内容被放进陌生场景后，重新产生兴趣。

• 典故/影视定位的本质：借大众已有认知来降低理解门槛。用户一看到月老、孟婆、霸道总裁、好婆婆，就能迅速理解账号要演什么。

• 特点的形成不只靠内容，还靠视觉与行为符号。比如不露脸、戴独角兽头套、口罩开口投食、卡点换装等，都属于“可被记住”的差异化元素。

• 梗点必须重复出现。书中强调，梗不是偶尔来一句，而是要反复强化，直到用户一看到某个动作、某句话、某个声音就想起你。

4. 书中涉及的流量、信任、转化相关打法

流量层面

• 用垂直度争取平台扶持：账号内容越稳定，平台越容易识别账号标签。

• 用差异化争取用户停留：没有特点就很难被关注。

• 用梗点争取被记住：记忆点越强，回访与传播越容易发生。

• 用互动数据争取进入更高流量池：完播率、点赞、评论、转发是书中反复提到的核心指标。

信任层面

• 人设要清晰、鲜明，用户才会对“这个人”建立偏好。

• 主页要统一，用户才会觉得账号专业、稳定、可信。

• 持续在一个领域深耕，用户才会形成“你就是做这个的”认知。

• 课程/咨询型IP需要在内容中持续输出专业知识，并在主页上放出联系入口。

转化层面

• 广告变现更适合已有粉丝基数或影响力的账号。

• 电商变现适合用故事或直接展示商品特征来带货，书中认为即使没有很多粉丝，也可能靠播放量出单。

• 直播变现既能靠打赏，也能靠直播带货。

• 私域变现是把抖音流量引到公众号、个人微信、社群，再通过课程和咨询转化。

• 线上给线下引流特别适合门店、景区和旅行类账号。

• 出版变现属于成熟IP的后端收益方式。

5. 书中案例里体现出的个人IP技巧

• “祝晓晗”：通过明确的家庭情景剧定位和主次人物关系，形成稳定家庭宇宙；后续还把成功人设延伸到“老丈人说车”。

• “七舅脑爷”：围绕“最难哄的女友”这种高辨识度关系设定做持续内容。

• “尿尿是只猫”：宠物IP不靠“可爱”本身，而靠调皮捣蛋、机灵有趣的个性。

• “黑脸V”：通过几乎不露脸这一反常规方式制造巨大好奇心。

• “毒角show”：头套、昵称、头像形成同一记忆符号，强化辨识度。

• “一条小团团OvO”：口罩不露脸 + 声音软萌 + 特殊口罩设计，共同构成吸睛点。

• “三支花”“北海爷爷”“时尚奶奶团”“汪奶奶”：都在证明年龄反差法可以极大提升新鲜感。

• “办公室小野”“小小食界”“乡村胡子哥”：都在证明场景切换法会让内容获得明显的新意。

6. 可直接提炼成“规则”或“原则”的内容

• 规则1：不要只做自己喜欢的内容，要做“自我表达与用户需求的交集”。

• 规则2：账号想长久，内容必须专一，垂直度不能乱。

• 规则3：没有特点，就没有关注理由；没有梗点，就没有记忆点。

• 规则4：定位一定要让用户一眼知道你和别人哪里不一样。

• 规则5：人设不是随便出镜，而是要像角色一样有主次、有关系、有持续性。

• 规则6：主页不是装饰，而是辅助人设、强化定位、提升信任感的入口。

• 规则7：前期先发散试错，再依据反馈聚焦，不要一开始就闭门造车。

• 规则8：个人IP最终最好能接到清晰的变现方式，否则账号很难形成经营闭环。

7. 书中未提及但很多人容易误以为提到的内容

• 书中没有系统提出一套完整的“个人IP理论定义框架”。它更像是用抖音实战方法在教你把账号做成可识别、可增长、可变现的IP。

• 书中没有系统讲“私域SOP、成交话术、销售跟进流程”。私域变现有提到，但主要停留在引流到微信/社群，再做课程或咨询的层面。

• 书中没有系统讲“跨平台个人品牌矩阵打法”。重点平台是抖音，个别地方会提到公众号、微信、社群，但不是多平台方法论。

• 书中没有讲AI写作、AI剪辑、自动化内容生产。这些都不在本书主题内。

• 书中没有系统展开“高端品牌咨询式个人IP定位模型”之类现代术语框架。至少在当前可核验文本中未见此类内容。

第三部分：文案写作技巧系统整理

1. 书中关于文案写作的核心观点

• 短视频文案不是孤立句子，而是和平台停留、完播、评论、转发直接相关的内容结构。

• 写法的核心不是辞藻，而是“留人—制造讨论—推动互动”。

• 书中最明确的总公式是：黄金3秒开头 + 2～5个爆点 + 白金结尾。

• 标题不是附属品，而是视频起跑线的一部分。好标题可以让普通画面获得很高互动；差标题会让内容直接沉下去。

• 爆点不只是情节刺激点，更是评论点。书中反复强调“爆点即是评论点”。

2. 书中提出的标题技巧

（1）疑问法

用疑问词加问号，把答案留给视频内容本身。书中特别强调，疑问法如果再叠加“反常识”，标题吸引力会大幅提高。典型例子是“520想给男朋友送个礼物，大家有没有好的男朋友推荐？”。

（2）数字法

标题加入数字，能降低理解成本、提高注意力。书中提醒：数字要有看点，最好夸张一点，让用户觉得震惊、想点开。常见写法如“99%的人都没看出来”“会做的人不超过2%”。

（3）热词法

把热点新闻、流量热词、明星名字、品牌名字等直接融入标题。核心不是随便带热词，而是借热词自带的关注度提高视频进入率与热度。

（4）俗语法

利用大家熟悉的话语结构，延展出你想表达的观点。书中认为这种写法的关键在于押韵、朗朗上口、容易形成温度和记忆。

（5）对话法

把标题写成一次对话：前半句正常描述问题，后半句给出有力或有趣的回答。它的好处是态度明确、观点鲜明，有时甚至可以直接拿后半句去做整条视频的内容。

（6）电影台词法

把自带流量的电影金句拿来做标题，例如“我命由我不由天”“你保护世界，我保护你”。这种方法本质上是借成熟文化记忆点来提高标题吸引力。

（7）好奇法

把话说一半，让用户想知道后半段。书中还特别指出：知识类视频可把好奇法和数字法结合，例如“记住三个电话，不会再有人欺负你了”。

（8）对比法

把强烈反差的东西放在一起，用标题先给出冲突和情绪。书中给出的判断是：标题与内容形成对比，是提升作品质量的好方法。

3. 书中提出的开头技巧

剧情类开头

• 陈述式：常见祈使句，如“来/放下/别/看看/赶紧+……”；吸引力来自强语气和冲突感。

• 反问式：常见句型如“你+凭什么/为什么/要什么/干什么/在哪里……？”；优点是能迅速把观众拉进对立或争执场景。

知识分享类开头

• 陈述式：常用“最……的……”句型，直接亮出价值点或反差点。

• 发问式：一类是“怎么……不要……”的自问自答式；另一类是“你知道最……的是……吗？”“在……里，有哪些实用的……”等疑问式写法。

• 降低门槛表达：书中提醒，像“1秒操作”“几个小技巧”这类说法，会降低用户对干货复杂度的心理预期，让他们更愿意看完。

4. 书中提出的正文结构/内容结构

总公式

书中将爆款内容结构明确概括为“黄金3秒开头 + 2～5个爆点 + 白金结尾”。这不是只针对一种类型，而是作为通用爆款公式提出。

知识分享类段落结构

• 逻辑结构：常用1、2、3……分点展开，适合对主题做分层说明；书中建议每一点陈述控制在30～40字。

• 时间结构：按步骤顺序呈现“怎么做”，适合手把手教学；如果步骤很多，可以省略1、2、3的连接语，让每一步直接以动词短语开头，单步尽量控制在15字以内。

时长与密度建议

书中建议知识分享类短视频时长尽量控制在15～60秒之间，且开始学习拍摄时不要选太多点。它给出的理由很直接：时长越短，完播率会越高，也更容易做出爆款。

5. 书中提出的爆点、冲突、情绪、观点、故事写法

（1）选题底层：先用“十大元素”找情绪基础

• 3种情感：爱情、亲情、友情。

• 5种情绪：愤怒、怀旧、愧疚、暖心、爱国。

• 2个因素：地域、群体。

• 书中的核心判断是：绝大多数爆款内容都涉及其中一种或几种，而且这些元素可以相互组合、交叉，产生新爆款。

（2）知识分享类爆点四法

• 对比法：把学习前后的差异、好坏差异、不同对象学习后的效果差异放出来，让用户有明显感受。

• 段子法：把选题相关段子、梗点与知识点结合。这样用户即使不深度学习，也会因为梗而留言互动。

• 抱大腿法：借热门歌曲、明星、童话人物、影视角色等自带流量素材做爆点。

• 新奇法：用非常规方式处理常见物品，让用户因“原来还能这样拍/这样表现”而惊讶。

（3）剧情类结尾四法

• 互动法：在结尾留悬念或留讨论空间，让用户猜测走向，从而引爆评论区。

• 共鸣法：让用户在结尾处产生同样的感觉，从而想点赞、评论和转发。

• 反转法：在结尾给出和预期不同的结果，让用户震惊、暖心或感动。

• 强化法：在结尾重复一个动作或一句话，让它成为用户评论区会不断复述的记忆点。

（4）知识分享类结尾三法

• 互动式：求赞、提问、下期预告、提醒关注，都属于这一类。

• 共鸣式：特别适合心理、情感、成长类知识，通过最后一句升华主题，打动用户。

• 反转式：在讲述过程中建立预期，到结尾给出不同走向，形成惊喜感。

6. 书中提出的收尾、转化、CTA技巧

• 直接求赞：例如视频结束时来一句“给老师点个赞呗”，配合评论区继续感谢点赞。

• 提问式收尾：在结尾抛一个用户愿意回答的问题，把评论热度做起来。

• 下期预告式收尾：让用户因为想看后续而关注账号。

• 直接价值承诺式收尾：如果你能稳定提供某类干货，可以直接告诉用户“关注后你能得到什么”。

• 把评论区当作文案的一部分：不是视频结束就结束，而是通过自己的评论继续挖爆点、抬互动。

7. 书中出现的文案公式、模型、模板、框架

• 通用爆款公式：黄金3秒 + 2～5个爆点 + 白金结尾。

• 剧情类黄金3秒：陈述式 / 反问式。

• 剧情类白金结尾：互动法 / 共鸣法 / 反转法 / 强化法。

• 知识分享类黄金3秒：陈述式 / 发问式。

• 知识分享类段落结构：逻辑结构 / 时间结构。

• 知识分享类爆点：对比法 / 段子法 / 抱大腿法 / 新奇法。

• 知识分享类结尾：互动式 / 共鸣式 / 反转式。

• 标题八法：疑问法 / 数字法 / 热词法 / 俗语法 / 对话法 / 电影台词法 / 好奇法 / 对比法。

8. 案例里体现出的文案技巧

• “最容易读错的9个英文字母”：标题与开头都用了“最……的……”和数字法，同时通过错误示范与正确示范形成对比爆点。

• “老外怎么回答Thank you，不要再说You are welcome了！”：知识分享类自问自答开头 + 互动争议点。

• “Excel里，有哪些实用的1秒操作呢？”：用“1秒”降低学习心理门槛。

• “记住三个电话，不会再有人欺负你了”：好奇法 + 数字法结合。

• “心里住着小星星，生活才会亮晶晶”：俗语改写 + 押韵，标题本身就有情绪氛围。

• “520想给男朋友送个礼物，大家有没有好的男朋友推荐？”：疑问法叠加反常识，明显提升进入率。

• “小时候常欺负妹妹，长大后总想给她最好的……”：对比法直接把情绪冲突写进标题。

9. 可直接提炼成“写作规则”的内容

• 规则1：开头必须先解决“用户为什么要继续看”。

• 规则2：不要把知识讲成平铺直叙；要用“最、几个、1秒、你知道吗、不要再……”等方式降低进入门槛。

• 规则3：视频里必须设置能被评论区接住的爆点，因为爆点本质上也是评论点。

• 规则4：标题不要只概括，要尽量制造好奇、冲突、反差、数字冲击或熟悉感。

• 规则5：结尾不能草草结束，要么求互动、要么升华、要么反转。

• 规则6：知识类内容也需要故事性、情绪性和可讨论性，而不是只堆信息。

• 规则7：字数不是越多越好，短视频文案要不断降低理解成本和观看阻力。

第七部分：最终浓缩版知识库

A.《本书中的个人IP打造技巧知识库》

一、定位层

• 目标：让用户一眼知道你是谁、做什么、和别人哪里不一样。

• 方法：年龄反差法 / 性别反差法 / 场景切换法 / 典故定位法 / 影视定位法。

• 判断标准：能否在用户心中留下“与众不同”的明确印象。

二、人设层

• 先做人格化IP：主要人物、次要人物、群像人物。

• 人设目标：让用户不仅记住内容，还记住“这个人/这组关系”。

• 要求：人设要清晰、鲜明、可重复出现、可持续扩展。

三、包装层

• 昵称：有辨识度、和定位相关、最好有标签性。

• 头像：与定位、人设一致，能强化品牌感。

• 简介：写最强特点，也可以适度自嘲，把缺点转成标签。

• 封面：统一风格，增强主页第一眼信任感。

四、内容层

• 不要过分自嗨；要找用户需求与自我表达的交集。

• 内容必须专一，维持账号垂直度。

• 持续放大自己的特点，形成差异化。

• 持续重复梗点，形成记忆点。

五、增长层

• 互动核心指标：完播率、点赞、评论、转发。

• 评论区也属于内容的一部分，要主动经营。

• 前期先发散试错，再根据反馈聚焦。

六、变现层

• 可行路径：广告 / 电商 / 直播 / 私域 / 线下引流 / 出版。

• 课程与咨询型IP，要在内容中先建立专业性，再在主页中放联系入口。

• 咨询型IP尤其依赖专业能力与交流体验。

B.《本书中的文案写作技巧知识库》

一、总公式

• 黄金3秒开头：先解决“观众为什么继续看”。

• 2～5个爆点：让视频中途持续有看点、有讨论点。

• 白金结尾：在结尾引导互动、制造共鸣或给出反转。

二、标题库

• 疑问法：提问，引导点开；叠加反常识更强。

• 数字法：用数字降低理解成本，放大震惊感。

• 热词法：用热点、明星、品牌、流量词借势。

• 俗语法：借熟悉句式与押韵提升好感。

• 对话法：前半句描述问题，后半句给态度/回答。

• 电影台词法：借经典台词自带记忆点。

• 好奇法：把话说一半，留悬念。

• 对比法：把冲突和反差直接写进标题。

三、开头库

• 剧情类：陈述式 / 反问式。

• 知识分享类：陈述式（“最……的……”）/ 发问式（“怎么……不要……”“你知道……吗”“在……里，有哪些实用的……”）。

• 降低门槛词：1秒、几个、最容易、最强、最难、不要再……

四、正文结构库

• 逻辑结构：按1、2、3分点展开，适合清单型知识。

• 时间结构：按步骤顺序展开，适合教程型知识。

• 控制原则：步骤清晰、每点不宜过长、视频整体尽量短。

五、爆点设计库

• 知识分享类：对比法 / 段子法 / 抱大腿法 / 新奇法。

• 剧情类：冲突开头 + 互动/共鸣/反转/强化式结尾。

• 底层原则：爆点要能被评论区接住。

六、结尾与CTA库

• 求赞：直接请求用户点赞。

• 提问：把问题留给评论区。

• 下期预告：用后续内容钩住关注。

• 价值承诺：直接告诉用户关注后能得到什么。

• 升华句：适合情感、成长、心理类知识视频。

七、评论区经营库

• 给他人评论：用符合自己人设的评论，在别人的评论区吸粉。

• 给自己评论：在自己视频下主动埋评论、抛问题、引导解释，带动更多真实用户下场互动。

八、边界说明

• 本知识库严格来自《爆款抖音短视频》当前可核验文本内容。

• 未加入书外的现代平台新规则、AI工具写法或跨平台运营经验。

• 若需要把本知识库进一步改造成“智能体可调用字段版/JSON版/Prompt版”，应另行处理，但不能把书外知识混入本版。

一、拍摄技巧与方式

拍摄技巧

- 手机与人距离：1米（约一臂半或一块地板的距离）

- 背景与人距离：为人与手机距离的2倍

- 固定机位：眉眼处于上三分线，桌子/手部动作处于下三分线

- 避免背景：避开大白墙

- 推荐背景：放置绿植、氛围灯，营造高级感

质感：苹果轻颜

 镜头运用：可采用前推、变焦等镜头运动方式增加画面动感

拍摄：眉毛对齐三分线，手机距离一个短臂

- 拍摄方式：

- 跟拍、走拍：采用45°侧拍，避免正面拍的压迫感

- 自拍：侧面拍，更具聊天的亲切感，右下角倾斜20度

坐着：一米，稍低，倾斜20度

- 谈判、面试：将手机放桌上拍，更显真实

- 拍揭秘、内幕：采用偷窥视角，代入感更强

- 口播、讲知识：手里拿东西，避免正面坐着的僵硬感

- 拍正面：运用透视，增加画面呼吸感且高级不死板

正面视角拍腻时，可选择偷窥视角

- 拍侧面：采用俯视，减少距离感，拉近心理关系

Ai口播流程

一、选题策划（从3小时→20分钟）

AI工具

- 选题：豆包/DeepSeek、新榜AI、蝉妈妈/灰豚AI热点

- 筛选：新榜流量评估、平台热词

全网热点监控：用 OpenClaw 这类AI代理工具，可以搭建一套“热点洞察服务”，让它每天自动监控知乎热榜、微博热搜、小红书趋势，早上8点准时推送到你的钉钉或微信，还会附带热度分析和爆文概率评估。这样你刷牙的时间就能决定今天拍什么。

· AI灵感碰撞：用 ChatGPT 或豆包进行“选题 brainstorming”。不要直接让它想选题，而是把你的账号定位和最近3篇爆款标题喂给它，让它分析共同点，并生成10个“衍生选题”。

步骤

1. 给AI发指令：“我是XX垂类博主，生成30个可拍选题，分教程/测评/避坑/干货，带黄金3秒钩子”

2. 导入新榜AI做流量打分+违规检测

3. 用数据工具看同赛道爆款，锁定3-5个高潜力选题

二、脚本文案（从1.5小时→10分钟）

AI工具

- 脚本：剪映AI写稿、豆包、GenApe

- 口播优化：讯飞听见、秘塔写作猫

步骤

1. 选题→AI生成30/60秒口播脚本（黄金3秒+分点+结尾引导）

2. 替换成你的人设语气，加停顿/重音

3. 导出纯文本，直接用于提词器+自动字幕

别让AI替你写（那会失去个人特色），而是让AI替你写“框架”和“润色”。

· 生成拍摄大纲：在确定选题后，让AI生成视频脚本框架。例如告诉它：“以‘独居女孩的夜晚’为主题，帮我写一个3分钟Vlog的分镜脚本，要有开头钩子、中间情绪转折和结尾共鸣。” Seedance 2.0 甚至支持直接参考文字分镜内容进行后续创作。

· 优化口播文案：写好的文案扔给AI进行“口语化转换”或“金句提炼”，让表达更接地气。

三、拍摄提词（从反复NG→一遍过）

AI工具

- 提词器：悬浮提词器AI（支持语速匹配）

步骤

1. 脚本导入提词器，AI适配语速

2. 固定机位+灯光，开自动对焦

 AI生成画面（空镜/封面） ：如果你做情感或知识类账号，需要大量空镜但没时间拍，用 可灵3.0 或 Seedance 2.0 生成。比如输入“黄昏城市车流，电影质感，光影流动”，直接生成视频素材。

· 生成图片封面：用 Canva 或 即梦AI。Canva现在的AI功能可以直接根据标题生成小红书或视频封面的几个版本供选择。

· 声音克隆：如果哪天嗓子哑了或不想录音，可以用AI克隆自己的声音。目前很多工具支持仅上传1分钟音频，就能生成跟你音色一模一样、且带有情绪顿挫的配音。

四、剪辑包装（从2小时→10分钟）

可优化点
