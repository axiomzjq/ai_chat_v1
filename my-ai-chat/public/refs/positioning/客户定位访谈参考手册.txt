================================================================================
创始人 IP 定位 · 客户访谈与定位分析参考手册
================================================================================

本文件由四份定位智能体文档提炼而成，用于在实际客户访谈与定位分析中
作为参考提示词使用。已过滤系统级技术细节（字段 ID、JSON 格式、状态
管理等），保留对定位判断与访谈提问有实质价值的核心内容。

================================================================================
第一部分：路线识别 — 先判 ToB 还是 ToC
================================================================================

【目的】
在正式生成定位报告前，先明确用户属于哪条主路线：
  · ToB 创始人 IP — 面向企业/组织，决策链长，依赖专业信任与案例
  · ToC 个人 IP   — 面向个人消费者，决策链短，依赖情绪共鸣与传播性

混合型用户也需在两者之间选定一条主路线，次路线只能作为补充。

────────────────────────────────────────
【8 维核心判定维度 — 用于访谈判断】
────────────────────────────────────────

维度1：目标客户类型
  → 产品/服务最终卖给谁？付钱的是企业还是个人？
  · 企业/工厂/政府/机构/组织 → ToB 强信号
  · 个人消费者 → ToC 强信号

维度2：主要收入来源
  → 现在最大单笔收入来自企业项目还是个人购买？
  · 企业项目服务 → ToB
  · 个人购买/课程/社群/个人服务 → ToC

维度3：成交链路长度
  → 从认识到成交需要多久？是否需要多轮沟通、方案、比价与审批？
  · 较长决策链、多人参与 → ToB
  · 个人快速决策、短链路 → ToC

维度4：成交依赖因素
  → 客户为什么买？依赖专业信任还是个人吸引/情绪打动？
  · 专业度、案例、交付能力、组织信任 → ToB
  · 个人魅力、内容吸引力、情绪/痛点共鸣 → ToC

维度5：内容主要目的
  → 内容主要服务什么结果？
  · 咨询、线索、成交前教育 → ToB
  · 吸粉、互动、种草、个人转化 → ToC

维度6：典型受众画像
  → 受众描述的是岗位角色还是生活身份？
  · "老板/采购/品牌总监/行政负责人" → ToB
  · "宝妈/大学生/职场新人/女生" → ToC

维度7：内容表达风格
  → 更适合讲行业认知与案例，还是故事、情绪与生活方式？
  · 行业观点、决策经验、案例拆解 → ToB
  · 故事表达、共鸣、个人成长、痛点转变 → ToC

维度8：平台优先级倾向（辅助维度，不单独决定路线）
  · 视频号 + 公众号 → 偏 ToB
  · 抖音 + 小红书 → 偏 ToC

────────────────────────────────────────
【路线判定后的关键追问】
────────────────────────────────────────

1. 这个人的内容表达主场更适合站在哪一边？
2. 主路线用于吸引谁？次路线用于证明什么？
3. 哪些内容可以借次路线增加厚度，哪些内容一旦混讲就会导致受众摇摆？
4. 首发内容更适合从主路线直接切入，还是先从次路线可传播的壳子切入再回归主路线？

────────────────────────────────────────
【首批受众收窄 — 路线判定后必须输出】
────────────────────────────────────────

· 首批最核心成交对象（必须具体到角色/行业/状态，禁止写成"老板/创业者"）
· 次级可覆盖对象
· 当前不优先服务对象

────────────────────────────────────────
【技巧库挂载规则】
────────────────────────────────────────

路线判定通过后，自动决定后续调用哪套创作技巧库：
  · ToB → 挂载 ToB 创作技巧库（侧重专业信任、客户教育、案例逻辑）
  · ToC → 挂载 ToC 创作技巧库（侧重人设记忆点、共鸣传播、转化路径）

次路线只能作为补充素材来源，不得改变主技巧库挂载方向。

第二部分：Demo 生成与确认 — 3 版差异化定位 Demo
================================================================================

【生成前必须读取的输入】
  结构化访谈报告、路线判定结果、技巧库来源与状态、表达主场判定

【3 版 Demo 的核心要求】
  · 必须一次性生成 3 版，且 3 版之间具备明确差异
  · 3 版都服从同一条主路线，但在人物主线、受众切入、表达方式、
    栏目重心、平台侧重上做差异化展开
  · 不能只是同一句话换几个词

【每版 Demo 必须包含的内容】
  · 一句话定位
  · 一句话价值主张
  · 最强身份标签
  · 最大反差点
  · 首批最核心成交对象
  · 次级可覆盖对象
  · 当前不优先服务对象
  · 核心内容主线
  · 首选平台
  · 首发形式建议
  · 最不建议走的方向
  · 为什么这版适合当前用户
  · 与另外两版的关键区别
  · 锁定后将带来什么内容与转化路径

【用户操作分支】
  · 直接选一版 → 进入正式报告
  · 定向修改 → 针对标签/受众/切口/平台做修改
  · 融合两版 → 按主路线不变、主场清晰、受众不打架、平台不冲突原则融合
  · 重生 3 版 → 若都觉得不对，保留路线结果重生；若本质是路线模糊，回流阶段1

第三部分：正式定位报告 — 10+1 模块
================================================================================

以下为正式定位报告 10+1 个模块的分析逻辑、双路线写法与关键提示词。

────────────────────────────────────────
模块01 · 报告基础信息
────────────────────────────────────────
必含：用户名称、身份与行业、核心业务一句话、分析日期/版本号
      主路线类型（ToB/ToC）、路线置信度、判定依据摘要（2-4条）

· ToB 写法重点：强调服务对象是企业/组织，说明企业客户类型与决策角色
· ToC 写法重点：强调服务对象是个人用户，说明典型人群和主要消费动机

────────────────────────────────────────
模块02 · 一句话结论与总判断
────────────────────────────────────────
必含：一句话 IP 定位结论、一句话战略建议、核心增长逻辑、
      最不建议做的方向、当前最优切入点

【ToB 写法】身份 + 行业 + 服务对象 + 信任价值
            战略建议优先回答：先做哪个平台、先打哪类内容、为什么

【ToC 写法】个人特色 + 用户痛点 + 核心价值 + 记忆点
            战略建议优先回答：先强化什么人设、先做什么内容类型、为什么

【追问/检验】
  · 这个人为什么值得被关注？
  · 最适合以什么身份做个人 IP？
  · 当前最优切入点为什么是这个？
  · 为什么不是普通 AI 工具号？

────────────────────────────────────────
模块03 · IP 定位分析
────────────────────────────────────────
必含：一句话 IP 定位、人设主标签（3-5个）、差异化优势（至少3条，
      每条必须有事实支撑）、信任来源、内容母题、
      适合的 IP 路线 / 不适合的 IP 路线

【ToB 写法】优先强化：专业感、决策经验、行业判断、结果导向
            差异化来源：行业经验、真实项目、方法论、交付能力

【ToC 写法】优先强化：个人特色、故事性、风格记忆点、用户代入感
            差异化来源：经历反差、表达风格、结果转变、观点锋芒

【追问/检验】
  · 最强身份标签是什么？
  · 最大反差点是什么？
  · 最适合被记住的个人主线是什么？
  · 别人为什么愿意持续关注这个人？

────────────────────────────────────────
模块04 · 内容方向建议
────────────────────────────────────────
必含：核心受众（1-3类，必须具体）、不想吸引的人、内容母题、
      一级栏目（3-5个，必须能长期成立）、高传播切口、高转化切口
      （必须分开写）、内容形式建议、前30天内容重点

【ToB 写法】
  内容母题：行业误区、客户教育、决策逻辑、案例拆解、方法说明
  高传播切口：反常识观点、项目故事、最赚钱/最后悔决定
  高转化切口：服务逻辑、案例过程、FAQ、客户教育内容
  栏目：案例拆解 / 行业认知 / 决策方法 / 客户教育

【ToC 写法】
  内容母题：个人故事、用户痛点、成长转变、体验结果
  高传播切口：情绪共鸣、冲突、反差、反常识表达
  高转化切口：解决方案、产品体验、方法步骤、结果证明
  栏目：痛点故事 / 转变过程 / 方法分享 / 结果展示

【追问/检验】
  · 受众能不能再收窄？必须细到角色/行业/状态
  · 传播切口和转化切口有没有混写？
  · 前30天建议能不能直接执行？

────────────────────────────────────────
模块05 · 平台与交付建议
────────────────────────────────────────
必含：适合的平台（排序）、平台角色分工、是否适合真人出镜、
      是否适合数字人、最佳交付方式、拍摄节奏建议、团队需求

【ToB 写法】优先：视频号、公众号；小红书/抖音视行业补充
            更强调：真人低频高质 + 数字人补充

【ToC 写法】优先：抖音、小红书；视频号/公众号按深度补充
            更强调：传播频率与内容活跃度，真人优先

【追问/检验】
  · 为什么适合真人/数字人/混合？比例怎么配？
  · 如果现在启动，最适合口播/访谈/混合/图文哪一种？
  · 不能只写"全平台都做"

────────────────────────────────────────
模块06 · 私域与转化建议
────────────────────────────────────────
必含：内容目标环节、私域承接建议、资料包建议、销售内容建议、
      转化钩子建议、客户教育内容建议
      信任资产如何承接成交（把资料包/样例/FAQ/案例与成交场景连起来）

【ToB 写法】企微/微信承接、资料包、案例/FAQ/客户教育内容
            销售内容：案例包、异议处理、FAQ、会后跟进

【ToC 写法】私信关键词、社群承接、清单/小课/体验页
            资料包更偏：清单、方法包、测评、入门指南

【追问/检验】
  · 承接建议不能只写"引导私信"，要有具体路径
  · 资料包与业务有没有直接关系？
  · 信任证明材料怎么服务成交？

────────────────────────────────────────
模块07 · 风险与限制建议
────────────────────────────────────────
必含：信息缺口、内容风险、表达风险、合规风险、执行风险、
      优先修复项（按优先级排序）
      这些风险将怎样具体影响后续 IP 打造

【ToB 重点风险】过官、过空、过度行业黑话、案例不足、内容与成交脱节
【ToC 重点风险】人设不鲜明、内容不抓人、表达没记忆点、转化链路弱

────────────────────────────────────────
模块08 · 账号定位建议
────────────────────────────────────────
必含：账号名称（至少3个）、账号简介（至少3版）、内容类型建议、
      账号风格建议、视觉建议
      名称/简介必须提供：首选方案、备选方案、暂不建议方案、适用时机

【ToB 写法】名称稳重清晰，简介突出身份+价值+服务对象
【ToC 写法】名称好记有特色，简介突出个人标签+用户价值

────────────────────────────────────────
模块09 · 内容阶段规划与选题建议
────────────────────────────────────────
必含：3-4个阶段（每阶段有明确目标）、每阶段 ≥20个选题/标题、
      爆款优先选题、高转化优先选题
      "小启动包"：第一条内容、第一周3条优先题、前8条建议顺序、
                    哪些题先别碰、当前最适合验证的2条内容母线

【ToB 阶段】认知 → 信任 → 咨询前教育 → 成交前信任
  选题方向：行业误区、决策经验、案例过程、客户教育

【ToC 阶段】认知 → 共鸣 → 信任 → 转化
  选题方向：痛点故事、情绪共鸣、变化过程、结果展示

【追问/检验】
  · 第一阶段最适合起号的内容切入口是什么？
  · 起号初期更适合讲判断、故事、方法还是项目过程？

────────────────────────────────────────
模块10 · 后续执行建议
────────────────────────────────────────
必含：最先做的3件事、未来30天建议、未来90天建议、
      最适合调用的后续模块、首轮验证指标
      现在是否适合马上开始做内容 / 适合什么频率 / 哪些题先别讲

【ToB 写法】先聚焦平台、先做高信任感内容、先建立咨询承接
【ToC 写法】先做高频验证、先强化人设、再放大转化路径

────────────────────────────────────────
模块11 · 生成说明与自检记录
────────────────────────────────────────
必含：路线识别结果摘要、补写次数、主要修订点、终版输出说明
      哪些内容已沉淀为可调用资产、哪些仍属低置信度判断

第四部分：增强模块 — 面向个人 IP 打造用途
================================================================================

以下 5 个增强模块在保留 10+1 主骨架的前提下追加，用于深化人物/IP 建模
与可调用资产沉淀。

────────────────────────────────────────
增强模块 A · 人物主线与 IP 核心识别
────────────────────────────────────────
必含：
  · 一句话人物定义
  · 一句话价值主张
  · 最强身份标签
  · 最大反差点
  · 最适合被记住的个人主线
  · 为什么别人应该关注他

写作提醒：核心不是"他做什么"，而是"他为什么值得被看见"。
          ToB 更强调"值得托付/值得咨询/值得长期信任"；
          ToC 更强调"记忆点/共鸣点/传播点"。

────────────────────────────────────────
增强模块 B · 关键故事资产卡（每张卡）
────────────────────────────────────────
必含字段：
  · 故事标题
  · 背景
  · 冲突点
  · 关键决策
  · 结果
  · 认知提炼（一句话）
  · 适合用于什么类型的内容（首发口播 / 信任内容 / 转化内容 /
                            长文 / 小红书笔记）

写作提醒：不得只写"我差点不做这个项目了"这类经历摘要；
          要写成后续选题与文案可直接调用的故事卡。

────────────────────────────────────────
增强模块 C · 表达风格结构化判断
────────────────────────────────────────
必含字段：
  · 句式偏好（短句 / 中长句 / 混合）
  · 开篇方式偏好（先讲判断 / 先讲故事 / 先抛问题）
  · 观点强度（强观点 / 温和解释 / 平衡型）
  · 修辞偏好（反问 / 对比 / 举例 / ...）
  · 潜在口头禅候选
  · 禁用表达
  · 哪些表达容易显得不像本人

写作提醒：目标是给后续文案系统提供"像本人"的语言规则，
          不是只给"真诚、理性、适合口播"这种粗标签。

────────────────────────────────────────
增强模块 D · 信任资产清单
────────────────────────────────────────
必含字段：
  · 已有的信任资产
  · 缺失的信任资产
  · 优先补齐的信任资产
  · 适用场景（主页 / 短视频 / 私域 / 销售页 / 成交前沟通）

覆盖素材类型：
  可量化成果、案例素材、系统输出样例、截图反馈、
  咨询数据、成交前证明材料、演讲/媒体/公开背书

────────────────────────────────────────
增强模块 E · 内容启动准备度判断
────────────────────────────────────────
必含：
  · 现在是否适合开始做内容
  · 适合什么形式（口播 / 访谈 / 混合 / 图文）
  · 适合什么频率
  · 适合什么第一批内容主线
  · 哪些题暂时不要讲

写作提醒：把报告从"能分析"升级为"能启动"。

第五部分：核心写作硬规则（生成智能体内化用）
================================================================================

以下规则适用于所有模块的生成质量判断。

1. 每个模块至少包含：1条方向判断、1条不建议事项、1条下一步动作

2. 核心受众不能写成"创业者/老板/普通用户"这类大词，
   必须细到角色、行业或状态

3. ToB 报告中，平台建议和选题建议不得整体娱乐化；
   ToC 报告中，内容不得全是企业顾问口吻

4. 账号名称和简介必须符合平台语感；
   标题和选题必须可直接使用

5. 前30天建议、阶段规划、选题库必须体现优先级，不可堆砌

6. 禁止模板化套话（如"持续输出价值""打造个人品牌""做好内容矩阵"）
   凡未落到用户具体业务的表述，视为低质量输出

7. 人物主线必须前置，不能让产品说明压住人物识别

8. 首批受众必须收窄到可执行层级（核心 / 次级 / 不优先）

9. 故事必须写成可调用的资产卡，不能只留经历摘要

10. 表达风格必须拆成语言规则，不能只写粗标签

11. 信任来源必须形成清单，不能只写缺口提醒

12. 必须压缩出"小启动包"（第一条内容 / 第一周3题 / 前8条顺序 /
    哪些题先别碰），不能只给大题库

================================================================================
第六部分：报告结构总览（快速索引）
================================================================================

完整报告骨架：

  00. 前置摘要与行动区          ← 用户最先看的内容
      └ 一句话人物定义 / 价值主张 / 关注理由 / 反差记忆点
      └ 当前最适合的账号定位 / 内容主线
      └ 第一条内容建议 / 第一周3条 / 最先做的3件事
      └ 平台建议 / 账号名简介首选 / 是否适合马上开始

  01. 报告基础信息

第八部分：客户访谈核心提问清单
================================================================================

以下问题可直接用于与客户的一对一访谈中，帮助收集定位分析所需的
关键信息。按维度分组，可根据客户回答选择性追问。

─── 业务与成交 ───
1. 你的产品/服务主要卖给谁？最终付钱的是企业还是个人？
2. 现在主要收入来自哪类客户？最大单笔来自企业还是个人？
3. 从客户第一次接触到成交，通常需要多久？中间经过几轮沟通？
4. 客户选择你而不是竞争对手，最主要的原因是什么？
5. 你最有代表性的成功案例是什么？（具体过程/结果/客户反馈）
6. 你做过哪些普通同行没有做过的事或决定？

─── 受众 ───
7. 你最想吸引的客户是什么样的人？（角色/行业/状态）
8. 你最不想吸引的是哪类人？明确说出来反而能帮你收窄定位
9. 你现有客户中，哪一类带来了最多收入和最高满意度？

─── 内容与表达 ───
10. 你觉得自己最擅长用哪种方式表达？口播/访谈/图文/长文？
11. 你的内容风格更偏向：讲行业判断和案例 / 讲故事和情绪共鸣？
12. 你平时刷什么平台？哪个平台上的内容最让你觉得"我也能讲"？
13. 你之前有没有发过内容？什么形式？反馈最好的一条是什么？
14. 你觉得别人最容易记住你的一个特点是什么？

─── 信任资产 ───
15. 你现在有哪些可以用来建立信任的材料？（案例/数据/截图/反馈）
16. 你缺什么信任资产？（案例样例/客户反馈/系统演示/数据证明）
17. 你最常被客户问到的前 3 个问题是什么？

─── 身份与故事 ───
18. 用一句话介绍你自己和你的业务
19. 你做过什么决定，回头看觉得特别正确？
20. 你经历过什么反差或转折，让你变成了现在的样子？
21. 你和其他做同类服务的人最大的区别是什么？
22. 如果用一个标签让别人记住你，你最希望是什么？

─── 执行条件 ───
23. 你现在有多少时间可以投入内容创作？
24. 有没有团队支持？能提供什么帮助？
25. 你最顾虑做 IP 的障碍是什么？（出镜/选题/持续性/...）
26. 你期望多久看到初步效果？

─── 启动判断 ───
27. 如果建议你现在就开始，你觉得第一条内容讲什么最自然？
28. 你最想先验证的假设是什么？（人设/受众/内容方向/平台）
