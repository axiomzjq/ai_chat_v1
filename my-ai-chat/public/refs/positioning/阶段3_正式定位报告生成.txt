12. 面向个人IP打造用途的增强生成要求

本文件在保留原有“路线识别 → 顺序生成 → 自检补写 → 装配终版”闭环的前提下，新增“面向个人IP打造用途”的生成要求。生成智能体产出的报告，不仅要能作为第二阶段定位分析输入底稿，还要能直接服务后续个人IP定位、内容主线设计、选题规划、文案生成与起号执行。

生成目标需从“项目与产品理解报告”为主，升级为“项目理解 + 人物/IP建模 + 可调用内容资产沉淀”的综合报告。项目、商业模式、系统链路、差异化仍要保留，但当“产品说明”和“人物/IP建模”发生篇幅冲突时，优先保证人物识别、人设主线、故事资产、表达风格、信任资产与启动建议写透。

生成时必须更强地回答以下问题：这个人为什么值得被关注；这个人最适合以什么身份做个人IP；他身上最有传播力、最有记忆点的故事与反差是什么；他最适合长期讲的内容主线是什么；他的表达风格怎样被结构化供后续文案系统调用；目前有哪些可直接用于建立信任的资产、哪些仍缺；现在是否已具备启动内容的条件，若启动，应如何启动。

禁止把报告写成“产品说明书 + 商业概括”的组合文本；必须让用户后续能直接从报告中提取定位句、人设标签、内容母题、故事素材、表达规则、信任资产和启动动作。

12.1 增强后的统一生成原则

人物优先：对于同一份输入，优先回答“这个人是谁、为什么值得被关注、适合以什么身份做IP”，而不是只解释“他做了什么业务”。

证据优先：新增判断尽量建立在结构化访谈结果、已生成模块与明确事实之上；不得无依据脑补人物经历、案例结果、表达风格或信任证明。

资产优先：凡是能够沉淀为后续内容系统输入的元素，都要优先转写为可调用资产，而不是停留在摘要层。例如：故事要转成故事资产卡，表达风格要转成语言规则，信任证明要转成信任资产清单。

执行优先：所有新增内容必须最终服务 Stage 2 定位、Stage 3 选题、Stage 4 文案与起号执行，不得只追求报告“看起来很完整”。

最小改动优先：在保持既有结构与原有有效内容的前提下做增强；如果某个模块已具备正确方向，则只补足人物/IP相关内容，不推翻已通过内容。

12.2 面向个人IP打造的模块级深度要求

除原有模板要求外，核心模块默认尽量补齐以下深度层：已确认信息、证据与支撑细节、深层分析与判断、对IP打造的具体价值、当前缺口与问题、待追问问题、下游调用建议。若某模块篇幅不适合完整展开，至少也要在 evidence_notes 与 generated_content 中体现事实、判断、缺口和下游用途。

模块2（一句话结论与总判断）除原要求外，必须显式回答“为什么这个人值得被关注”“最适合以什么身份做个人IP”“当前最优切入点是什么”。

模块3（IP定位分析）除原要求外，必须补强人物/IP主线，尽量写出最强身份标签、最大反差点、最适合被记住的个人主线，以及后续为什么应围绕这条主线展开。

模块4（内容方向建议）除原要求外，必须进一步收窄“首批最核心成交对象”“当前最适合切入的人群”“当前最不优先服务的人群”，并尽量形成可直接对外表达的收窄版本。

模块5（平台与交付建议）除平台与形式外，还要判断“如果现在启动内容，最适合用口播 / 访谈 / 混合 / 图文中的哪一种作为主形式”，以及为什么。

模块6（私域与转化建议）除转化路径外，必须同步沉淀“信任资产如何承接成交”，避免只写“引导私信”而不说明要拿什么建立信任。

模块7（风险与限制建议）必须说明这些风险将怎样具体影响后续IP打造，而不仅是列问题；并按优先修复顺序排列。

模块8（账号定位建议）账号名称、简介、内容风格建议，应尽量贴合人物主线而不仅贴合业务描述。

模块9（内容阶段规划与选题建议）在保留分阶段选题的前提下，应尽量指出“第一阶段最适合起号的内容切入口”“当前最适合的首发内容方向”“起号初期更适合讲判断、故事、方法还是项目过程”。

模块10（后续执行建议）除30天/90天动作外，应尽量明确“现在是否适合马上开始做内容”“若开始，适合什么频率”“哪些题暂时不要讲”。

12.3 新增的增强模块（供生成智能体在不破坏原骨架的前提下追加写入）

增强模块A｜人物主线与IP核心识别：至少包含一句话人物定义、一句话价值主张、最强身份标签、最大反差点、最适合被记住的个人主线、为什么别人应该关注他。该增强模块可作为模块3的子段，也可在模块2与模块3之间单独输出。

增强模块B｜关键故事资产卡：把已有的重要经历与决策故事整理成结构化故事资产卡，至少补出故事标题、背景、冲突点、关键决策、结果、认知提炼、适合用于什么类型的内容。该增强模块可作为模块3或模块9的引用资产池。

增强模块C｜表达风格结构化判断：把较抽象的风格判断改成可供文案系统使用的语言规则，尽量补充短句/长句偏好、先讲判断还是先讲故事、强观点还是温和解释、是否适合反问/对比/举例、潜在口头禅、哪些表达不自然。该增强模块建议挂载到模块3或模块8之前。

增强模块D｜信任资产清单：明确已有 / 缺失 / 优先补齐 / 适用场景。至少覆盖可量化成果、案例素材、系统输出样例、截图反馈、咨询数据、成交前证明材料。该增强模块建议挂载到模块6或模块7之后。

增强模块E｜内容启动准备度判断：明确现在是否适合开始做内容、适合什么形式、适合什么频率、适合什么第一批内容主线、哪些题暂时不要讲。该增强模块建议挂载到模块10之后。

12.4 重点避免的低质量生成问题

不要只是把原文句子写长一点，或把“分析与判断”换个说法重复一遍。

不要只停留在“项目理解”和“产品链路”层面，而忽略人物识别、人设主线、故事反差与表达规则。

不要让报告变长但没有更深。只要新增内容不能服务后续定位、选题、文案或起号执行，就不算有效增强。

不要只给结论，不给证据与支撑；不要只给建议，不说明为什么这个问题会影响后续IP打造。

不要将高价值故事写成经历摘要；不要将表达风格写成“真诚、理性、适合口播”这类过粗标签后就停止展开。

12.5 终版装配前的增强自查补充

终版是否已经从“项目与产品理解报告”升级为“IP分析与定位报告 + IP可用资产沉淀报告”。

终版是否能够明确回答：为什么值得关注、适合什么身份、最强故事与反差、长期内容主线、表达风格规则、信任资产现状、内容启动条件。

终版是否不仅可供第二阶段定位，也能直接服务第三阶段选题、第四阶段文案与起号执行。

若上述问题未被清晰回答，则应优先回到模块2、模块3、模块4、模块6、模块10或增强模块A-E补写，而不是只优化措辞。

13. 针对当前定位报告质量问题的专项增强执行要求

在保持现有“路线识别 → 顺序生成 → 自检补写 → 装配终版”闭环不变的前提下，生成智能体新增以下专项增强要求：不要让产品说明压住人物主线；不要让受众保持过宽；不要把故事只写成经历摘要；不要把表达风格只写成粗标签；不要把信任来源只写成缺口提醒；不要让主路线识别完成后仍然主场摇摆；不要只给大题库而不给小启动包。

生成终版时，必须优先让用户记住“这个人为什么值得被关注”，再让用户理解“他在做什么产品”。

模块4 以后，必须对首批受众做三层拆分：首批最核心成交对象、次级可覆盖对象、当前不优先服务对象。

凡是访谈中已经出现的重要经历、关键决策、重大犹豫、反差经历、交付痛点、测试突破，都应优先被转写为故事资产卡。

表达风格不得只停留在“真诚、理性、适合口播”这类标签，必须尽量产出可供文案系统调用的语言规则。

信任来源不得只写“有访谈经验、缺案例样例”，必须尽量拆成信任资产清单，并标注适用场景。

路线识别完成后，除主路线外，必须同时说明主路线表达主场、次路线补充方式与混讲风险。

阶段规划完成后，必须额外压缩出一份“小启动包”，让用户能立即开始。

13.1 统一新增输出层

速读层：核心模块尽量前置“模块一句话结论、最重要的 3 个点、这一段看完应该立刻做什么”。

证据层：核心模块尽量同时写出原话证据、行为证据或故事证据中的一类或多类。

资产层：凡是后续能被选题、文案、起号执行调用的元素，尽量转成资产化表达。

13.2 模块级新增执行要求

S1 一句话结论与总判断：除原字段外，尽量输出一句话人物定义、关注理由、反差记忆点、为什么不是普通 AI 工具号。

S2 IP定位分析：除原字段外，尽量输出最强身份标签、最大反差点、最适合被记住的个人主线、为什么值得持续关注。

S3 内容方向建议：必须进一步收窄首批最核心成交对象、次级可覆盖对象、当前不优先服务对象，并形成可直接对外说的人群表达。

S4 / S5：除平台与转化建议外，尽量说明哪类形式最适合当前启动、哪些信任资产将承接成交。

S7 账号定位建议：除名称与简介列表外，尽量给出首选方案、备选方案、暂不建议方案、适用时机。

13. 路线识别后的技巧库自动调用与约束源挂载规则

13.1 自动调用原则

当 route_type 判定完成后，生成智能体必须自动挂载与主路线一致的创作技巧库，作为后续生成报告的正式约束源，而不是可有可无的参考资料。主路线为 ToB 时，必须调用 ToB 创作技巧库；主路线为 ToC 时，必须调用 ToC 创作技巧库。技巧库一旦挂载，后续的人设判断、内容主线判断、栏目建议、平台建议、转化建议、账号定位建议、阶段规划、选题与执行建议，都必须显式服从对应技巧库中的分析逻辑、表达逻辑、内容逻辑、平台逻辑与转化逻辑。

13.2 建议新增字段

skill_library_source：当前挂载的技巧库来源，例如 ToB 创作技巧库 / ToC 创作技巧库。

skill_library_status：loaded / missing / fallback，用于标记技巧库是否成功挂载。

skill_library_route_match：pass / mismatch，用于标记技巧库路线是否与主路线一致。

skill_library_fallback_note：若技巧库缺失或路线不匹配，必须显式说明已按主路线默认规则降级生成。

13.3 技巧库调用优先级

当前用户请求 > 结构化访谈结果 > route_type 与 route 识别结果 > 对应路线技巧库 > 模块规则 > 报告输出模板 > 通用兜底写法。

技巧库不得压过真实用户事实；若技巧库逻辑与真实访谈事实冲突，必须优先保留真实事实，并在 evidence_notes 或 low_confidence_note 中说明冲突点。

若 route_type 已确认但技巧库未成功加载，不得假装已调用，必须在 generated_content 或终版说明中写清“当前缺少对应技巧库支持，已按主路线默认规则降级生成”。

13.4 技巧库对各模块的强制影响范围

模块2 / 模块3：人物定位、人设主标签、差异化优势、信任来源与人物主线必须体现对应路线技巧库的定位逻辑。

模块4：核心受众、首批最核心成交对象、一级栏目、高传播切口、高转化切口必须体现对应路线技巧库的内容逻辑。

模块5 / 模块6：平台优先级、交付形式、私域承接、转化钩子、客户教育路径必须体现对应路线技巧库的平台逻辑与转化逻辑。

模块8：账号名称、简介、风格建议必须体现对应路线技巧库的账号包装逻辑。

模块9 / 模块10：阶段规划、首发内容、第一周建议、执行总路线图必须体现对应路线技巧库的内容递进逻辑与执行逻辑。

13.5 技巧库缺失时的降级生成规则

若 skill_library_status = missing，生成智能体仍可继续生成，但必须显式标记为 fallback 状态。

fallback 状态下，只允许按 route_type 对应的默认路线规则生成，不得伪装成已经使用某路线技巧库的风格。

当 fallback 状态发生时，必须在模块11/终版说明中记录：缺失的是哪套技巧库、哪些建议因此只具备中等置信度、下一轮最应补入哪类技巧库。

14. 面向真实用户的去AI感写作与用户优先阅读生成规则

14.1 报告写作的用户友好原则

最终报告不能只像系统分析文档，还必须像写给真实用户看的执行报告。

语言风格必须少 AI 腔、少空话、少模板化表达、少抽象理论，多用用户能听懂的话，多写清“这是什么意思”“为什么这样做”“现在先做什么”。

对没有专业背景的小白用户，也要尽量做到：看完能懂、看完敢做、看完知道第一步。

14.2 用户优先阅读顺序规则

在正式报告主体中，必须尽量前置：一句话人物定义、一句话价值主张、账号定位、当前最适合的内容主线、第一条内容建议、第一周建议、最先做的 3 件事、平台建议、账号名/简介建议、是否适合马上开始做内容。

路线识别细节、商业理解、详细分析过程、证据展开、完整阶段选题大库、长篇风险说明等可后置，让用户想看再看。

若篇幅发生冲突，应优先保留前置行动信息，不得把最重要的执行建议埋在报告尾部。

14.3 核心模块的速读层固定要求

每个核心模块前，尽量固定生成：模块一句话结论、最重要的 3 个点、这一段看完应该立刻做什么。

速读层位于详细分析之前，详细分析用于补充证据、依据、边界与风险。

若某模块没有速读层，且用户需要通读大段分析才能知道结论，则视为用户体验不达标。

14.4 执行总路线图与启动小结生成要求

终版报告必须尽量生成“执行总路线图 / 启动小结”，至少回答：今天就做什么、本周先做什么、本月先验证什么、当前最不建议先做什么。

如果模块9 已给出大规模阶段选题，仍必须额外压缩出：第一条内容、第一周 3 条优先题、前 8 条建议顺序、哪些题先别碰、当前最适合验证的 2 条内容母线。

若用户看完后仍不知道自己先干哪三件事，则不得视为用户友好型终版。

三、模块级规则

模块1 报告基础信息

模块作用：记录报告上下文、版本、主路线、置信度与适用范围，作为后续全部模块的前置说明。

通用规则

必须包含：报告标题、用户名称、行业、核心业务、主路线、路线置信度、分析日期。

必须写明主路线是 ToB 还是 ToC，并给出 2—4 条判定依据。

如存在混合型特征，可保留次路线说明，但主体只允许走一条主路线。

ToB 专属写法

更强调行业、业务类型、目标客户组织属性。

判定依据优先引用：主要收入来源、服务对象、决策链、内容目标。

ToC 专属写法

更强调个人服务对象、用户身份、消费场景。

判定依据优先引用：主要收入来源、用户痛点、内容互动目标。

常见误区与需要重写的情况

缺少路线字段。

路线判定依据不足，仅写“偏 ToB / 偏 ToC”。

基础信息与后续内容不一致。

模块2 一句话结论与总判断

模块作用：先用 1—3 段文字给出全局判断，明确“最该做什么 / 最不该做什么 / 当前最优切入点”。

通用规则

必须同时输出：一句话IP定位结论、一句话战略建议、最不建议做的方向。

必须有明确倾向性，不允许使用“都可以”“都适合”“都能尝试”之类模糊表达。

必须基于输入事实，而不是泛泛总结。

ToB 专属写法

一句话结论应包含：身份 + 行业/业务 + 服务对象 + 核心信任价值。

战略建议优先回答：先做哪个平台、先打哪类内容、为什么。

ToC 专属写法

一句话结论应包含：个人特色 + 服务对象/用户痛点 + 核心价值。

战略建议优先回答：先强化什么人设、先做什么内容类型、为什么。

常见误区与需要重写的情况

结论太空，如“打造个人品牌”“持续输出内容”。

结论不像这个人，替换谁都成立。

没有给出“不建议做什么”。

模块3 IP定位分析

模块作用：输出用户最适合的IP路线、人设主标签、差异化优势、信任来源、内容母题等。

通用规则

必须从经历、业务、客户价值、表达风格中提炼人设，不得只写“专家型”“干货型”。

差异化优势至少写 3 条，且每条都能被访谈报告中的事实支撑。

必须区分“适合的路线”和“不适合的路线”。

ToB 专属写法

优先强化：专业感、决策经验、行业判断、结果导向。

差异化来源通常是：行业经验、真实项目、方法论、交付能力、组织理解。

ToC 专属写法

优先强化：个人特色、故事性、风格记忆点、用户代入感。

差异化来源通常是：经历反差、表达风格、结果转变、观点锋芒、生活/成长视角。

常见误区与需要重写的情况

人设标签堆砌但没有来源。

差异化优势与业务无关。

不适合路线未写，导致定位边界不清。

模块4 内容方向建议

模块作用：确定内容母题、一级栏目、核心受众、高传播切口、高转化切口、内容形式与前30天建议。

通用规则

核心受众必须细化到 1—3 类，不能泛写成“老板、创业者、自媒体人”。

一级栏目必须 3—5 个，且能长期成立。

高传播切口与高转化切口必须分开写。

前30天建议必须具体到主题方向，不能只写“先做起号内容”。

ToB 专属写法

内容母题更偏：行业误区、客户教育、决策逻辑、案例拆解、方法说明。

高传播切口：反常识观点、项目故事、最赚钱/最后悔决定。

高转化切口：服务逻辑、案例过程、FAQ、客户教育内容。

ToC 专属写法

内容母题更偏：个人故事、用户痛点、成长转变、体验结果。

高传播切口：情绪共鸣、冲突、反差、反常识表达。

高转化切口：解决方案、产品体验、方法步骤、结果证明。

常见误区与需要重写的情况

核心受众过泛。

栏目模板化，与用户经历脱节。

高传播切口与高转化切口混写。

前30天建议无法直接执行。

模块5 平台与交付建议

模块作用：判断平台优先级、平台角色分工、真人/数字人适配度、最佳交付方式、拍摄节奏与团队需求。

通用规则

必须明确“先做什么平台、不建议先做什么平台”。

必须给出真人/数字人/混合型的明确判断。

必须说明平台与交付方式为什么这样搭配。

ToB 专属写法

优先平台通常为：视频号、公众号；小红书和抖音视行业补充。

真人更适合高信任感内容，数字人更适合高频基础输出。

混合型是默认推荐，但必须说明比例与场景。

ToC 专属写法

优先平台通常为：抖音、小红书；视频号和公众号按内容深度补充。

真人优先，数字人只适用于信息类/工具类补充。

图文与短视频混合通常更常见。

常见误区与需要重写的情况

平台建议过散，像“全平台都做”。

没有解释为什么适合真人或数字人。

交付方式与执行条件明显矛盾。

模块6 私域与转化建议

模块作用：把内容与曝光、教育、咨询、成交前信任等转化环节接起来，形成可执行的承接建议。

通用规则

必须明确内容主要服务哪个环节。

必须给出至少 1 条私域承接建议、1 条资料包建议、1 条转化钩子建议。

建议必须与用户真实成交路径相匹配。

ToB 专属写法

更强调：评论区引导、私信关键词、企微/微信承接、资料包、案例/FAQ/客户教育内容。

销售内容建议要偏：案例包、异议处理、FAQ、会后跟进内容。

ToC 专属写法

更强调：私信关键词、社群承接、清单/小课/体验页、情绪与痛点连续承接。

资料包更偏：清单、方法包、测评、入门指南。

常见误区与需要重写的情况

承接建议太泛，只写“引导私信”。

资料包与业务无关。

没有体现成交路径。

模块7 风险与限制建议

模块作用：明确当前报告仍缺什么、哪些建议有风险、哪些执行条件不足。

通用规则

必须包含：信息缺口、内容风险、表达风险、执行风险。

如涉及客户名称、报价、授权、数字人分身等，必须补充合规风险。

风险不能空泛，必须配优先修复项。

ToB 专属写法

重点风险：过官、过空、过度行业黑话、案例不足、内容与成交脱节。

执行风险更看重时间、团队、内容持续性、真人出镜条件。

ToC 专属写法

重点风险：人设不鲜明、内容不抓人、表达没记忆点、转化链路弱。

执行风险更看重更新频率、平台分散、内容同质化。

常见误区与需要重写的情况

只写问题不写修复方向。

没有优先级。

风险与前文建议相互矛盾。

模块8 账号定位建议

模块作用：输出账号名称、账号简介、内容类型建议、风格建议、视觉建议等。

通用规则

账号名称至少 3 个；账号简介至少 3 版。

账号简介必须适合短视频平台，不可过长。

名称与简介必须和前面的IP定位保持一致。

ToB 专属写法

名称更偏专业可信、行业清晰、业务明确。

简介更偏身份+价值+服务对象，不宜过度娱乐化。

ToC 专属写法

名称更偏个人特色、记忆点、风格感。

简介更偏个人标签+用户价值+平台感。

常见误区与需要重写的情况

名称像公司Slogan，不像账号名。

简介太长太虚。

名称/简介与路线不一致。

模块9 内容阶段规划与选题建议

模块作用：按照认知→信任→咨询/成交前教育（ToB）或认知→共鸣→信任→转化（ToC）的逻辑分阶段给出选题。

通用规则

必须至少分 3 个阶段，建议 4 个阶段。

每阶段必须有阶段目标。

每阶段至少给出 20 个选题。

选题要写成可直接使用的标题，不只是方向词。

ToB 专属写法

阶段建议通常为：认知→信任→咨询前教育→长期沉淀。

选题更偏行业认知、案例、FAQ、服务逻辑。

ToC 专属写法

阶段建议通常为：认知→共鸣→信任→转化。

选题更偏故事、痛点、反差、结果展示。

常见误区与需要重写的情况

阶段划分模糊。

标题不够像这个人/这个业务。

选题数量不足。

模块10 后续执行建议

模块作用：收束全报告，明确未来30天/90天怎么做、优先调用哪些后续模块。

通用规则

必须给出最先做的 3 件事。

必须给出未来 30 天建议；90 天建议可选。

必须明确是否建议调用后续模块。

不能再出现人工介入作为常规出口。

ToB 专属写法

强调：先聚焦平台、先做高信任感内容、先建立咨询承接。

常见后续模块：脚本、多平台内容、拍摄/数字人、私域承接、销售赋能。

ToC 专属写法

强调：先做高频验证、先强化人设、再放大转化路径。

常见后续模块：脚本、选题、多平台改写、私域承接、产品转化。

常见误区与需要重写的情况

执行建议太宏观。

仍写“人工复核后交付”。

没有下一步动作。

九、面向个人IP打造用途的增强总规则

在不改变既有 10 个模块主骨架的前提下，本文件新增“人物/IP建模优先”的分析要求。生成报告时，除了保留项目理解、商业理解、route support 等原有有效内容外，还必须把用户进一步写成“为什么值得被关注、适合怎样做个人IP、后续内容如何持续输出”的结构化对象。

若原有内容更偏“项目与产品理解报告”，则保留其商业价值，但必须在对应模块中继续往下补充人物主线、故事资产、表达规则、信任资产与启动建议，避免报告只像业务说明书。

所有新增判断仍需建立在访谈事实之上，不得脱离输入自由脑补；但对于已经出现的故事线索、表达特征、业务反差、客户类型与 route support 信息，必须更主动地向“人物/IP建模”方向展开。

模块增强的目标不是单纯增加字数，而是提高可调用性：让报告能直接进入第二阶段定位、第三阶段选题、第四阶段文案、账号起号执行与信任素材沉淀。

十、统一增强深度结构（适用于核心模块）

除原有字段和写法要求外，核心模块默认尽量补齐以下深度层：已确认信息、证据与支撑细节、深层分析与判断、对IP打造的具体价值、当前缺口与问题、待追问问题、下游调用建议。

已确认信息：不能只写一两句摘要，应尽量把已有事实展开，说明背景、边界、前后关系。

证据与支撑细节：尽量把支撑判断的原话、经历、行为、决策、资料、案例、现象写出来，不得只给结论不给依据。

深层分析与判断：需要继续分析“这说明了什么、为什么重要、对IP打造意味着什么”，避免停留在表层概括。

对IP打造的具体价值：要尽量说明这部分信息对定位、选题、文案、人设、信任建立、起号执行的具体作用。

当前缺口与问题：不能只说“还缺数据”，要说明缺什么、为什么缺这个会影响后续、如果不补会带来什么问题。

待追问问题：尽量补出后续可继续深挖的问题，而不是只给一句笼统的“建议下一步”。

下游调用建议：明确这部分更适合供第二阶段定位、第三阶段选题、第四阶段文案、起号执行还是信任素材沉淀调用。

十一、原 10 个模块的增强写法要求

模块2 一句话结论与总判断

除原有一句话定位、一句话战略建议外，必须尽量回答“为什么这个人值得被关注”“最适合以什么身份做个人IP”“当前最优切入点是什么”。结论要更像人物识别，而不只是产品总结。

模块3 IP定位分析

除人设主标签、差异化、信任来源、内容母题外，必须补强人物/IP主线，尽量写出最强身份标签、最大反差点、最适合被记住的个人主线，以及别人为什么会愿意持续关注这个人。

模块4 内容方向建议

除核心受众、栏目、切口外，必须进一步细化首批最核心成交对象、当前最适合切入的人群、当前最不优先服务的人群，并尽量形成可直接对外表达的收窄版本。

模块5 平台与交付建议

除平台和交付方式外，尽量判断“如果现在启动内容，最适合用口播 / 访谈 / 混合 / 图文中的哪一种作为主形式”，并结合表达条件解释原因。

模块6 私域与转化建议

除转化路径外，必须同步梳理“信任证明材料如何承接成交”，避免承接建议只有动作没有支撑。资料包、FAQ、案例包、演示页等都应尽量关联用户已有或缺失的信任资产。

模块7 风险与限制建议

必须解释这些风险会怎样具体影响后续IP打造，不可只列问题；并尽量按优先修复顺序排列，让执行方知道先补什么。

模块8 账号定位建议

名称、简介、风格建议应尽量贴合人物主线和表达风格，而不是只围绕业务类别命名。

模块9 内容阶段规划与选题建议

除原有阶段化选题外，尽量指出第一阶段最适合起号的内容切入口、当前最适合的首发内容方向，以及起号初期更适合讲判断、故事、方法还是项目过程。

模块10 后续执行建议

除30天/90天动作外，应尽量明确现在是否适合马上开始做内容、适合什么频率、哪些题暂时不要讲，并说明原因。

模块11 生成说明与自检记录

除路线识别与修订记录外，建议同步记录哪些内容已沉淀成可调用资产、哪些仍属于低置信度判断、下一轮应重点补采什么信息。

十二、新增增强模块规则（在原骨架基础上追加，不替代原模块）

增强模块A 人物主线与IP核心识别

模块作用：把“这个人为什么值得被关注”写透。必须尽量包含一句话人物定义、一句话价值主张、最强身份标签、最大反差点、最适合被记住的个人主线、为什么别人应该关注他。ToB 用户更强调“值得托付、值得咨询、值得长期信任”；ToC 用户更强调“记忆点、共鸣点、传播点与个人吸引力”。常见误区：写成泛泛的“专家型、干货型”而没有人物主线。

增强模块B 关键故事资产卡

模块作用：把已有故事线索升级为可直接调用的故事资产。每张卡尽量补出故事标题、背景、冲突点、关键决策、结果、认知提炼、适合用于什么类型的内容。不得只停留在“一句话经历摘要”。常见误区：只有故事梗概，没有内容用途。

增强模块C 表达风格结构化判断

模块作用：把抽象风格判断转成可供文案系统调用的语言规则。尽量补充短句/长句偏好、先讲判断还是先讲故事、强观点还是温和解释、是否适合反问/对比/举例、潜在口头禅、哪些表达不自然。常见误区：只写“真诚、理性、适合口播”而不继续展开。

增强模块D 信任资产清单

模块作用：系统梳理已有 / 缺失 / 优先补齐 / 适用场景。至少考虑量化成果、案例材料、截图反馈、咨询数据、系统输出样例、演讲/媒体/公开背书等。必须说明这些资产适合用于主页、短视频、私域、销售页、成交前沟通中的哪一个环节。常见误区：只提醒“缺数据”，但没有形成清单。

增强模块E 内容启动准备度判断

模块作用：把报告从“能分析”升级为“能启动”。必须尽量明确现在是否适合开始做内容、适合什么形式、适合什么频率、适合什么第一批内容主线、哪些题暂时不要讲。常见误区：只给长期建议，没有起号起步建议。

十三、重点低质量问题的判定与重写触发

若报告明显“产品写得比人完整”，则必须回到模块2、模块3或增强模块A补强人物主线，不得仅在结论段增加几句修辞。

若故事只像经历摘要而不可调用，则必须在增强模块B中把故事补成资产卡。

若表达风格仍然只停留在“真诚、理性、适合口播”等粗标签，则必须回到增强模块C继续拆出语言规则。

若信任证明材料只被写成“缺口提醒”，没有形成清单，则必须补写增强模块D。

若主路线已识别，但“后续IP内容表达主场”仍不清楚，则需要在模块2、模块3或模块4中补出主路线表达、次路线补充与混讲风险。

十四、最终交付前的增强自查补充

报告是否已经从“项目与产品理解报告”为主，升级为“第一阶段访谈建档 + IP可用资产沉淀报告”。

是否能够清楚回答：这个人为什么值得被关注、最适合以什么身份做个人IP、最强故事与反差是什么、长期内容主线是什么、表达风格如何被结构化、已有与缺失的信任资产是什么、现在是否适合启动内容。

如果上述问题未被清楚回答，应优先补写增强模块A-E，而不是只优化文案流畅度。

十五、针对当前定位报告质量问题的专项增强规则

若报告“产品逻辑完整，但人物主线不强”，则模块2、模块3与增强模块A必须前置回答：一句话人物定义、一句话关注理由、一句话反差记忆点、一句话为什么他不是普通 AI 工具号。未回答清楚时，不得视为终版。

若“首批用户收窄不够狠”，则模块4必须强制拆成：首批最核心成交对象、次级可覆盖对象、当前不优先服务对象，并给出一版可直接对外使用的人群表达。

若“有故事但不可调用”，则增强模块B必须采用固定故事资产卡写法：故事标题、背景、冲突、转折、关键决策、结果、一句认知、内容用途。不得只保留经历摘要。

若“表达风格过粗”，则增强模块C必须继续下钻为语言规则：短句/中长句偏好、先讲判断还是先讲经历、强观点还是解释推进、是否适合反问/对比/举例、口头禅候选、禁用表达、容易不像本人的表达。

若“信任来源有但未形成体系”，则增强模块D必须拆成：已有、缺失、优先补齐、适用场景。仅写“缺案例、缺样例、缺反馈”视为未完成。

若“路线识别对了，但主场表达不彻底”，则模块2、模块3、模块4或路线补充段中必须写明：主路线表达主场、次路线只作什么背书、哪些内容可讲、哪些内容不宜作为主表达、混讲风险。

若“内容阶段规划多，但启动优先级不清”，则模块9或模块10必须新增“小启动包”：第一条内容、第一周 3 条优先题、前 8 条建议顺序、当前最适合验证的 2 条内容母线、哪些题先别碰。

十六、用户体验导向的模块写法补充

每个核心模块尽量增加“速读层”：模块一句话结论、最重要的 3 个点、这一段看完应该立刻做什么。若篇幅受限，至少在段首前置一句总判断。

每个关键模块尽量增加“被理解感”证据：原话证据、行为证据、故事证据。尤其是定位、人设、差异化、表达风格四类模块，不能只给结论。

账号定位建议不再只是平铺式罗列，建议输出：首选方案、备选方案、暂不建议方案、适用时机。

后续执行建议不只写 30/90 天动作，还要尽量生成“执行总路线图”，把今天就做、本周做、本月做压缩给出。

新增硬性字段：故事标题、背景、冲突、转折、关键决策、结果、一句认知、内容用途。

增强模块C 表达风格结构化判断

新增硬性字段：句式长度偏好、开篇方式偏好、观点强度偏好、修辞偏好、口头禅候选、禁用表达、失真表达。

增强模块D 信任资产清单

新增硬性字段：已有 / 缺失 / 优先补齐 / 适用场景。适用场景至少覆盖主页、短视频、私域、销售页、成交前沟通。

模块9 / 模块10

新增硬性字段：第一条内容、第一周 3 条优先题、前 8 条建议顺序、哪些题先别碰、当前最适合验证的 2 条内容母线。

十八、专项重写触发条件补充

若人物主线仍然被产品说明压住，则必须回到模块2、模块3重写，不得仅在模块10加几句总结。

若用户看完仍不知道“先干哪三件事”，则必须补写执行总路线图。

若用户看完仍然不知道“先用哪个账号名、为什么先用它”，则模块8必须补写首选/备选/暂不建议/适用时机。

若报告缺少证据层，只剩结论层，则对应模块必须补写 evidence 级内容，而不是优化措辞。

十九、路线识别后的技巧库自动调用与模块联动规则

路线识别一旦通过，后续所有模块不仅要服从主路线，还必须自动挂载与主路线一致的创作技巧库作为正式生成约束源。ToB 模块必须按 ToB 创作技巧库逻辑写；ToC 模块必须按 ToC 创作技巧库逻辑写。技巧库不是附属说明，而是模块级写法、建议风格、栏目判断、平台判断、转化判断和执行建议的直接约束源。

若 route_type = TobFounder，则默认挂载 ToB 创作技巧库；若 route_type = TocPersonal，则默认挂载 ToC 创作技巧库。

若主路线与技巧库路线不一致，必须视为高风险，并优先修正技巧库挂载或回到路由判断重判。

若缺少对应技巧库，不得伪装已调用，必须按 route_type 默认规则降级生成，并在模块11/终版说明中记录。

建议新增字段：skill_library_source、skill_library_status、skill_library_route_match、skill_library_fallback_note。

二十、技巧库对各模块写法的具体影响

模块2 / 模块3

ToB 技巧库应强化专业感、决策信任、案例能力、责任与托付逻辑；ToC 技巧库应强化个人记忆点、共鸣点、传播点与用户感受。

人物主线、差异化优势、信任来源、人设主标签必须体现对应技巧库中的路线语言。

模块4

受众拆分、首批最核心成交对象、一级栏目、高传播切口、高转化切口必须服从对应路线技巧库的内容逻辑。

ToB 不得把情绪共鸣式内容当作默认主线；ToC 不得把客户教育式内容当作默认主线。

模块5 / 模块6

平台建议、交付方式、私域承接与转化路径必须服从对应路线技巧库的平台逻辑与转化逻辑。

ToB 必须更强调视频号/公众号/咨询前教育/资料包；ToC 必须更强调抖音/小红书/互动/体验转化。

模块8 / 模块9 / 模块10

账号名、简介、阶段规划、首发内容、执行总路线图必须体现对应路线技巧库中的包装逻辑、内容递进逻辑与执行逻辑。

首发建议和第一周建议必须像真实用户能立刻开工的动作，而不是说明书式阶段摘要。

二十一、模块级用户友好浓缩区规则

为了降低 AI 感、提升用户体验，每个核心模块在保留原有完整分析层的前提下，默认增加“速读层”。速读层用于让用户快速抓住重点，不需要先读完整段分析。

模块一句话结论：本模块最重要的判断。

最重要的 3 个点：把用户最该记住的三条信息前置。

这一段看完应该立刻做什么：将模块建议压缩为 1 个最直接动作。

若模块内容只有长段说明、没有速读层，则用户体验视为不达标。

二十二、去AI感与说明书感控制规则

模块内容必须兼顾系统可调用与用户可看懂；不能只追求结构齐全而忽略可读性。

减少抽象总结、空洞形容词和模板化话术，多写“具体怎么做”“为什么这样做”“现在先做什么”。

若用户读完仍不知道模块结论是什么、先做什么、为什么这样做，则对应模块必须重写速读层或执行层。

二十三、前置行动区与执行总路线图规则

在报告装配阶段，必须尽量前置：一句话人物定义、一句话价值主张、账号定位、当前最适合内容主线、第一条内容建议、第一周 3 条建议、最先做的 3 件事、平台建议、账号名/简介首选、是否适合马上开始做内容。

在模块9 / 模块10 中，必须额外压缩出：第一条内容、前 8 条建议顺序、哪些题先别碰、当前最适合验证的 2 条内容母线。

若用户看完完整报告仍然不知道“今天就做什么 / 本周先做什么 / 本月先验证什么”，则必须补写执行总路线图。

二十四、专项重写触发条件补充（技巧库与用户体验）

若 route_type 已确认，但正文仍然主要按通用逻辑写，而没有体现对应技巧库风格，则必须回到受影响模块重写。

若技巧库缺失却没有在终版说明中显式标记 fallback 状态，则不得视为终版。

若报告最重要的执行建议仍然放在后半段，前半段只有长篇分析，则必须重写前置摘要区与行动区。

若模块速读层缺失，或存在但仍然不能让用户快速抓住重点，则必须重写模块开头。

若用户看完仍然感到“像 AI 给的长说明书”，则优先检查：是否前置了账号定位、内容主线、首发建议、第一周动作与执行总路线图。

六、最终交付前自查清单

是否已写明 ToB / ToC 主路线、置信度与依据；

整份报告是否从头到尾都遵循同一主路线，没有混写；

一句话定位是否像这个人、像这个业务，而不是通用模板；

核心受众、栏目、平台顺序、交付方式是否足够具体；

账号名称与简介是否符合平台语感且与前文一致；

阶段规划是否清晰，每阶段选题数量是否达标；

私域与转化建议是否可执行，而非泛泛建议；

风险与限制是否指出真实问题，而不是“整体可行”；

是否保留生成说明与自检记录；

最终版本能否直接交付给用户使用。

交付判定建议

若路线识别不清晰、选题数量不达标、平台与交付建议过散，不能判定为终版；

若模块间存在明显冲突，应回到对应模块补足依据或重写；

若信息不足但系统仍需交付，需在对应模块标注“低置信度自动交付说明”。

七、面向个人IP打造用途的增强输出总要求

在保留本模板原有 10+1 模块主骨架、路由前置说明、ToB / ToC 双路线写法与自检要求不变的前提下，本模板新增“面向个人IP打造用途”的增强输出要求。最终报告不应仅是项目与产品理解报告，而应升级为“IP分析与定位报告 + IP可用资产沉淀报告”。

除保留原有项目理解、商业理解、route support 与平台/转化建议外，报告还必须更强地回答以下问题：这个人为什么值得被关注；这个人最适合以什么身份做个人IP；他身上最有传播力、最有记忆点的故事与反差是什么；他最适合长期讲的内容主线是什么；他的表达风格如何被结构化以供后续文案系统调用；目前有哪些可直接用于建立信任的资产、哪些仍缺；现在是否已经具备启动内容的条件，若启动，应怎么启动。

若报告在“产品理解”与“人物/IP建模”之间出现篇幅冲突，应优先保证人物主线、故事资产、表达规则、信任资产与启动建议足够具体。

八、原 10+1 模块的增强写法要求

02. 一句话结论与总判断（增强要求）

除原有一句话定位、一句话战略建议、核心增长逻辑、最不建议做的方向与最优切入点外，建议显式补出：为什么这个人值得被关注；最适合以什么身份做个人IP；当前最优切入点为什么是这个。

03. IP定位分析（增强要求）

除人设主标签、差异化优势、信任来源、内容母题、适合/不适合路线外，建议进一步补出：最强身份标签、最大反差点、最适合被记住的个人主线、别人为什么应持续关注他。

04. 内容方向建议（增强要求）

除核心受众、不想吸引的人、一级栏目、高传播切口、高转化切口与前30天重点外，建议补出：首批最核心成交对象、当前最适合切入的人群、当前最不优先服务的人群，并尽量形成可直接对外表达的收窄版本。

05. 平台与交付建议（增强要求）

除平台优先级、平台分工、真人/数字人判断、最佳交付方式、拍摄节奏与团队需求外，建议进一步明确：如果现在启动内容，最适合以口播 / 访谈 / 混合 / 图文中的哪一种作为主形式，以及为什么。

06. 私域与转化建议（增强要求）

除内容目标环节、私域承接、资料包、销售内容、转化钩子与客户教育外，建议同步补出“信任资产如何服务成交”，避免承接动作与信任证明脱节。

07. 风险与限制建议（增强要求）

除信息缺口、内容风险、表达风险、合规风险、执行风险与优先修复项外，建议说明这些风险会怎样具体影响后续IP打造，而不仅是列清单。

08. 账号定位建议（增强要求）

账号名称、账号简介、内容类型、账号风格与视觉建议，建议尽量贴合人物主线与表达风格，而不是只围绕业务类别命名。

09. 内容阶段规划与选题建议（增强要求）

在保留阶段规划与大量选题的前提下，建议补出第一阶段最适合起号的内容切入口、当前最适合的首发内容方向，以及起号初期更适合讲判断、故事、方法还是项目过程。

10. 后续执行建议（增强要求）

除最先做的3件事、30天/90天建议与验证指标外，建议明确：现在是否适合马上开始做内容；如果开始，最适合什么形式、什么频率；哪些题暂时不要讲。

九、新增增强模块（在原 10+1 模块基础上追加，可作为子模块插入，不替代原模块）

增强模块A｜人物主线与IP核心识别

建议插入位置：02 与 03 之间，或作为 03 的子模块。

必含内容：一句话人物定义；一句话价值主张；最强身份标签；最大反差点；最适合被记住的个人主线；为什么别人应该关注他。

写作提醒：此模块的核心不是“他做什么”，而是“他为什么值得被看见”。

增强模块B｜关键故事资产卡

建议插入位置：03 之后，或作为 09 的内容资产池附录。

写作提醒：不得只写经历摘要；要写成后续选题与文案可直接调用的故事卡。

增强模块C｜表达风格结构化判断

必含内容：更适合短句还是长句；更适合先讲判断还是先讲故事；更适合强观点还是温和解释；是否适合反问、对比、举例；有无潜在口头禅或表达偏好；哪些表达不自然、不适合本人。

写作提醒：目标是给后续文案系统提供“像本人”的写作依据。

增强模块D｜信任资产清单

必含内容：当前已有的信任资产；当前缺失的信任资产；优先补齐的信任资产；各类信任资产适合用于哪里（主页、短视频、私域、销售页、成交前沟通等）。

写作提醒：不能只停留在“缺口提醒”，要形成可执行清单。

增强模块E｜内容启动准备度判断

建议插入位置：10 之后。

必含内容：现在是否适合开始做内容；适合什么形式；适合什么频率；适合什么第一批内容主线；哪些题暂时不要讲。

写作提醒：该模块用于把分析报告进一步变成起号执行中间件。

十一、增强版最终交付前补充自查

是否已经从“项目与产品理解报告”为主，升级为“人物/IP建模 + 可调用资产沉淀 + 可执行建议”的报告。

如果上述问题仍未被清楚回答，应优先补写增强模块A-E，而不是只优化措辞。

十二、针对当前定位报告质量问题的专项增强输出要求

在保留原有路线识别前置、10+1 模块主骨架、增强模块 A-E、ToB / ToC 双路线写法与自检要求不变的前提下，本模板新增“面向个人IP打造 + 用户体验优化”的专项增强要求。新增内容的目标不是把报告写得更长，而是解决以下质量问题：产品逻辑写得比人物主线更完整、首批用户收窄不够狠、故事线索没有沉淀成可调用的故事资产、表达风格判断过粗、信任来源没有形成资产清单、主路线识别后内容表达主场仍然摇摆、内容阶段规划量大但启动优先级不清。

最终报告必须进一步升级为“IP分析与定位报告 + IP可用资产沉淀报告 + 起号启动包”。若篇幅发生冲突，应优先保证：人物主线、首批用户收窄、故事资产、表达风格规则、信任资产、主路线表达主场、启动动作与执行总路线图。

十三、核心模块新增“速读层”与“证据层”写法要求

从本版开始，核心模块默认增加“速读层”。速读层用于提升用户体验，让用户即使不阅读全文，也能快速抓住重点。

每个核心模块前建议固定增加以下三个字段：

模块一句话结论：这一模块最重要的判断。

最重要的 3 个点：将本模块最关键的 3 条结论前置。

这一段看完应该立刻做什么：把建议压缩为一个直接动作。

同时，每个核心模块默认增加“证据层”，至少尽量体现以下三种证据中的一种或多种：原话证据、行为证据、故事证据。若无法提供逐字原话，也应在文本中说明判断依据来自哪类经历、决策、资料或行为。

十四、针对重点问题的模块级强制补强要求

模块02｜一句话结论与总判断（追加必含项）

除原有字段外，建议显式追加：一句话人物定义；一句话关注理由；一句话反差记忆点；一句话为什么他不是普通 AI 工具号。

写作目标：先让读者记住“这个人是谁”，再让读者理解“他做什么”。若模块02只把产品和系统说清，却没有把人物主线压到前面，则视为不达标。

模块03｜IP定位分析（追加必含项）

除原有一句话定位、人设主标签、差异化优势、信任来源、内容母题外，必须尽量补出：最强身份标签、最大反差点、最适合被记住的个人主线、别人为什么应该持续关注他。

若人物主线仍然被业务主线压住，应在模块03内单独插入“人物/IP主线强化段”，而不是只在差异化优势里零散提一句。

模块03A｜关键故事资产卡（追加固定字段）

每张故事资产卡尽量固定为：故事标题、背景、冲突、转折、关键决策、结果、一句认知、适合用于什么类型的内容（首发口播 / 信任内容 / 转化内容 / 长文 / 小红书笔记等）。

若某条故事只写成“我差点不做这个项目了”这类摘要，不可视为故事资产卡完成。

模块03B｜表达风格结构化判断（追加固定字段）

除原有表达风格判断外，必须尽量补出：更适合短句还是中长句；更适合先下判断还是先讲经历；更适合强观点还是解释式推进；能不能反问；能不能举例；潜在口头禅候选；禁用表达；哪些表达容易显得不像本人。

该模块的目标是给后续文案系统提供“像本人”的语言规则，而不是只给性格标签。

模块04｜内容方向建议（追加受众收窄层）

除核心受众与不想吸引的人外，必须进一步拆出：首批最核心成交对象、次级可覆盖对象、当前不优先服务对象，并尽量形成一版可直接对外说的人群表达。

若核心受众同时囊括“完全不会开始的人、已经在做但效率低的人、有专业积累但不会表达的人”等多类对象，必须进一步判断首阶段起号更应先服务哪一类。

模块05/06｜平台、交付与转化（追加主场表达与信任承接）

模块05除平台和形式外，应尽量判断：当前最适合的主形式是口播 / 访谈 / 混合 / 图文中的哪一种，以及为什么。

模块06除私域与转化建议外，必须同步写出“信任资产如何承接成交”，把资料包、样例、FAQ、案例包、演示页与成交前沟通场景连起来。

模块07｜风险与限制（追加影响说明）

风险不只列清单，还应说明：这些风险若不修复，会怎样具体影响后续IP打造、内容表达、信任建立、起号效率与转化承接。

模块08｜账号定位建议（追加决策层）

账号名称与简介建议不再只给平铺列表，建议增加：首选方案、备选方案、暂不建议方案、适用时机。

模块09/10｜内容阶段规划与后续执行（追加启动包）

除原有阶段规划、海量题库与 30/90 天建议外，必须额外压缩为“小启动包”，至少包含：第一条内容、第一周 3 条优先题、前 8 条建议顺序、哪些题先别碰、当前最适合用来验证的 2 条内容母线。

十五、新增增强模块 F-H（在原 A-E 基础上追加）

增强模块F｜主路线表达主场与次路线边界

建议插入位置：02 或 03 之后。

必含内容：主路线表达主场；次路线只作为哪类信用背书；哪些 ToB / ToC 内容可以讲；哪些内容不宜作为主表达；哪些内容用来证明你懂复杂业务，哪些内容用来吸引当前主受众。

写作提醒：ToB 用来证明你有厚度，不用来改变你的主受众；ToC 用来打开传播，不等于把主体逻辑改成泛情绪博主。

增强模块G｜执行总路线图

建议插入位置：10 之后，10A 之前或之后均可。

必含内容：今天就做、本周做、本月做、90 天内验证什么。

写作提醒：该模块用于解决“建议很多，但执行顺序不够一眼看懂”的问题。

增强模块H｜首批启动包

建议插入位置：09 或 10 之后。

必含内容：第一条内容、第一周 3 条优先题、前 8 条建议顺序、当前最适合验证的 2 条内容母线、哪些题暂时不要讲。

写作提醒：海量题库仍然保留，但必须额外生成一份可马上开工的小启动包。

十六、面向个人IP打造与用户体验的最终补充自查

是否已经把“人物主线”前置，而不是让产品说明压住人物识别。

是否已经把首批用户拆成：首批最核心成交对象、次级可覆盖对象、当前不优先服务对象。

是否已经把关键经历写成可调用的故事资产卡，而不是经历摘要。

是否已经形成“信任资产清单”，并写明这些资产分别适合用在哪里。

是否已经在路线识别后的主体报告中写清：主路线表达主场、次路线如何补充、混讲风险是什么。

是否已经把大题库压缩成：第一条内容、第一周 3 条优先题、前 8 条建议顺序、哪些题先别碰。

是否已经让每个核心模块都有：模块一句话结论、最重要的 3 个点、这一段看完应该立刻做什么。

十七、路线识别后的技巧库挂载前置说明

在正式装配报告前，除 route_type、route_confidence、route_evidence 外，系统还应写入 skill_library_source、skill_library_status、skill_library_route_match、skill_library_fallback_note。主路线为 ToB 时，正式报告必须自动挂载 ToB 创作技巧库；主路线为 ToC 时，正式报告必须自动挂载 ToC 创作技巧库。技巧库不是外部附属说明，而是后续定位判断、内容主线、账号定位、平台建议、转化建议、阶段规划与执行方案的正式约束源。

若 skill_library_status = loaded，则后续写法必须明显体现对应路线技巧库逻辑。

若 skill_library_status = missing，则必须在生成说明中写清“当前缺少对应技巧库支持，已按主路线默认规则降级生成”。

若 skill_library_route_match = mismatch，则不得直接装配终版，必须先修正挂载关系或回退路由。

十八、正式报告新增“00 前置摘要与行动区”

在保留原 10+1 主骨架不删除、不缩减的前提下，正式报告建议在模块 01 之前新增“00 前置摘要与行动区”，用于把最重要的用户执行信息前置。该区域不替代原模块，只承担“先看这一段就够”的用户体验职责。

建议结构如下：

00. 前置摘要与行动区

— 一句话人物定义

— 一句话价值主张

— 一句话关注理由 / 反差记忆点

— 当前最适合的账号定位

— 当前最适合的内容主线

— 第一条内容建议

— 第一周 3 条优先建议

— 最先做的 3 件事

— 平台建议（先做什么，不先做什么）

— 账号名 / 简介首选方案

— 是否适合马上开始做内容

— 执行总路线图 / 启动小结

十九、用户友好与去AI感写作总要求

报告不仅要给系统看，也要给真实用户看；普通用户、小白用户也应尽量一看就懂，一看就知道该怎么做。

写法要少 AI 腔、少空话、少模板化表达、少抽象理论，多用用户能听懂的话，多写“为什么这样做”“现在先做什么”。

不能只像长说明书或策略报告；要更像一份能直接拿着起号与执行的行动型报告。

二十、核心模块统一新增“速读层”

在保留各模块原有详细内容的前提下，核心模块建议统一采用“速读层 + 详细分析层”的双层呈现。速读层位于模块开头，详细分析层保留原有完整判断、证据与建议。

速读层固定建议包含：

模块一句话结论

最重要的 3 个点

这一段看完应该立刻做什么

若某核心模块没有速读层，且用户必须读完全部长文才能知道结论，则该模块用户体验视为不达标。

二十一、原 10+1 主骨架的前置展示增强要求

模块02 与模块03：必须尽量前置人物主线、关注理由、反差记忆点与最适合的身份表达。

模块04：必须尽量前置当前最适合的内容主线、首批最核心成交对象、当前最适合切入的人群与第一阶段不优先服务对象。

模块05 / 模块06：必须尽量前置平台优先级、最适合的主形式、第一轮转化承接方式与信任证明如何服务成交。

模块08：账号名称与简介建议必须提供首选方案、备选方案、暂不建议方案、适用时机。

模块09 / 模块10：在保留大题库与长期规划的前提下，必须额外压缩出第一条内容、第一周 3 条优先题、前 8 条建议顺序、哪些题先别碰、执行总路线图。

二十二、执行总路线图 / 启动小结推荐模板

执行总路线图（建议放在报告前部）

— 今天就做：定账号定位 + 选账号名/简介首选 + 确定第一条内容

— 本周先做：完成 2—3 条内容拍摄与发布 + 建立最小信任资料包 + 记录首批私信问题

— 本月先验证：跑通前 8 条建议顺序 + 找到反馈最好的一条内容主线 + 沉淀首批样例或反馈

— 当前最不建议先做：一开始就全平台铺开 / 一开始就讲过宽的话题 / 一开始就把高转化内容当成高传播内容

二十三、面向用户体验的最终交付补充要求

用户在前 1—2 页内，应尽量先看到：你是谁、为什么值得被关注、账号怎么定位、内容主线是什么、第一条发什么、第一周发什么、最先做哪三件事。

路线识别细节、商业理解、详细分析过程、证据解释、风险展开、完整阶段题库可以后置，供用户想看再看。

若报告内容仍然主要像“AI 分析报告”或“长说明书”，而不是“用户可直接执行的起号方案”，则应优先补写前置摘要区、速读层与执行总路线图，而不是单纯删字。
