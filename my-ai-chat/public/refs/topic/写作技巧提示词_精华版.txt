───────────────────────────────────────────────────────────────
一、总纲：创作核心原则
───────────────────────────────────────────────────────────────

1. 先服从用户本人与真实资料，再调用技巧库；技巧不能压过人、压过事实、压过路线。
2. ToB 更重责任、机制、案例、托付；ToC 更重代入、成长、获得感、行动门槛低。
3. 内容不是孤立句子，而是和平台停留、完播、评论、转发直接相关的完整结构。
4. 先给钩子，再讲道理；先给获得感，再给方法。
5. 金句不是修辞，而是替用户下判断。
6. 真正的信任来自细节、机制、代价和结果，不来自自夸。
7. 模板是骨架，个人纹理才是灵魂。
8. 收口不要硬卖，先做轻承接，再做深转化。
9. 去AI味的关键：多用样本、素材、口语和真实经验，不是不用AI。
10. 先判断内容类型（破圈/信任/转化），再选结构；不要所有内容都用一种写法。

───────────────────────────────────────────────────────────────
二、IP定位与人设构建
───────────────────────────────────────────────────────────────

【核心原则】
  · 我的专业、经历是什么？我想吸引哪类人群？我能长期提供什么价值？
  · 我是什么领域的、帮助谁、解决什么问题。
  · 人设突出 = 领域 + 风格。
  · 固定开场白、口头禅、重复出现的视觉元素 = 记忆锚点。

【ToB 人设类型与可信度来源】
  · 创始人型：重方向判断、业务结果、关键决策、代价感与责任感。
  · 老板型：重行业经验、踩坑经历、交付标准、客户视角与托付逻辑。
  · 专家顾问型：重框架、标准、问题拆解、行业误区纠偏、方法论落地。
  · 操盘手型：重一线执行、案例拆解、流程细节、复盘能力与实操判断。
  · 交付者型：重交付过程、风控、协同、验收、细节控制与客户反馈。
  · 服务商型：重口碑、响应、项目经验、价格逻辑、合同边界、售后与托付理由。

【ToB 人设表达规则】
  · 不是装专家，而是让用户感觉你真懂业务。
  · 不是空喊结果，而是能讲清客户为什么信你。
  · 不是流量型人设，而是值得托付的人设。
  · 兼顾"懂行业、懂用户、懂表达"三个层面。

【ToC / 个人IP 表达逻辑（按优先级）】
  优先级一：真实经历。优先调用用户真实经历、真实卡点、真实变化。
  优先级二：真实判断。需要"我为什么这样想"的判断感。
  优先级三：真实口吻。尽量保留用户原有表达、句式、语气、停顿感和价值观。
  优先级四：真实边界。共鸣不是卖惨，情绪不是表演，转折不是强行逆袭。

【差异化定位五法（抖音适用）】
  · 年龄反差法：人物行为/气质与年龄形成反差。
  · 性别反差法：打破常规形象与声音预期。
  · 场景切换法：熟悉内容放进陌生场景。
  · 典故定位法：借用大众熟悉的神话/童话/典故人物。
  · 影视定位法：把已被市场验证的影视角色类型挪到账号中。

【账号主页包装规则】
  · 昵称：有辨识度、与定位相关、最好带标签。
  · 头像：与定位人设一致，能强化品牌感。
  · 简介：突出最强特点，可用适度自嘲把缺点转成标签。
  · 视频封面：统一边框、字体与风格，增强主页第一眼信任感。
  · 梗点必须反复出现，直到用户看到某个动作/某句话就想起你。

三、内容分层逻辑（破圈 / 信任 / 转化）
───────────────────────────────────────────────────────────────

【ToB 内容目标五层】

1. 破圈型内容
   · 目标：被更多潜在目标群体看见。
   · 切口：行业误区、反常识判断、反差显真、决策代价、趋势冲击。
   · CTA：宜轻，不宜硬卖。

2. 信任型内容
   · 目标：让客户判断你是否靠谱。
   · 方向：老板故事、案例拆解、机制说明、托付逻辑、客户视角、复盘内容。
   · 平台：视频号、公众号、小红书、朋友圈转发承接。

3. 转化型内容
   · 目标：把已有兴趣推进到咨询、加微信、看资料包、约沟通。
   · 必须强化：流程透明、边界清晰、案例结果、下一步动作。
   · 平台：配合私域与视频号/公众号承接。

4. 口碑扩散型内容
   · 目标：让老客户、熟人、合作方愿意转发背书。
   · 方向：客户证言、交付故事、合作复盘、团队责任感与过程感。
   · CTA：偏转发、分享、收藏。

5. 咨询前教育型内容
   · 目标：降低沟通成本、提升客户判断质量、减少无效咨询。
   · 方向："找服务商前先看什么""预算怎么判断""为什么你之前踩坑"。
   · CTA：偏"先看完资料包""先私信拿清单"。

【ToC / 个人IP 内容三层】

1. 破圈型：核心关键词"新、奇、爽"；负责流量。
2. 信任型：核心关键是"如何讲好一个故事"；负责相信你。
3. 转化型：使用结构模型推进用户行动（AIDA/PAS/FAB）；负责成交。

【各阶段推荐配比】
  · 冷启动曝光期：破圈 + 部分信任
  · 认知教育期：信任 + 咨询前教育
  · 咨询承接期：转化 + 信任补强
  · 成交辅助期：案例 + 流程 + 边界
  · 复购与口碑期：交付故事 + 口碑扩散

四、开头钩子技巧（ToB + ToC 通用精华）
───────────────────────────────────────────────────────────────

【核心要求】
  · ToB：任务不是仅仅抓眼球，而是让对的人快速判断"这条和我的业务有关，且说话的人像是懂行的"。
  · ToC：前3秒给冲突、给反差、给结果、给强代入，至少完成一项；不得用两句以上空铺垫。
  · 钩子必须15～30字，口语化，可直接对着镜头念。
  · 不空洞、不官方、有情绪、有节奏。

【ToB 高适配开头类型】（每种附句式）

1. 行业误区型
   · "很多公司在___上花了钱，但真正亏的不是这笔钱，而是___。"
   · "你以为___最重要，其实先错的往往是___。"

2. 认知纠偏型
   · "真正决定___的，不是___，而是___。"
   · "大多数人做不好___，问题根本不在___。"

3. 客户决策代价型
   · "这件事一旦选错，后面补救成本比你想的高得多。"
   · "不是贵不贵的问题，而是后面谁来兜底的问题。"

4. 机制证明型
   · "我们之所以敢这样做，不是因为胆子大，而是因为这套流程已经把风险拆掉了。"

5. 案例结果型
   · "这个项目最后能跑出来，靠的不是运气，而是前面这三步没做错。"

6. 责任托付型
   · "客户最后愿意把这事交给你，不是因为你说得多好，而是因为他知道出了问题谁负责。"

7. 反差显真型
   · "外面看起来___，真正做起来最难的其实是___。"
   · "别人觉得我们做的是___，但每天真正花时间的其实是___。"

8. 时间/数据对照型
   · "从___到___，中间真正拉开差距的不是资源，是___。"

【ToC / 个人IP 开头类型】（每种附句式）

9. 结果承诺 + 身份升级
   · "看完这条，你会变成一个___的人。"
   · "1分钟之后，你就能___。"

10. 反常识纠偏
    · "真正关键根本不是___。"
    · "大部分人做不好___的核心原因是___。"

11. 时间/数据对照
    · "X年前我还___，现在我已经___。"
    · "从___到___。"

12. 反差显真
    · "别人以为我___，但真实的我___。"
    · "看起来___，做起来___。"

13. 挑战式提问
    · "如果___，你会___吗？"
    · "你敢不敢用___个月把自己变成___？"

14. 情绪共鸣 + 悬念
    · "很多人正在担心___。"
    · "你有没有觉得___。"
    · "这可能是你最后一次___。"

【开头通用检查项】
  · 开头是否在1-3句内让用户知道"这条和我的业务/生活有关"？
  · 是否体现业务判断或观点，而不是只有情绪？
  · 是否避免ToC化强共鸣句式（如过强代入、泛人生感、纯情绪宣泄）？
  · 是否与正文目标一致，而非标题党？

───────────────────────────────────────────────────────────────
五、文案结构模板（5大骨架 + 多种变体）
───────────────────────────────────────────────────────────────

【通用爆款总公式】
  黄金3秒开头 + 2～5个爆点 + 白金结尾

【ToB 五大正文骨架】

结构一：判断 → 代价 → 案例 → 机制 → 边界 → 结论
  · 适合：顾问型、判断型、信任型内容。

结构二：行业问题 → 客户损失 → 原因 → 解决机制
  · 适合：客户教育、咨询前教育、服务商表达。

结构三：案例背景 → 决策难点 → 解决动作 → 结果 → 方法提炼
  · 适合：案例拆解、复盘型内容、老板出镜分享。

结构四：老板故事 → 吃苦/踩坑 → 结果证据 → 托付逻辑
  · 适合：创始人、老板IP、人设建立。

结构五：口碑背书 → 反套路对比 → 合同/流程透明 → 行动建议
  · 适合：转化辅助型内容。

【ToC / 个人IP 四大结构骨架】

C1. 低起点 → 受挫 → 小胜利 → 大坠落 → 翻盘 → 口头禅收口
  · 最稳定的情绪结构，适合人设故事。

C2. 观点 → 错误示范 → 类比解释 → 正确路径
  · 把抽象方法讲透的高可复制结构。

C3. 三步/路线图模板
  · 把复杂事情变成"我今天就能做"的事，降低行动门槛。

C4. 榜单/分点 + 升维收口
  · 中段用分点满足信息获取，尾段升到更大命题。

【白皮书确认的7种内容类型结构】

1. 干货型（信息差型）：一句话吸引 + 分析 + 具体方法 + 总结/钩子
2. 干货型（解决问题型）：一句话痛点 + 具体场景/方法 + 解决方案 + 钩子
3. 故事型：背景 + 行动 + 取得 + 转机 + 胜利 + 总结感悟
4. 观点型：一句话说明观点 + 观点支撑点 + 金句总结
5. 访谈型：精彩片段 + 嘉宾介绍 + 成长故事/项目介绍 + 金句总结
6. 用户证言型：身份与场景 + 使用前问题 + 使用后收获 + 信心变化 + 与他人的不同
7. 转化型（AIDA/PAS/FAB压缩）：用户痛点 + 解决方案 + 钩子

【抖音爆款结构变体】
  · 观点 + 故事 + 总结
  · 提问 + 分析论据 + 总结
  · 奇遇 + 故事 + 反转（升华）
  · 前面当钩子 + 后面引方法 + 引导互动
  · 价值 + 如何获得 + 案例 + 钩子

【正文推进核心规则】
  · 每一段都必须承担具体功能：判断、代价、案例、机制、边界、动作。
  · 不要连续三段都在讲观点。
  · ToB优先使用"案例—机制—标准—边界"推进，而非"情绪—共鸣—金句—鸡汤"。
  · 每8-12秒至少一个新信息点、情绪点、反差点或转折点。
  · 优先使用"场景—情绪—卡点—判断—变化"推进顺序。

───────────────────────────────────────────────────────────────
六、金句与高记忆点表达
───────────────────────────────────────────────────────────────

【金句的核心作用】
  · 不是制造热闹，而是做观点压缩、价值收束、身份表达与信任增强。
  · 解决"观点如何被记住"，而非"观点如何被完整解释"。
  · 替用户下判断，而非描述事实。

【五种高频金句写法】

B1. 观点压缩金句
  · 句式：真正决定___的，不是___，而是___。 / 你缺的从来不是___，你缺的是___。
  · 步骤：先把长论证写出来 → 抽出最底层的一组关系 → 压成一句话。

B2. 对立句式金句
  · 句式：不是___，是___。 / 要么___，要么___。 / 看起来___，其实___。
  · 注意：对立不要过于极端到失真。

B3. 使命/身份宣言金句
  · 句式：我生来是要___的。 / 与其___，不如___。 / 我想成为一个___的人。
  · 必须从真实经历里长出来，不能凭空装腔。

B4. 信任型商业金句
  · 句式：客户买的不是___，买的是___。 / 做的不是___，做的是___。
  · 不能完全脱离业务本身，否则像煽情。

B5. 结论先行 + 可复述金句
  · 句式：我先告诉你答案___。 / 这件事真正关键的只有一个___。
  · 短视频先吃结论，后吃证明。

【金句设置位置】
  · 开头：制造第一印象。
  · 核心观点结尾：强化判断。
  · 最终升华处：完成收口。

七、正文推进与层层展开技巧
───────────────────────────────────────────────────────────────

【模块化推进（让正文不平）】
  · 把一篇口播拆成几个功能位，而不是一口气讲完。
  · 每2-4句切一个功能位；每段只承担一个任务。
  · 功能位示例：开场钩子 → 人物名片/问题定义 → 证据/案例 → 方法/方案 → 收口互动。

【用类比把抽象事讲懂】
  · 找一个用户人人懂的场景 → 对应映射核心关系 → 再回到原问题。
  · 句式：这件事就像___。 / 你可以把___理解成___。

【案例/细节/行业黑话增强真实感】
  · 每讲一个观点，就补一个具体场景/术语/动作。
  · 术语出现后最好顺手解释一下。
  · 把做过的流程拆成3个真实动作写进去。

【先给获得感，再给方法】
  · 开头给收益 → 中段给方法 → 结尾再提醒可带走什么。
  · 句式：我会把___都交给你。 / 这条会直接帮你解决___。

【从方法走到世界观/价值观】
  · 不要停在步骤，把方法背后的更大判断说出来。
  · 让账号从"会教"升级成"有立场"。

───────────────────────────────────────────────────────────────
八、情绪调动与冲突制造
───────────────────────────────────────────────────────────────

【爆点设计（知识分享类四法）】
  · 对比法：学习前后的差异、好坏差异。
  · 段子法：把段子/梗点与知识点结合。
  · 抱大腿法：借热门歌曲、明星、童话人物做爆点。
  · 新奇法：用非常规方式处理常见物品。

【爆点设计（剧情类结尾四法）】
  · 互动法：留悬念或讨论空间，引爆评论区。
  · 共鸣法：让用户在结尾产生同样感觉，想点赞/评论/转发。
  · 反转法：给出和预期不同的结果。
  · 强化法：重复一个动作或一句话，成为记忆点。

【情绪推进路径】
  被钩住 → 被验证 → 被说服 → 被鼓动行动

【情绪使用规则】
  · ToB：情绪只能作为辅助，服务判断、责任、代价、转折，不可替代业务逻辑。
  · ToC：直击用户痛点、提供解决方案、引发共同经历。
  · 讲故事时，情绪状态 > 内容本身。

───────────────────────────────────────────────────────────────
九、信任建立与说服体系
───────────────────────────────────────────────────────────────

【ToB 信任五证体系】

1. 结果证明：项目结果、增长前后、成本变化、工期优化、成交结果、复购事实。
   · 结果必须真实，且尽量给上下文。

2. 案例证明：补背景、难点、动作、过程、变化与适用边界。
   · 案例不是只报结果，不要写成宣传喜报。

3. 机制证明：流程、标准、表单、审核机制、沟通机制、复盘机制、验收机制、风控机制。

4. 流程透明：把合作前、合作中、合作后各自怎么做写清。

5. 风险控制：主动说清哪些风险会提前规避、哪些需要客户配合、哪些边界不会越过。

【ToB 故事写法规则】
  · 老板创业故事：起点、关键决策、吃过的亏、如何扛、最终形成的判断与标准。
  · 创始人踩坑故事：为什么错、代价是什么、后来怎么改、现在如何避免。
  · 客户案例：背景、难点、动作、结果、复盘与适用边界。
  · 项目复盘：项目为何难、关键节点发生了什么、怎么判断、怎么调整。
  · 所有故事都应带"代价感""决策感""结果感""方法感"至少两项。

【ToC 信任建立方法】
  · 量化结果、具体流程、行业黑话、真实代价、反套路机制、顾客为什么买单。
  · 用户证言结构：身份场景 + 使用前问题 + 使用后收获 + 信心变化 + 与他人的不同。

───────────────────────────────────────────────────────────────
十、转化收尾与行动引导（CTA）
───────────────────────────────────────────────────────────────

【CTA 六种类型及适用场景】

1. 评论互动型
   · 适用：破圈和轻信任内容。
   · 句式："你更担心哪一块？""你们公司现在卡在哪一步？"

2. 资料包承接型
   · 适用：咨询前教育期。
   · 句式："需要我整理的清单/模板/避坑表，可以留言或私信。"

3. 企微/微信承接型
   · 适用：已有明确兴趣的人群，前提是内容已完成一定信任建立。
   · 应写清适合谁来找、带着什么问题来聊。

4. 咨询预约型
   · 适用：转化内容与成交辅助期。
   · 重点不是催促，而是明确下一步："如果你现在正处在___阶段，我们可以先聊清楚你该不该做。"

5. 案例深聊型
   · 适用：案例复盘与项目介绍内容。
   · 引导："如果你更想看你这个行业的类似做法，我可以继续拆。"

6. 下一条内容承接型
   · 适用：系列化信任建立。
   · 作用：把复杂决策链拆分成多条内容，降低单条压力。

【CTA 通用规则】
  · ToB最佳状态不是"催单"，而是"帮客户更清楚地判断下一步值不值得做"。
  · 冷流量与破圈内容宜轻CTA；转化辅助可强CTA，但要配合边界说明。
  · 轻转化收尾：引导关注、私信、收藏或下一条承接。
  · 判断收尾：用一句可截图判断收住，提升记忆点。
  · 互动收尾：抛问题，但不能每条都强行"评论区告诉我"。

【抖音ToC CTA 写法】
  · 求赞：直接请求用户点赞。
  · 提问：把问题留给评论区。
  · 下期预告：让用户因为想看后续而关注。
  · 价值承诺：直接告诉用户关注后能得到什么。

───────────────────────────────────────────────────────────────
十一、标题技巧（8法）
───────────────────────────────────────────────────────────────

1. 疑问法：用疑问词加问号，叠加反常识更强。
2. 数字法：降低理解成本，放大震惊感（"99%的人都没看出来"）。
3. 热词法：借热点、明星、品牌、流量词提高进入率。
4. 俗语法：借熟悉句式与押韵提升好感。
5. 对话法：前半句描述问题，后半句给态度/回答。
6. 电影台词法：借经典台词自带记忆点。
7. 好奇法：把话说一半，留悬念；知识类可叠加数字法。
8. 对比法：把冲突和反差直接写进标题。

【标题核心规则】
  · 标题越情绪化越容易火。
  · 标题/封面/开头三件套要协同：标题负责点击预期，开头负责兑现预期。
  · 标题与内容形成对比，是提升作品质量的好方法。

───────────────────────────────────────────────────────────────
十二、拍摄与出镜表达
───────────────────────────────────────────────────────────────

【口播表达核心】
  · 不要演：重点不是表演式感染力，而是判断感、责任感、稳定感与真实感。
  · 语速不必过快，但要有信息密度；停顿要服务判断而不是戏剧化。
  · 语气要稳，避免夸张情绪摆动。
  · 先判断，再解释，再给例子；不要从背景铺垫起。

【语言风格】
  · 口语化，短句为主，每句不超过15字。
  · 多用"你"字，增强代入感。
  · 适当用数字、对比、反问增加节奏感。
  · 短句多、停顿自然、适合镜头前口播。

【场景化拍摄】
  · 会议室：适合判断型与方案型。
  · 工厂/现场：适合流程与交付型。
  · 办公室：适合顾问型与复盘型。
  · 项目点位：适合案例型与风险控制型。

【拍摄技术要点】
  · 手机与人距离：1米（一臂半）。
  · 背景与人距离：为人与手机距离的2倍。
  · 固定机位：眉眼处于上三分线，桌子/手部动作处于下三分线。
  · 避免大白墙；推荐绿植、氛围灯。
  · 跟拍/走拍：45°侧拍，避免正面压迫感。
  · 口播松弛感：机位放低至与胃齐平，手机向后倾斜15-20度。

【口播表现力训练】
  · 切片口播：记不住台词时，一个场景只讲一句话。
  · 连推法：将文案拆分成多段，分别拍摄后再剪辑。
  · 每天对着镜头输出1分钟，看镜头，用表情、语气、手势、情绪表达。
  · 多看有镜头表现力的节目，学习抖包袱的节奏。

───────────────────────────────────────────────────────────────
十三、风险边界与黑名单（硬边界）
───────────────────────────────────────────────────────────────

【ToB 硬性禁止】
  · 不得硬写结果：没有结果证据时，不得写绝对化结果、确定性收益、必然成功。
  · 不得伪造案例：不能凭空制造客户案例、项目细节、合作结果、客户评价。
  · 不得为了传播过度夸张：不能把局部经验写成普适真理。
  · 不得把ToB写成泛鸡汤：不能只讲坚持、真诚、努力、成长，而不讲业务、机制、标准、代价与边界。
  · 不得只有情绪没有业务。
  · 不得只有专业黑话没有用户翻译。
  · 不得把高传播内容误当高转化内容。
  · 不得把"创始人IP"写成"老板吹牛录"。
  · 不得越权改路：出现ToC化倾向时只能提示风险，不得自行转ToC写法。

【通用禁用词与表述】
  · 禁用词："赋能""抓手""闭环""首先其次最后"。
  · 禁止承诺："月入十万""躺赚""无脑操作"。
  · 禁止：空话、套话、鸡汤、专业术语堆砌。
  · 输出时不加任何多余解释。

【反面清单（不建议使用的写法）】
  · 只有大词、没有场景。
  · 只有人设标签、没有经历与证据。
  · 只有反常识，没有后续证明。
  · 只有情绪，没有出口。
  · 只有步骤，没有结果承诺。
  · 只有结果，没有过程代价。
  · 一味追求漂亮句子，忽略口语节奏。
  · 套模板不套纹理：结构像了，但没有你的口头禅、行业细节、个人判断。

【信息不足时的降级规则】
  · 缺痛点：降级为轻共鸣稿，不写强转化。
  · 缺真实经历：不写完整反转故事，可写判断型或状态型文案。
  · 缺结果证明：不写绝对化变化感，改写成"我目前更认可/更有感的是……"。
  · 缺表达风格：优先使用平衡型口语，不强行模仿用户。
  · 任何高风险出现时：弱承诺、弱案例、强机制、强边界、轻CTA。
  · 关键事实缺失时：宁可少写，也不补写。

───────────────────────────────────────────────────────────────
十四、平台适配规则
───────────────────────────────────────────────────────────────

【视频号 ToB 写法】
  · 优先信任与行业判断。
  · 内容可稍长，判断密度可高，案例与感悟并重。
  · 标题可稳，不宜过度夸张。
  · CTA可偏评论、资料包、私聊。

【公众号深度内容写法】
  · 适合咨询前教育、深度案例拆解、行业方法论、客户教育。
  · 结构更完整：必须写清背景、问题、方法、边界、适用对象与结论。
  · CTA可偏资料下载、预约沟通、延展阅读。

【抖音 ToB 破圈写法】
  · 适合行业误区、反常识判断、反差显真、决策代价、短案例。
  · 开头必须强，但正文不能丢业务。
  · 建议时长相对短，信息点切得更快。
  · CTA宜轻，先信任再承接。

【小红书 ToB 写法】
  · 适合案例展示、过程展示、标准清单、误区清单、方案前后对比。
  · 重可读性、图文结构、清单与步骤。
  · 标题比视频号更可包装，但不宜过度标题党。

【抖音 ToC 个人IP 写法】
  · 前3秒给冲突/反差/结果/强代入。
  · 每8-12秒至少一个新信息点。
  · 黄金3秒 + 2-5个爆点 + 白金结尾。
  · 轻转化，先建立熟悉感与信任。

【平台错配风险】
  · 强转化内容直接发抖音冷流量往往效果差。
  · 深度逻辑内容硬压成抖音短视频会损失判断。
  · 过于书面化的公众号稿直接搬视频号易显沉闷。
  · 过于轻飘的抖音话术搬公众号会显空。

───────────────────────────────────────────────────────────────
十五、内容审核快速清单
───────────────────────────────────────────────────────────────

【发布前必查】
  □ 主路线是否明确锁定（ToB / ToC）？
  □ 当前稿件目标层级是否清晰：破圈 / 信任 / 转化 / 口碑 / 咨询前教育？
  □ 是否明确主要对象是谁：观看者、转发者还是拍板者？
  □ 开头是否属于高适配钩子，而非化强共鸣句式？
  □ 正文是否包含至少一种证明：案例、结果、机制、流程、客户视角、交付细节？
  □ 是否能看出客户为什么会信你、为什么会托付？
  □ 平台是否匹配？
  □ CTA是否与业务阶段匹配，是否存在过早强转化？
  □ 是否存在无证据强结果、伪案例、鸡汤化、黑话堆砌、老板吹牛录等风险？
  □ 若资料不足，是否已明确降级，而不是硬写？

【ToB 专属检查】
  □ 是否体现行业判断而不是泛道理？
  □ 是否能看出客户场景、决策难点、风险点？
  □ 是否把"靠谱"拆成机制、案例、流程、结果、托付理由？
  □ 是否区分了高传播切口与高转化切口？
  □ 是否避免只有情绪没有业务？

【ToC 专属检查】
  □ 是否像真人、像本人、像平台（抖音），而不是像"会写稿的模板机"？
  □ 是否踩中"痛点"或"爽点"？
  □ 是否在3秒内抓住注意力？
  □ 内容是否有"传播钩子"？
  □ 是否有人设锚点：经历、判断、情绪、变化至少占两项？

───────────────────────────────────────────────────────────────
十六、高频有效句式库（可直接调用）
───────────────────────────────────────────────────────────────

【高频有效句式】
  · 你以为___，但其实___。
  · 真正决定___的，不是___，而是___。
  · 大部分人做不好___的核心原因，就是___。
  · 很多人都在___，但真正重要的是___。
  · 你缺的从来不是___，你缺的是___。

【高频有效开头表达】
  · 看完这条，你会变成一个___的人。
  · 1分钟之后，你就能___。
  · 如果此生注定___，你会___吗？
  · 别人以为我___，但真实的我___。
  · 从___到___，中间我踩过最深的坑是___。

【高频有效转折表达】
  · 但问题恰恰出在这里。
  · 真正难的不是第一步，而是___。
  · 说到这里，你可能会觉得___，但___。
  · 打个比方，这件事就像___。
  · 如果只做到这里，还是不够。

【高频有效结尾表达】
  · 说到底，这件事拼的不是___，而是___。
  · 所以我越来越确定一个判断：___。
  · 客户最后买单的，从来不是___，而是___。
  · 如果你也在经历___，希望这句话能给你一点底气。
  · 如果你想让我把下一步也拆给你，评论区留___。

【高频有效金句句型】
  · 不是___，是___。
  · 与其___，不如___。
  · 要么___，要么___。
  · 做的不是___，做的是___。
  · 人生不是___，是___。

───────────────────────────────────────────────────────────────
十七、用户旅程拆解法与分阶段内容
───────────────────────────────────────────────────────────────

【客户五阶段内容覆盖】
  认知阶段：他甚至不知道自己有这个问题。
    → 内容方向：行业痛点、管理误区、未来趋势。

  兴趣阶段：他知道有问题，开始寻找解决方案。
    → 内容方向：解决方案、方法论、工具对比。

  考虑阶段：他在对比不同的解决方案。
    → 内容方向：产品功能、客户案例、实施方法。

  决策阶段：他准备选择一个供应商。
    → 内容方向：服务承诺、价格策略、行业背书。

  复购阶段：他成为老客户，可能会介绍新客户。
    → 内容方向：客户成功案例、最佳实践、产品更新。

───────────────────────────────────────────────────────────────
十八、AI 辅助写作工作流提示
───────────────────────────────────────────────────────────────

【AI使用原则】
  · 别让AI替你写（那会失去个人特色），而是让AI替你写"框架"和"润色"。
  · 生成拍摄大纲后，替换成你的人设语气，加停顿/重音。
  · 把写好的文案扔给AI进行"口语化转换"或"金句提炼"。

【选题策划】
  · 给AI发指令："我是XX垂类博主，生成30个可拍选题，分教程/测评/避坑/干货，带黄金3秒钩子。"

【脚本文案】
  · 选题 → AI生成30/60秒口播脚本（黄金3秒+分点+结尾引导）。
  · 用AI拆解爆款视频文案框架：他为什么能火，钩子/话术/情绪/痛点在哪里。
  · 按照"身份、行业、语气、习惯"框架写。

【内容比例参考（黄金三角）】
  · 70% 价值干货
  · 20% 案例背书
  · 10% 品牌/个人IP
