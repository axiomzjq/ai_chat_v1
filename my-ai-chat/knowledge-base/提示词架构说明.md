# 提示词架构说明

> 五页面独立提示词架构，每个页面使用独立的 System Prompt。
> 最后更新：2026-07-09

---

## 1. 提示词总览

| 页面 | Prompt 变量 | 用途 | 模式 |
|------|------------|------|------|
| **访谈** | `INTERVIEW_SYSTEM_PROMPT` | 深度访谈对话 | 对话 |
| **定位** | `POSITIONING_SYSTEM_PROMPT` | IP 定位分析 | 对话 + 生成报告 |
| **选题** | `TOPIC_SYSTEM_PROMPT` | 阶段选题生成 | AI 生成（JSON 输出） |
| **文案** | `COPYWRITING_GENERATE_SYSTEM_PROMPT` | 脚本生成 | 生成（JSON 输出） |
| **信息** | `INFO_SYSTEM_PROMPT` | 企业/行业分析 | 生成报告（嵌入访谈流程） |

---

## 2. 各页面 Prompt 调用方式

### 2.1 访谈页面

```typescript
system: INTERVIEW_SYSTEM_PROMPT + (knowledgeContext ? `\n\n请参考以下专业语料...` : "")
messages: chatMsgs // 多轮对话历史
```

**调用函数：** `deepseek.chat()`（支持多轮对话）
**触发时机：** 用户发送消息时

---

### 2.2 定位页面

```typescript
// 生成定位报告
system: POSITIONING_SYSTEM_PROMPT + (knowledgeContext ? `\n\n请参考以下专业语料...` : "")
prompt: `请根据以下信息生成定位报告：\n【访谈报告】...`
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击"生成定位报告"按钮

---

### 2.3 选题页面

```typescript
// 生成选题池
system: TOPIC_SYSTEM_PROMPT
prompt: buildTopicPrompt(interviewReport, positioningReport)
// → 包含【访谈报告】、【定位报告】、【参考语料】
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击"生成选题池"按钮
**输出格式：** 强制纯 JSON

**解析流程：**
```
AI 返回文本
  → parseTopicPool() 解析
    → 策略1: 直接 JSON.parse
    → 策略2: 提取 Markdown 代码块
    → 策略3: 正则提取 { ... }
    → 策略4: 降级到 Demo 数据
```

---

### 2.4 文案页面

**流程：** 用户在选题页面点击某张选题卡片 → 自动调用 `generateCopywriting(topic)` → 跳转到文案页面 → AI 直接输出结果。

```typescript
// 选题卡片点击事件
onClick={() => {
  setSelectedTopic(topic);
  setCurrentStep('copywriting');
  generateCopywriting(topic);  // 直接生成，无需对话
}}
```

```typescript
// 文案生成
system: COPYWRITING_GENERATE_SYSTEM_PROMPT + copywritingRefContent
prompt: `基于以下信息创作3个标题和1篇口播文案：
【选题详情】...（含标题、爆款类型、3秒钩子、平台等）
【定位方案】...
【个人背景】...
【企业背景】...
【上传资料内容】...`
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击选题卡片时自动触发
**输出格式：** 强制纯 JSON（`{ titles: [], content: '' }`）
**max_tokens：** 16000（文案输出量大）

**解析流程：**
```
AI 返回文本
  → parseJsonResponse() 解析
    → 策略1: 直接 JSON.parse
    → 策略2: 提取 Markdown 代码块
    → 策略3: 正则提取 { ... }
    → 策略3b: repairTruncatedJson 补全括号
```

## 3. Prompt 文件结构

```
src/lib/prompts.ts
├── INTERVIEW_SYSTEM_PROMPT      // 访谈对话
├── INFO_SYSTEM_PROMPT           // 企业/信息分析
├── POSITIONING_SYSTEM_PROMPT    // 定位对话+报告
├── COPYWRITING_GENERATE_SYSTEM_PROMPT  // 文案生成（3标题+1脚本，JSON）
└── TOPIC_SYSTEM_PROMPT          // 选题生成（JSON）
```

> **注意：** `COPYWRITING_SYSTEM_PROMPT`（对话模式）已移除，文案页面使用单一 `COPYWRITING_GENERATE_SYSTEM_PROMPT`。

---

## 4. 关键设计原则

### 4.1 一个页面一个 Prompt

每个页面有独立的 System Prompt。文案页面使用单一 `COPYWRITING_GENERATE_SYSTEM_PROMPT`，选中选题后直接生成文案，无需对话交互。

### 4.2 JSON 输出强制规则

需要结构化输出的 Prompt（选题、文案生成）必须包含：
```
## 绝对强制规则
你只能输出纯 JSON，不能输出任何其他内容。
不要输出 markdown 代码块标记。
不要输出注释。
```

### 4.3 字段名统一

所有 Prompt 中的 JSON 字段名必须与前端 TypeScript 接口一致：
- 使用 **驼峰命名**（`hook3s` 而不是 `hook_3s`）
- 前端 `Topic` 接口与 Prompt JSON Schema 必须对齐

### 4.4 上下文注入

所有 Prompt 的 User Message 必须包含：
- 前序阶段报告（访谈报告 → 定位报告 → 选题池）
- 知识库语料（`knowledgeBase` 中对应分类的内容）
- 用户当前明确要求

## 5. 解析函数

### parseJsonResponse()

**位置：** `src/App.tsx`
**用途：** 通用 JSON 解析器，处理 AI 返回的各种格式
**策略：**
1. 直接 `JSON.parse()`
2. 提取 Markdown 代码块（` ```json ... ``` `）
3. 正则提取第一个 `{` 到最后一个 `}`
4. `repairTruncatedJson` 补全缺失的 `]` 和 `}`

**调用方：** `parseTopicPool()`（选题）、`generateCopywriting()`（文案）

### parseTopicPool()

**位置：** `src/App.tsx`（选题页面逻辑中）
**用途：** 从 AI 返回的文本中提取并解析选题 JSON，调用 `parseJsonResponse()` 后进行归一化

**归一化规则：**
- 格式 A：`{ "stages": [...] }` → 直接返回
- 格式 B：`[...]`（AI 直接返回数组）→ 包装为 `{ stages: [...] }`
- 格式 C：其他数组字段 → 尝试匹配 `stage` 字段

---

## 6. 降级策略

| 场景 | 处理方式 |
|------|---------|
| AI 返回格式错误 | `parseJsonResponse` 逐层尝试，全部失败返回 null |
| AI 网络超时 | catch 捕获，alert 提示用户重试 |
| JSON 被截断 | `repairTruncatedJson` 自动补全缺失括号 |
| 选题解析失败 | `topicGenerationStatus: 'demo_fallback'`，显示 Demo 数据 |
| 文案解析失败 | alert 提示，不保存降级数据 |

**选题降级标识：** `topicGenerationStatus: 'demo_fallback'`
**用户可见：** 按钮文案变为"重新生成"
