# 提示词架构说明

> 四页面独立提示词架构，每个页面使用独立的 System Prompt。
> 最后更新：2026-06-28

---

## 1. 提示词总览

| 页面 | Prompt 变量 | 用途 | 模式 |
|------|------------|------|------|
| **访谈** | `INTERVIEW_SYSTEM_PROMPT` | 深度访谈对话 | 对话 |
| **定位** | `POSITIONING_SYSTEM_PROMPT` | IP 定位分析 | 对话 + 生成报告 |
| **选题** | `TOPIC_SYSTEM_PROMPT` | 阶段选题生成 | AI 生成（JSON 输出） |
| **文案** | `COPYWRITING_SYSTEM_PROMPT` | 思路整理对话 | 对话 |
| **文案** | `COPYWRITING_GENERATE_SYSTEM_PROMPT` | 脚本生成 | 生成（JSON 输出） |
| **信息** | `INFO_SYSTEM_PROMPT` | 企业/行业分析 | 生成报告（嵌入访谈流程） |

---

## 2. 各页面 Prompt 调用方式

### 2.1 访谈页面

```typescript
system: INTERVIEW_SYSTEM_PROMPT + (knowledgeContext ? `\n\n请参考以下专业语料...` : "")
messages: chatMsgs // 多轮对话历史
```

**调用函数：** `deepseek.chat()`（支持多轮对话）
**触发时机：** 用户发送消息时

---

### 2.2 定位页面

```typescript
// 生成定位报告
system: POSITIONING_SYSTEM_PROMPT + (knowledgeContext ? `\n\n请参考以下专业语料...` : "")
prompt: `请根据以下信息生成定位报告：\n【访谈报告】...`
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击"生成定位报告"按钮

---

### 2.3 选题页面

```typescript
// 生成选题池
system: TOPIC_SYSTEM_PROMPT
prompt: buildTopicPrompt(interviewReport, positioningReport)
// → 包含【访谈报告】、【定位报告】、【参考语料】
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击"生成选题池"按钮
**输出格式：** 强制纯 JSON

**解析流程：**
```
AI 返回文本
  → parseTopicPool() 解析
    → 策略1: 直接 JSON.parse
    → 策略2: 提取 Markdown 代码块
    → 策略3: 正则提取 { ... }
    → 策略4: 降级到 Demo 数据
```

---

### 2.4 文案页面

#### 2.4.1 对话模式（思路整理）

```typescript
system: COPYWRITING_SYSTEM_PROMPT + "\n\n当前处于【思路整理阶段】..."
messages: chatMsgs // 多轮对话历史
```

**调用函数：** `deepseek.chat()`（支持多轮对话）
**触发时机：** 用户发送消息时
**输出格式：** 自然语言对话

#### 2.4.2 生成模式（脚本生成）

```typescript
system: COPYWRITING_GENERATE_SYSTEM_PROMPT + "\n\n请结合..."
prompt: `基于以下信息创作3个标题和1篇口播文案：\n【选题/主题】...\n【定位方案】...`
```

**调用函数：** `deepseek.generateText()`（单次生成）
**触发时机：** 用户点击"生成文案"按钮
**输出格式：** 强制纯 JSON（`{ titles: [], content: '' }`）

## 3. Prompt 文件结构

```
src/lib/prompts.ts
├── INTERVIEW_SYSTEM_PROMPT      // 访谈对话
├── INFO_SYSTEM_PROMPT           // 企业/信息分析
├── POSITIONING_SYSTEM_PROMPT    // 定位对话+报告
├── COPYWRITING_SYSTEM_PROMPT    // 文案对话（思路整理）
├── COPYWRITING_GENERATE_SYSTEM_PROMPT  // 文案生成（脚本+JSON）
└── TOPIC_SYSTEM_PROMPT          // 选题生成（JSON）
```

---

## 4. 关键设计原则

### 4.1 一个页面一个 Prompt

每个页面有独立的 System Prompt，不共用。文案页面拆成两个 Prompt：
- 对话模式：`COPYWRITING_SYSTEM_PROMPT`（引导式对话）
- 生成模式：`COPYWRITING_GENERATE_SYSTEM_PROMPT`（结构化输出）

### 4.2 JSON 输出强制规则

需要结构化输出的 Prompt（选题、文案生成）必须包含：
```
## 绝对强制规则
你只能输出纯 JSON，不能输出任何其他内容。
不要输出 markdown 代码块标记。
不要输出注释。
```

### 4.3 字段名统一

所有 Prompt 中的 JSON 字段名必须与前端 TypeScript 接口一致：
- 使用 **驼峰命名**（`hook3s` 而不是 `hook_3s`）
- 前端 `Topic` 接口与 Prompt JSON Schema 必须对齐

### 4.4 上下文注入

所有 Prompt 的 User Message 必须包含：
- 前序阶段报告（访谈报告 → 定位报告 → 选题池）
- 知识库语料（`knowledgeBase` 中对应分类的内容）
- 用户当前明确要求

## 5. 解析函数

### parseTopicPool()

**位置：** `src/App.tsx`（选题页面逻辑中）
**用途：** 从 AI 返回的文本中提取并解析选题 JSON
**策略：**
1. 直接 `JSON.parse()`
2. 提取 Markdown 代码块
3. 正则提取 `{...}`
4. 降级到 Demo 数据

---

## 6. 降级策略

| 场景 | 处理方式 |
|------|---------|
| AI 返回格式错误 | `parseTopicPool` 逐层尝试，最终降级 |
| AI 网络超时 | catch 捕获，降级到 Demo 数据 |
| 解析后数据为空 | 检查 `stages.length > 0`，否则降级 |
| Demo 数据 | `getDemoTopicPool()` 返回硬编码数据 |

**降级标识：** `topicGenerationStatus: 'demo_fallback'`
**用户可见：** 按钮文案变为"重新生成"
