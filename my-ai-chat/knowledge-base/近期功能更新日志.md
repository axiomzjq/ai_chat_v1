## 1. 提示词架构重构（2026-07-05）

### 变更内容

将原来的"一份大提示词"改为"基础提示词 + 参考文件"分离架构：

```
src/lib/prompts.ts          ← 6 个基础提示词（角色定义 + 职责说明）
refs/                        ← 7 个参考 .txt 文件（详细方法论）
public/refs/                 ← 浏览器可访问的副本（fetch 加载）
```

### 基础提示词（prompts.ts）

| Prompt | 用途 | 输出格式 |
|--------|------|---------|
| `INTERVIEW_SYSTEM_PROMPT` | 访谈对话 | 自然语言 |
| `INFO_SYSTEM_PROMPT` | 企业/信息分析 | 自然语言 |
| `POSITIONING_SYSTEM_PROMPT` | 定位对话+报告 | 自然语言 |
| `COPYWRITING_SYSTEM_PROMPT` | 文案思路整理 | 自然语言（对话） |
| `COPYWRITING_GENERATE_SYSTEM_PROMPT` | 文案脚本生成 | 强制 JSON |
| `TOPIC_SYSTEM_PROMPT` | 选题生成 | 强制 JSON |

### 参考文件（refs/）

| 步骤 | 参考文件 | 说明 |
|------|---------|------|
| 访谈 | `refs/interview/客户访谈参考手册.txt` | 访谈方法论 |
| 定位 | `refs/positioning/客户定位访谈参考手册.txt` | 定位判断方法 |
| 定位 | `refs/positioning/写作技巧提示词_精华版.txt` | 写作技巧（与选题共用） |
| 选题 | `refs/topic/选题提示词_基于访谈结果.txt` | 选题方法论 |
| 选题 | `refs/topic/写作技巧提示词_精华版.txt` | 写作技巧（与定位共用） |
| 文案 | `refs/copywriting/客户采访与选题提示词.txt` | 采访框架 |
| 文案 | `refs/copywriting/文案审核提示词.txt` | 审核标准 |

### 上下文注入方式

```typescript
// 1. 加载参考文件（从 public/refs/ fetch，带缓存）
const refContent = await loadStepRefs('interview');

// 2. 注入到 AI 请求
system: BASE_PROMPT + (refContent ? `

【参考文件 · xxx】：
${refContent}` : '')
```

### 关键特性

- **参考文件缓存**：`refsCache` 只加载一次，后续直接读取缓存
- **按步骤加载**：每个步骤只加载自己需要的参考文件
- **基础提示词精简**：只定义角色和职责，详细方法论从 txt 文件加载

- `knowledge-base/提示词文件架构.md` — 完整文件架构

## 2. 多用户数据隔离修复（2026-07-05）

### 问题

同一台浏览器上，不同账号登录会共享 localStorage 数据，导致管理员测试的数据被普通用户看到。

### 根因

所有 localStorage key 都是全局的，没有按用户 ID 做区分：

```typescript
// ❌ 修复前 - 共享
localStorage.getItem('founder_ip_history')
localStorage.getItem('result_interview_report')

// ✅ 修复后 - 按用户隔离
localStorage.getItem(`founder_ip_history_${userId}`)
localStorage.getItem(`result_interview_report_${userId}`)
```

### 修复方案

1. **新增 `getUserScopedKey()`** — 生成用户隔离的 key
2. **`saveResultToStorage` / `loadResultFromStorage`** — 新增 `userId` 参数
3. **`loadUIState()`** — 新增 `userId` 参数
4. **`loadUserLocalData()`** — 登录后加载该用户的所有本地数据
5. **`handleLogout`** — 登出时清除该用户的全部 localStorage 数据

### 隔离的 localStorage key

| 修复前（共享） | 修复后（按用户隔离） |
|--------------|-------------------|
| `founder_ip_history` | `founder_ip_history_{userId}` |
| `result_interview_report` | `result_interview_report_{userId}` |
| `result_positioning_report` | `result_positioning_report_{userId}` |
| `result_info_report` | `result_info_report_{userId}` |
| `result_topic_pool` | `result_topic_pool_{userId}` |
| `ai_chat_ui_state` | `ai_chat_ui_state_{userId}` |

### 数据存储说明

| 存储位置 | 数据 | 按用户隔离 | 状态 |
|---------|------|-----------|------|
| **后端数据库** | 对话消息、访谈结果、定位报告、选题池、文案 | ✅ 按 `user_id` 隔离 | 正常 |
| **前端 localStorage** | 同上 + UI 状态 | ✅ 按 `userId` 隔离 | **已修复** |

## 3. 步骤解锁逻辑修复（2026-07-05）

### 问题

`isStepUnlocked` 调用时传入了空数组 `[]` 而非 `state.topicPool`，导致文案步骤的锁检查 `!!topicPool.length` 永远为 `false`。

### 影响

用户生成选题池后，导航栏和侧边栏的"文案"标签一直显示为锁定状态，与实际情况不一致。

### 修复

所有 4 处 `isStepUnlocked` 调用都改为传入 `state.topicPool`：

| 位置 | 修复前 | 修复后 |
|------|--------|--------|
| 主导航（行 334） | `[]` | `state.topicPool` |
| 访谈页"下一步"按钮（行 2325） | `[]` | `state.topicPool` |
| 访谈页"下一步"按钮样式（行 2333） | `[]` | `state.topicPool` |
| 侧边导航（行 4217） | `[]` | `state.topicPool` |

## 4. 选题页面空状态提示（2026-07-05）

### 变更

选题页面不再默认显示 Demo 数据，而是根据状态显示不同内容：

| 状态 | 条件 | 显示内容 |
|------|------|---------|
| **空状态** | `topicPool` 为空 + `idle` | 图标 + "尚未生成选题池" + 生成按钮 |
| **加载中** | `generating` | 旋转动画 + "AI 正在生成..." |
| **有数据** | `completed` | 真实 AI 生成的选题内容 |
| **降级** | `demo_fallback` | Demo 数据 + "重新生成"按钮 |

### 关键修改

- AI 降级数据（解析失败/网络错误）不再保存到 localStorage
- 刷新后无数据时显示空状态，不会自动加载 Demo
- `loadUserLocalData` 登录后正确设置 `topicGenerationStatus`

## 5. 定位方案 3 版独立输出（2026-07-05）

### 问题

AI 只生成一版定位方案，没有按要求输出 3 个独立方案。

### 修复

1. **用户 prompt 增加明确的格式要求**：
   ```
   ### 定位方案 1：[方案名称]
   [完整方案内容]

   ### 定位方案 2：[方案名称]
   [完整方案内容]

   ### 定位方案 3：[方案名称]
   [完整方案内容]
   ```

2. **分割正则**从 `/(?=###\s+定位分析|###\s+定位方案)/` 改为 `/(?=###\s+定位方案\s*\d)/`，精确匹配带编号的方案标题。

## 6. 调试模式增强（2026-07-05）

### 新增按钮

在调试模式（`DEBUG_MODE !== 'off'`）下，页面右上角新增 **"清除本地缓存"** 按钮：

| 按钮 | 图标 | 功能 |
|------|------|------|
| 导出调试日志 | Bug | 导出 console 日志 |
| 一键还原 | Trash2 | 清空当前会话所有数据 |
| **清除本地缓存** | Database | 清除当前用户的 localStorage |

### 清除本地缓存功能

- 清除该用户的所有 localStorage 数据
- 清除 auth token 缓存
- **不影响后端数据库**
- 清除后自动刷新页面

### 当前调试模式

```typescript
// src/lib/debug.ts
export const DEBUG_MODE: 'dev' | 'internal' | 'off' = 'internal';
```

## 9. 五步工作流（2026-06-25）

### 变更

从四步工作流（访谈→定位→文案→历史）改为五步工作流（访谈→定位→选题→文案→历史）。

### 步骤解锁逻辑

| 步骤 | Agent | 解锁条件 |
|------|-------|---------|
| **访谈** | 访谈顾问 | 始终可进 |
| **定位** | 定位顾问 | 完成访谈并生成报告 |
| **选题** | 选题顾问 | 完成访谈 |
| **文案** | 文案顾问 | 完成访谈并生成选题池 |
| **历史** | — | 始终可进 |
