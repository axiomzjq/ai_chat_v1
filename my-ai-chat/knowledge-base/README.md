# 知识库目录

存放 RAG 系统使用的原始文档和处理后的数据。

## 目录说明

| 目录 | 用途 | 是否提交 Git |
|------|------|-----------|
| `raw/` | 原始文档（PDF、Markdown、TXT） | ✅ 建议提交小文件 |
| `processed/` | 清洗后的纯文本，按文档分文件存储 | ❌ 生成产物，不提交 |
| `embeddings/` | 向量索引（JSON / SQLite / HNSW 索引） | ❌ 生成产物，不提交 |

## 运维文档

| 文件 | 说明 |
|------|------|
| `服务运维脚本指南.md` | 服务检查、重启脚本使用指南 ⭐ |

## 支持的文档格式

1. **Markdown (`.md`)** — 保留标题结构，用于智能分块
2. **Plain Text (`.txt`)** — 直接按段落/长度分块
3. **JSON (`.json`)** — 结构化数据，按条目分块
4. **PDF (`.pdf`)** — 需解析为文本后处理（待实现）

## 添加文档流程

1. 将文件放入 `raw/` 目录
2. 运行索引构建脚本：
   ```bash
   cd server && npm run build:index
   ```
3. 或在前端管理界面上传，自动触发解析与索引
