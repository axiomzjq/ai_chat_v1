## 1. check-services.sh

### 功能
检查 AI Chat 服务的 5 项核心指标

### 使用方法

```bash
cd /home/admin/work/ai_chat_v1/my-ai-chat
./scripts/check-services.sh
```

### 检查项

| 序号 | 检查项 | 检查方法 | 预期结果 |
|------|--------|---------|---------|
| 1/5 | PostgreSQL 16 | `systemctl is-active postgresql-16` | 运行中 |
| 2/5 | PM2 后端服务 | `pm2 describe ai-chat-backend` | status: online |
| 3/5 | PM2 前端服务 | `pm2 describe ai-chat-frontend` | status: online |
| 4/5 | 端口监听 | `ss -tlnp \| grep ':3001'` 和 `:5173` | 两个端口都在监听 |
| 5/5 | HTTP 健康检查 | `curl http://localhost:3001/health` | 返回 200 |

### 输出示例

```

========================================

[1/5] PostgreSQL 16      ... ✓ 运行中
[2/5] PM2 后端服务      ... ✓ 运行中 (online)
[3/5] PM2 前端服务      ... ✓ 运行中 (online)
[4/5] 端口 3001/5173    ... ✓ 正常监听
[5/5] HTTP 健康检查     ... ✓ 后端+前端均正常

========================================
  检查结果: 全部正常
========================================
```

## 2. restart-all.sh ⭐

### 功能
一键重启前后端 PM2 服务并验证状态

### 使用方法

```bash
cd /home/admin/work/ai_chat_v1/my-ai-chat
./scripts/restart-all.sh
```

### 执行流程

```
1. 检查 PM2 是否安装
   ↓
2. 检查 PM2 进程是否存在
   ↓
3. 重启或启动后端服务
   ↓
4. 重启或启动前端服务
   ↓
5. 等待服务启动（3秒）
   ↓
6. 检查服务状态和端口
   ↓
7. 输出结果和访问地址
```

### 功能特性

- ✅ **智能判断**：进程存在则重启，不存在则启动
- ✅ **自动等待**：重启后等待 3 秒确保服务启动
- ✅ **状态检查**：验证 PM2 状态、端口监听、HTTP 健康
- ✅ **友好提示**：彩色输出，清晰的错误提示
- ✅ **访问地址**：重启成功后自动输出访问地址

### 输出示例

```
========================================
  AI Chat 一键重启服务
========================================

[1/5] 检查 PM2 ... ✓
[2/5] 检查 PM2 进程 ... ✓ (后端: true, 前端: true)

[3/5] 重启服务 ...
  → 重启后端 (ai-chat-backend)
  → 重启前端 (ai-chat-frontend)

[4/5] 等待服务启动 ...

[5/5] 检查服务状态 ...
    ✓ 后端: online
    ✓ 前端: online

========================================
  重启成功！所有服务正常运行

  访问地址:
    前端: http://localhost:5173
    后端: http://localhost:3001
    健康: http://localhost:3001/health
========================================
```

### 跳过检查模式

```bash
./scripts/restart-all.sh --skip-check
```

## 🔄 标准运维流程

### 代码修改后重启

```bash
# 1. 保存代码修改
git add .
git commit -m "feat: 新功能描述"

# 2. 重启服务
cd /home/admin/work/ai_chat_v1/my-ai-chat
./scripts/restart-all.sh

# 3. 验证
./scripts/check-services.sh

# 4. 查看日志（如果失败）
pm2 logs ai-chat-backend
pm2 logs ai-chat-frontend
```

### 首次部署

```bash
# 1. 克隆代码
git clone git@github.com:axiomzjq/ai_chat_v1.git
cd ai_chat_v1

# 2. 安装依赖
npm install
cd server && npm install && cd ..

# 3. 配置环境变量
cp .env.example .env.local
cp server/.env.example server/.env
# 编辑 .env.local 和 server/.env

# 4. 初始化数据库
# 参考 docs/ONBOARDING.md

# 5. 启动服务
./scripts/restart-all.sh
```

### 故障排查

```bash
# 1. 检查服务状态
./scripts/check-services.sh

# 2. 重启服务
./scripts/restart-all.sh

# 3. 查看日志
pm2 logs ai-chat-backend --lines 50
pm2 logs ai-chat-frontend --lines 50

# 4. 检查数据库
sudo systemctl status postgresql-16

# 5. 手动重启特定服务
pm2 restart ai-chat-backend
pm2 restart ai-chat-frontend
```

## 📝 注意事项

### PM2 配置

- **当前运行模式：** `vite preview`（生产环境）
- **开发环境：** 应使用 `npm run dev`（而非 PM2）
- **保存配置：** `pm2 save`（重启服务器后自动恢复）

### 端口占用

| 端口 | 服务 | 说明 |
|------|------|------|
| 3001 | 后端 API | Express 服务 |
| 5173 | 前端页面 | Vite 预览服务器 |

### 环境变量

- `.env.local` - 前端环境变量（Vite 需要 `VITE_` 前缀）
- `server/.env` - 后端环境变量
- **两个文件都不提交 git**（已在 `.gitignore` 中）
