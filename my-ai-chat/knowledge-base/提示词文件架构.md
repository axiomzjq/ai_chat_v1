# 提示词文件架构

> 各步骤提示词与参考文件的结构化说明
> 最后更新：2026-07-05

---

## 1. 目录结构

```
refs/                                          ← 参考文件根目录
├── interview/
│   └── 客户访谈参考手册.txt                     ← 访谈步骤必读
├── positioning/
│   ├── 客户定位访谈参考手册.txt                  ← 定位步骤必读
│   └── 写作技巧提示词_精华版.txt                  ← 定位+选题步骤共用
├── topic/
│   ├── 选题提示词_基于访谈结果.txt                ← 选题步骤必读
│   └── 写作技巧提示词_精华版.txt                  ← 定位+选题步骤共用
├── copywriting/
│   ├── 客户采访与选题提示词.txt                   ← 文案步骤必读
│   └── 文案审核提示词.txt                        ← 文案步骤参考
└── results/                                    ← 各步骤输出结果
    ├── 访谈结果/
    ├── 定位报告/
    └── 选题池/
```

---

## 2. 各步骤文件引用关系

| 步骤 | 基础提示词（prompts.ts） | 参考文件（refs/） | 前置输入 | 输出结果 |
|------|------------------------|------------------|---------|---------|
| **访谈** | `INTERVIEW_SYSTEM_PROMPT` | `interview/客户访谈参考手册.txt` | 无 | 访谈结果 |
| **定位** | `POSITIONING_SYSTEM_PROMPT` | `positioning/客户定位访谈参考手册.txt`<br>`positioning/写作技巧提示词_精华版.txt` | 访谈结果 | 定位报告 |
| **选题** | `TOPIC_SYSTEM_PROMPT` | `topic/选题提示词_基于访谈结果.txt`<br>`topic/写作技巧提示词_精华版.txt` | 访谈结果 + 定位报告 | 选题池（JSON） |
| **文案-对话** | `COPYWRITING_SYSTEM_PROMPT` | `copywriting/客户采访与选题提示词.txt` | 访谈结果 + 定位报告 + 选题池 | 对话 |
| **文案-生成** | `COPYWRITING_GENERATE_SYSTEM_PROMPT` | `copywriting/客户采访与选题提示词.txt`<br>`copywriting/文案审核提示词.txt` | 访谈结果 + 定位报告 + 选题池 + 对话上下文 | 文案（JSON） |

---

## 3. 基础提示词职责

基础提示词（`src/lib/prompts.ts`）只做两件事：
1. **定义角色**：告诉 AI 它是谁、做什么
2. **说明参考文件**：说明需要参考哪些 txt 文件

**不包含**详细方法论内容（那些在 txt 文件中）。

---

## 4. 上下文注入方式

### 4.1 参考文件注入

App.tsx 通过 `loadStepRefs(step)` 加载参考文件内容，注入到 AI 请求中：

```typescript
// 1. 加载参考文件
const refContent = await loadStepRefs('interview');

// 2. 注入到 system prompt
system: BASE_PROMPT + (refContent ? `

【参考文件 · xxx】：
${refContent}` : '')
```

### 4.2 前序结果注入

前一步骤的结果通过 User Message 注入：

```typescript
// 示例：选题页面
prompt: `
【访谈报告】：
${interviewReport}

【定位报告】：
${positioningReport}

请基于以上信息生成选题规划池...`
```

## 5. 数据流转

```
用户输入
  │
  ├──→ 访谈 → 访谈结果 ─────────────────┐
  │                                    │
  ├──→ 定位 → 定位报告 ────────────────┤
  │                                    │
  ├──→ 选题 → 选题池 ─────────────────┤
  │                                    │
  └──→ 文案 → 文案（脚本）             │
                                       │
                    所有结果保存到 localStorage + 可下载
```

---

## 6. 结果文件命名规范

| 步骤 | 结果文件名 | 格式 | localStorage Key |
|------|-----------|------|-----------------|
| 访谈 | 访谈结果 | Markdown | `result_interview_report` |
| 定位 | 定位报告 | Markdown | `result_positioning_report` |
| 选题 | 选题池 | JSON | `result_topic_pool` |
| 文案 | 文案脚本 | JSON | copywritingOutput（state 中） |

---

## 7. 缓存机制

参考文件只加载一次，缓存到 `refsCache`：

```typescript
const refsCache = useRef<Record<string, string>>({});

const getRefContent = async (path: string): Promise<string> => {
  if (refsCache.current[path]) return refsCache.current[path];
  const content = await loadRefFile(path);
  refsCache.current[path] = content;
  return content;
};
```
