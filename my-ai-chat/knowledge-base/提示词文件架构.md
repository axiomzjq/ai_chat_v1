# 提示词文件架构

> 各步骤提示词与参考文件的结构化说明
> 最后更新：2026-07-09

---

## 1. 目录结构

```
public/refs/                                    ← 浏览器可访问的参考文件
├── interview/
│   └── 客户访谈参考手册.txt                     ← 访谈步骤必读
├── positioning/
│   ├── 客户定位访谈参考手册.txt                  ← 定位步骤必读
│   └── 写作技巧提示词_精华版.txt                  ← 定位+选题步骤共用
├── topic/
│   ├── 选题提示词_基于访谈结果.txt                ← 选题步骤必读
│   └── 写作技巧提示词_精华版.txt                  ← 定位+选题步骤共用
├── copywriting/
│   ├── 客户采访与选题提示词.txt                   ← 文案步骤必读
│   └── 文案审核提示词.txt                        ← 文案步骤参考
└── results/                                    ← 各步骤输出结果（预留）
    ├── 访谈结果/
    ├── 定位报告/
    └── 选题池/
```

> **注意：** 参考文件同时存在于项目根目录 `refs/`（源码维护）和 `public/refs/`（浏览器访问）。前端通过 `fetch('/refs/...')` 加载 `public/refs/` 中的文件。

---

## 2. 各步骤文件引用关系

| 步骤 | 基础提示词（prompts.ts） | 参考文件（public/refs/） | 前置输入 | 输出结果 |
|------|------------------------|----------------------|---------|---------|
| **访谈** | `INTERVIEW_SYSTEM_PROMPT` | `interview/客户访谈参考手册.txt` | 无 | 访谈结果 |
| **信息** | `INFO_SYSTEM_PROMPT` | 无（嵌入访谈流程） | 访谈报告 | 企业/行业分析 |
| **定位** | `POSITIONING_SYSTEM_PROMPT` | `positioning/客户定位访谈参考手册.txt`<br>`positioning/写作技巧提示词_精华版.txt` | 访谈结果 | 定位报告 |
| **选题** | `TOPIC_SYSTEM_PROMPT` | `topic/选题提示词_基于访谈结果.txt`<br>`topic/写作技巧提示词_精华版.txt` | 访谈结果 + 定位报告 | 选题池（JSON） |
| **文案** | `COPYWRITING_GENERATE_SYSTEM_PROMPT` | `copywriting/客户采访与选题提示词.txt`<br>`copywriting/文案审核提示词.txt` | 访谈结果 + 定位报告 + 选题池 | 文案（3标题 + 1脚本） |

---

## 3. 基础提示词职责

基础提示词（`src/lib/prompts.ts`）只做两件事：
1. **定义角色**：告诉 AI 它是谁、做什么
2. **说明参考文件**：说明需要参考哪些 txt 文件

**不包含**详细方法论内容（那些在 txt 文件中）。

---

## 4. 上下文注入方式

### 4.1 参考文件注入

App.tsx 通过 `loadStepRefs(step)` 加载参考文件内容，注入到 AI 请求中：

```typescript
// 1. 加载参考文件
const refContent = await loadStepRefs('interview');

// 2. 注入到 system prompt
system: BASE_PROMPT + (refContent ? `

【参考文件 · xxx】：
${refContent}` : '')
```

### 4.2 前序结果注入

前一步骤的结果通过 User Message 注入：

```typescript
// 示例：选题页面
prompt: `
【访谈报告】：
${interviewReport}

【定位报告】：
${positioningReport}

请基于以上信息生成选题规划池...`
```

## 5. 数据流转

```
用户输入
  │
  ├──→ 访谈 → 访谈结果 ─────────────────┐
  │                                    │
  ├──→ 信息 → 企业信息报告 ─────────────┤（嵌入访谈流程）
  │                                    │
  ├──→ 定位 → 定位报告 ────────────────┤
  │                                    │
  ├──→ 选题 → 选题池 ─────────────────┤
  │                                    │
  └──→ 文案 → 文案（脚本）             │
                                       │
                    所有结果保存到 localStorage + 后端数据库
```

> **注意：** "信息"页面已嵌入访谈流程，用户无需手动切换。访谈结束后自动触发信息分析。

## 6. 结果文件命名规范

| 步骤 | 结果文件名 | 格式 | 存储位置 |
|------|-----------|------|---------|
| 访谈 | 访谈结果 | Markdown | localStorage + `user_profiles.interview_data` |
| 信息 | 企业信息报告 | Markdown | localStorage + `user_profiles.information_report` |
| 定位 | 定位报告 | Markdown | localStorage + `user_profiles.positioning_report` |
| 选题 | 选题池 | JSON | localStorage + `user_profiles.topic_pool` |
| 文案 | 文案脚本 | JSON | state（内存）+ `user_profiles.copywriting_data` |

> **注意：** localStorage key 已改为用户隔离格式（`result_{type}_{userId}`），登出时自动清除。

## 7. 缓存机制

### 7.1 参考文件缓存

参考文件只加载一次，缓存到 `refsCache`：

```typescript
const refsCache = useRef<Record<string, string>>({});

const getRefContent = async (path: string): Promise<string> => {
  if (refsCache.current[path]) return refsCache.current[path];
  const content = await loadRefFile(path);
  refsCache.current[path] = content;
  return content;
};
```

### 7.2 用户数据缓存

登录后自动缓存到 localStorage（key 带 `_{userId}` 后缀），登出时清除：

```typescript
// 生成用户隔离的 key
const getUserScopedKey = (baseKey: string, userId: string) => `${baseKey}_${userId}`;

// 登录后加载该用户的全部本地数据
const loadUserLocalData = (userId: string) => {
  // 加载访谈、定位、选题、文案等数据
};
```

## 8. JSON 解析策略

`parseJsonResponse(aiResponse)` 通用解析器，用于解析 AI 返回的 JSON（选题 + 文案共用）：

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| 1 | `JSON.parse` 直接解析 | AI 返回纯 JSON |
| 2 | 正则提取 ` ``` ` 代码块 | AI 用 Markdown 包裹 JSON |
| 3 | 提取第一个 `{` 到最后一个 `}` | JSON 前后有废话/说明文字 |
| 3b | `repairTruncatedJson` 补全括号 | AI 输出被截断 |

**调用方：**
- `parseTopicPool()` — 选题页面解析
- `generateCopywriting()` — 文案页面解析
